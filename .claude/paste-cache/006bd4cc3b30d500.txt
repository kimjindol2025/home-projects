### 확장된 Adaptive Attack Examples (AIOps 관점에서 5가지 대표 유형)

| # | 공격 이름 (논문 출처) | 구체적 공격 전략 | AIOps 환경에서의 치명적 영향 | 기존 Defense의 취약점 (왜 우리 ARES-FL이 필요한가) |
|---|-----------------------|------------------|-----------------------------|---------------------------------------------------|
| 1 | **Reconnecting Malicious Clients (RMC)**<br>(Szeląg et al., Adaptive Adversaries Survey 2025/510) | - 탐지/격리 → 즉시 disconnect<br>- stake penalty 기간 후 또는 새 세션으로 재접속<br>- 재접속 시 이전 공격 전략을 살짝 변형 (e.g. flip ratio 30% → 45%) | - AIOps peak-time (야간 배치 처리)에 재접속 → sudden false positive storm<br>- alert fatigue + missed critical anomaly | Aggregation-only 방법 (Krum, GeoMedian)은 identity tracking 없음 → 재접속 후 다시 poisoning 시작 |
| 2 | **Delayed / Timing-based Poisoning**<br>(Adaptive Adversaries Survey 2025 + arXiv:2510.01261) | - 처음 50~100 라운드 완전 honest (정상 gradient 제출)<br>- 특정 조건(peak traffic, maintenance window, 특정 task 활성화 시)에만 poisoning 실행 | - AIOps maintenance window 중 RCA task poisoning → 잘못된 root cause 지목 → cascading failure (장애 확산)<br>- Forecasting task에서만 공격 → autoscaling 실패 → 비용 폭증 | Single-round snapshot 기반 defense → early honest 행동으로 신뢰 쌓고 나중에 폭발 |
| 3 | **Strategy Evolution Attack**<br>(arXiv:2503.20884 GAN-based defense 논문, 2025) | - 공격자가 서버 aggregation rule을 관찰<br>- 초기: label-flipping<br>- 중기: gradient sign-flip<br>- 후기: adaptive scaling (global model과 cosine sim 0.95 이상 유지하면서 magnitude만 증폭) | - Multi-task AIOps에서 anomaly detection → RCA → forecasting 순으로 전략 변경<br>- cross-task contamination 극대화 | Static threshold / fixed rule (Krum, SignGuard) → 공격자가 실시간으로 rule을 우회하도록 진화 |
| 4 | **Adaptive Magnitude & Direction**<br>(SketchGuard 2026, arXiv:2510.07922) | - 현재 global model 상태를 추정<br>- poisoning 강도를 동적으로 조절 (e.g. 오늘은 1.2x scaling, 내일은 0.8x + 방향 미세 회전)<br>- Byzantine fraction f가 변동될 때마다 공격 벡터 재계산 | - AIOps time-series 데이터에서 gradual drift 유발 → 모델이 서서히 “blind” 해짐 (예측 정확도 95% → 60%로 천천히 하락) | 대부분 defense가 fixed f 가정 → dynamic f + adaptive magnitude에 취약 |
| 5 | **Hybrid Recon + Collusion**<br>(Adaptive Defences via DQN 2025, arXiv:2510.01261) | - 여러 malicious client가 collusion<br>- 한 클라이언트가 quarantine되면 다른 클라이언트가 “backup” 역할로 공격 강도 높임<br>- reconnect 후 전체 collusion network 재구성 | - 대규모 AIOps 클러스터 (수백 edge node)에서 10~20% malicious가 서로 협력 → 전체 시스템 신뢰도 붕괴 | Decentralized FL에서도 colluding adaptive group 대응 거의 없음 |

### AIOps 특화 Motivation 문장 (논문에 바로 사용 가능)
"In AIOps environments, adaptive Byzantine attacks are particularly devastating because they exploit the real-time and multi-task nature of operations. For instance, a reconnecting malicious client (RMC, Szeląg et al. 2025) can wait until a critical maintenance window to inject targeted RCA poisoning, causing cascading failures that traditional single-round robust aggregation methods (Krum, Geometric Median) completely fail to prevent. Similarly, strategy-evolving adversaries dynamically adjust their attack vectors based on observed global model states and aggregation rules, rendering static defenses brittle (Zafar et al. 2025). These adaptive behaviors, combined with temporal delays and cross-task propagation, represent the core open challenge that existing 2024–2026 literature has identified but not yet fully addressed in practical AIOps settings."
