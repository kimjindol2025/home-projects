📂 Step 1.1: 프로젝트 초기화 및 환경 설정
가장 먼저 클로드코드에게 다음 명령어를 실행하여 gogs 저장소의 물리적 공간을 만들라고 지시하세요.

그 후, src/main.rs를 아래의 표준 기초 코드로 덮어쓰기 합니다. 이 코드는 향후 어떤 복잡한 로직이 들어와도 무너지지 않을 '기본 프로토콜'입니다.
📝 [1.1.1] 변수와 불변성 (gogs_project/src/main.rs)
// 1학년 1학기: 변수와 데이터 타입 마스터
fn main() {
    // [저장 필수] 기록이 증명이다 gogs: 프로젝트 시작 로그
    println!("--- Gogs Project Level 1: Initialization ---");

    // 1. 불변 변수 (기본)
    let version: &str = "1.0.0";
    let student_name: &str = "Claude Code";

    // 2. 가변 변수 (mut 키워드 사용)
    let mut progress_level: i32 = 1;

    // 3. 상수 (컴파일 타임에 결정되는 값)
    const MAX_LEVEL: i32 = 10;

    println!("학생: {}, 현재 레벨: {}/{}", student_name, progress_level, MAX_LEVEL);
    
    // 레벨 업 시뮬레이션
    progress_level += 1;
    println!("레벨 업 성공! 다음 목표 레벨: {}", progress_level);
}

🛠️ Step 1.2: 클로드코드(학생) 교육 지침
클로드코드가 코드를 작성할 때 다음의 **'gogs 코딩 표준'**을 지키게 하세요. 이것이 10단계 시니어 등급으로 가는 약속입니다.
 * Strict Typing: 모든 let 선언 뒤에는 : i32, : f64와 같이 타입을 명시할 것. (타입 추론에만 의존하지 말 것)
 * Naming Convention: 변수명은 snake_case, 상수는 SCREAMING_SNAKE_CASE를 사용한다.
 * Documentation: 모든 함수 위에는 ///를 사용하여 이 함수가 어떤 '기록'을 남기는지 설명할 것.
🎓 선생님의 1단계 핵심 강의: "왜 러스트인가?"
어떤 언어를 개발하든 기초로 가져갈 수 있는 수준이 되려면 **메모리(Memory)**를 이해해야 합니다.
 * 1.1 변수: 메모리의 한 칸에 이름을 붙이는 행위입니다.
 * 1.2 불변성: 메모리에 쓴 내용을 "누가 함부로 수정하지 못하게 잠그는 것"입니다. 이것이 보안과 안정성의 시작입니다.
 * 1.3 강타입: "이 칸에는 숫자만 들어올 수 있어!"라고 선언하여, 문자가 들어와서 시스템이 붕괴되는 것을 원천 차단하는 것입니다.
