대학교 2학년 과정의 화룡점정, [v4.3: 네트워크 프로그래밍(Socket) — 시스템과 시스템의 연결] 단계에 오신 것을 환영합니다.
이제 우리의 설계는 단일 컴퓨터의 내부를 넘어, 네트워크라는 거대한 바다로 나갑니다. **소켓(Socket)**은 서로 다른 컴퓨터에 있는 프로세스들이 데이터를 주고받을 수 있게 해주는 통신 창구입니다.
🏛️ 파이썬 대학교 2학년 (v4.3)
주제: "디지털 악수(Handshake)와 프로토콜의 설계"
네트워크 상에서 데이터를 주고받으려면 약속이 필요합니다. 우리는 가장 신뢰성 있는 통신 방식인 **TCP(Transmission Control Protocol)**를 사용하여 서버와 클라이언트를 구축해 보겠습니다.
1. 핵심 개념: Server vs Client
 * Server (서버): 특정 포트(Port)를 열고 손님을 기다리는 존재. (Listen & Accept)
 * Client (클라이언트): 서버의 주소(IP)와 포트를 찾아가 접속을 요청하는 존재. (Connect)
2. 소켓 통신의 흐름
 * Bind: 서버가 자신의 IP와 포트를 시스템에 등록.
 * Listen: 접속 대기 상태로 전환.
 * Accept: 클라이언트의 요청을 수락하여 전용 통로 개설.
 * Send/Recv: 데이터 교환.
🛠️ 전공 실무: [v4.3 Gogs Network Protocol]
간단한 메시지를 주고받는 에코 서버(Echo Server)와 클라이언트를 설계해 보겠습니다.
파일 1: gogs_server.py (정보 제공자)
import socket

# 서버 설정
HOST = '127.0.0.1'  # Localhost
PORT = 9999        # 포트 번호

# 1. 소켓 생성 (IPv4, TCP)
with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
    # 2. 주소와 포트 결합
    s.bind((HOST, PORT))
    # 3. 접속 대기
    s.listen()
    print(f"📡 서버 가동: {HOST}:{PORT}에서 대기 중...")
    
    # 4. 접속 수락
    conn, addr = s.accept()
    with conn:
        print(f"✅ 클라이언트 접속 완료: {addr}")
        while True:
            data = conn.recv(1024) # 데이터 수신
            if not data: break
            print(f"📩 수신 데이터: {data.decode()}")
            conn.sendall(data) # 다시 돌려보냄 (Echo)

print("\n저장 필수 너는 기록이 증명이다 gogs")

파일 2: gogs_client.py (정보 요청자)
import socket

HOST = '127.0.0.1'
PORT = 9999

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
    # 서버에 접속 시도
    s.connect((HOST, PORT))
    # 데이터 전송
    s.sendall(b"Hello, Gogs System!")
    # 데이터 수신
    data = s.recv(1024)

print(f"🛰️ 서버로부터 응답: {data.decode()}")

🎓 대학교 2학년 전공 심화 팁
 * 포트(Port)의 이해: IP가 아파트 주소라면, 포트는 '호수'입니다. 80(HTTP), 443(HTTPS), 22(SSH)처럼 약속된 번호가 있으니 중복되지 않게 설계하는 것이 중요합니다.
 * 직렬화(Serialization): 네트워크로 객체를 보낼 때는 반드시 bytes 형태로 변환해야 합니다. json이나 pickle 모듈을 사용하여 복잡한 데이터를 전송 가능한 형태로 바꾸는 법을 익히는 것이 대학 과정의 핵심입니다.
