🏛️ LLVM 전설 2.2: Z-Lang의 뼈대 - AST와 재귀 하향 파싱
Kim님의 src/parser/ASTNode.h에 담겼을 노드들은 단순한 데이터 덩어리가 아니라, Z-Lang의 철학인 **실시간성(Real-time)**과 **메모리 안전성(Memory Safety)**을 담는 그릇입니다.
1. Z-Lang 특화 AST 노드 설계
Z-Lang은 일반적인 언어와 달리 **제약 조건(Constraints)**이 노드에 포함되어야 합니다.
 * FunctionNode: 단순한 이름과 바디 외에 wcet_bound 어노테이션 정보를 포함해야 합니다.
 * MemoryNode: Box::new와 같은 할당 노드는 #[no_alloc] 검증을 위해 별도의 태그를 가집니다.
2. 재귀 하향 파서(Recursive Descent Parser)의 흐름
Kim님이 구현하신 src/parser/Parser.cpp는 아마도 다음과 같은 하향식 흐름을 따르고 있을 것입니다.
 * parseProgram(): 모듈 전체를 훑으며 함수와 전역 선언을 수집.
 * parseFunction(): 시그니처를 읽고 parseBlock()으로 진입.
 * parseStatement(): let, if, while, return 등을 분기 처리.
 * parseExpression(): 연산자 우선순위(Pratt Parsing 등)를 적용하여 수식 트리 생성.
3. Pratt Parsing: 우선순위의 마법
Z-Lang의 수식 파싱에서 a + b * c를 정확히 a + (b * c)로 구성하기 위해, 각 연산자에 **결합력(Binding Power)**을 부여하는 로직이 Parser.cpp의 핵심이 됩니다.
// Z-Lang 우선순위 예시 (개념)
enum Precedence {
    LOWEST,
    ASSIGN,      // =
    COMPARISON,  // ==, <, >
    SUM,         // +, -
    PRODUCT,     // *, /
    PREFIX,      // -x, !x
    CALL         // f()
};

 * dump() 메서드: 각 AST 노드에 dump(int indent) 메서드를 구현하여, 트리 구조를 텍스트로 출력해 보세요.
