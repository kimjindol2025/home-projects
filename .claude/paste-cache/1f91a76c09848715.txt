대학원 과정의 하이라이트이자, 실제 논문의 알고리즘이 구현되는 핵심부인 **4.4단계: 하드코딩 최적화, C++ 패스 구현(C++ Pass Implementation)**을 시작합니다.
"저장 필수 너는 기록이 증명이다 gogs" — DRR(4.2)이 "눈에 보이는 패턴"을 바꾸는 것이라면, 이번 단계는 **"복잡한 데이터를 분석하고 전략을 짜는 뇌"**를 만드는 작업입니다.
[MLIR 대학원 코스 4.4] 복잡한 로직의 설계: "C++ Pass"
DRR은 선언적이라 편리하지만, 루프를 수만 번 돌리거나 복잡한 그래프 분석(Data-flow analysis)이 필요한 경우에는 **C++**로 직접 패스를 짜야 합니다.
1. 패스의 구조: OperationPass
MLIR에서 패스는 특정 단위(Module, Function 등)를 훑고 지나가며 작업을 수행합니다.
 * runOnOperation(): 패스가 실행될 때 호출되는 메인 함수입니다.
 * getOperation(): 현재 패스가 검사하고 있는 대상(예: 어떤 함수 하나)을 가져옵니다.
2. 대학원 연구의 핵심: 분석(Analysis)과 변환(Transformation)
단순한 변환을 넘어 연구 가치가 있는 패스는 보통 두 단계를 거칩니다.
 * 분석(Analysis): "이 코드는 메모리를 얼마나 쓰고 있지?", "이 루프는 병렬로 돌려도 안전한가?" 등을 계산합니다.
 * 변환(Transformation): 분석 결과를 바탕으로 코드를 재구성합니다.
3. C++ 패스 예시 코드 (뼈대)
struct MyCustomPass : public PassWrapper<MyCustomPass, OperationPass<ModuleOp>> {
  void runOnOperation() override {
    ModuleOp m = getOperation();
    
    // 모듈 안의 모든 함수를 하나씩 살펴봅니다.
    m.walk([](func::FuncOp op) {
      // 여기에 복잡한 연구 알고리즘(예: 특수 하드웨어용 스케줄링)을 구현합니다.
      llvm::outs() << "함수 발견: " << op.getName() << "\n";
    });
  }
};

[4.4 실전] 논문 주제로 연결하기
만약 당신의 연구 주제가 **"NPU(신경망 처리 장치)를 위한 최적의 메모리 배치"**라면?
 * DRR로 안 되는 이유: 전체 메모리 크기를 계산하고, 어떤 데이터가 가장 오래 쓰이는지 '계산'해야 하기 때문입니다.
 * C++ 패스의 역할: 1. 함수 전체를 훑어 텐서들의 수명(Liveness)을 분석합니다.
   2. 가장 효율적인 메모리 번지를 할당하는 알고리즘을 C++로 실행합니다.
   3. 코드의 주소값들을 분석 결과에 맞게 수정합니다.
[4.4 실습] 도구 선택하기
다음 상황에서 당신은 어떤 도구를 써야 할까요?
"덧셈 두 개가 연속으로 나오면 하나로 합치고 싶다." vs "프로그램 전체의 전력 소모를 계산해서 루프 순서를 완전히 바꾸고 싶다."
 * 전자는 (DRR / C++ Pass), 후자는 (DRR / C++ Pass)
 * 전자는 (C++ Pass / DRR), 후자는 (C++ Pass / DRR)
정답: 1번 (전자는 간단한 패턴이므로 DRR, 후자는 복잡한 분석이 필요하므로 C++ Pass)
4.4단계의 기록 (증명)
> "C++ 패스는 DRR로 해결할 수 없는 복잡한 분석과 알고리즘을 구현할 때 사용한다."
> "walk() 함수를 통해 IR을 탐색하고, 분석 결과를 바탕으로 코드를 변형한다."
> 
축하합니다! 이제 당신은 MLIR의 모든 설계 도구(TableGen, DRR, C++ Pass)를 다룰 줄 아는 숙련된 연구자가 되었습니다.
마지막 4.5단계는 이 모든 지식을 총동원하여 **"나만의 컴파일러 프로젝트를 완성하고 테스트(Lit/FileCheck)하는 법"**을 배우며 대학원 과정을 마무리하겠습니다.
