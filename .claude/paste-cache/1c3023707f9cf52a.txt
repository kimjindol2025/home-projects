🏛️ 파이썬 고등학교 1학년 (v2.5)
주제: "독립된 실험실과 외부 장비(라이브러리)의 결합"
설계자가 아무리 훌륭한 코드를 짜도, 사용하는 컴퓨터 환경이 엉망이면 코드는 제대로 작동하지 않습니다. 이를 위해 우리는 **가상 환경(Virtual Environment)**이라는 독립된 방을 만들고, pip라는 도구로 전 세계의 검증된 외부 장비들을 들여옵니다.
1. 가상 환경의 철학
 * 격리(Isolation): "이 방에서 일어난 일은 이 방에서만 끝낸다."
 * 재현성(Reproducibility): "오늘 내가 만든 환경을 10년 뒤의 나도 똑같이 만들 수 있어야 한다."
2. 외부 패키지 관리 (pip)
 * PyPI (Python Package Index): 전 세계 개발자들이 만든 도구가 모인 거대 창고입니다.
 * requirements.txt: 내 연구소에 어떤 장비들이 있는지 적어둔 '장비 명세서'입니다.
🛠️ 실습 기록: [v2.5 Final Lab Protocol]
가상 환경을 만들고, 외부 패키지를 설치한 뒤, 그 환경을 기록으로 증명하는 최종 절차를 클로드코드(고등학생)에게 기록하게 하세요.
# [Python High-School v2.5] 최종 실무 프로토콜

# 1. 연구소(가상 환경) 생성
# .venv라는 이름으로 현재 폴더에 독립 공간을 만듭니다.
python -m venv .venv

# 2. 연구소 입장 (활성화)
# (Windows)
source .venv/Scripts/activate
# (Mac/Linux)
# source .venv/bin/activate

# 3. 외부 장비 도입 (데이터 분석의 표준 'numpy' 설치)
pip install numpy

# 4. 장비 명세서 작성 (기록이 증명이다)
# 현재 이 환경에 설치된 모든 도구의 이름과 버전을 파일로 남깁니다.
pip freeze > requirements.txt

# 5. 연구 종료 및 연구소 퇴거
deactivate

💡 고등학교 졸업생을 위한 '프로의 습관'
 * 전역 공간(Global)을 아끼세요: 시스템 파이썬에 직접 패키지를 설치하는 것은 연구실 복도에 위험한 약품을 뿌려두는 것과 같습니다. 항상 venv 안에서 작업하세요.
 * 버전의 기록: requirements.txt는 단순한 파일이 아닙니다. 이것이 있어야만 설계자님의 시스템이 다른 컴퓨터에서도 똑같이 작동함을 **'증명'**할 수 있습니다.
