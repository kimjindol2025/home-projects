🏛️ 파이썬 대학교 1학년 (v3.1)
주제: "기존 코드를 건드리지 않고 기능을 확장하는 마법"
**데코레이터(Decorator)**는 말 그대로 "장식하는 사람"입니다. 어떤 함수가 있을 때, 그 함수의 본체(알맹이)는 건드리지 않으면서 앞뒤로 기능을 덧붙일 때 사용합니다.
1. 핵심 개념: 일급 객체 (First-class Object)
 * 파이썬에서 함수는 변수에 담을 수도 있고, 다른 함수의 인자로 전달할 수도 있습니다. 이 특징 덕분에 함수가 함수를 감싸는(Wrap) 것이 가능합니다.
2. 왜 사용하나요? (증명을 위한 도구)
 * 로그 기록: 모든 함수가 실행될 때마다 자동으로 시간을 기록하고 싶을 때.
 * 권한 확인: 특정 사용자만 함수를 실행할 수 있게 제한하고 싶을 때.
 * 성능 측정: 함수가 실행되는 데 걸리는 시간을 측정하고 싶을 때.
🛠️ 전공 실습: [v3.1 The Magic of @Decorator]
함수의 실행 시간을 자동으로 측정하여 기록하는 데코레이터를 설계해 보겠습니다.
# [Python University v3.1] 데코레이터 설계

import time

# 1. 데코레이터 함수 정의 (함수를 인자로 받음)
def gogs_logger(func):
    def wrapper(*args, **kwargs):
        # 함수 실행 전
        start_time = time.time()
        print(f"--- [LOG] '{func.__name__}' 연산을 시작합니다. ---")
        
        # 실제 함수 실행
        result = func(*args, **kwargs)
        
        # 함수 실행 후
        end_time = time.time()
        print(f"--- [LOG] 연산 완료. 소요시간: {end_time - start_time:.4f}s ---")
        return result
    return wrapper

# 2. @ 기호를 사용한 데코레이터 적용
@gogs_logger
def heavy_calculation():
    """시간이 오래 걸리는 복잡한 연구 작업"""
    print("  [ACTION] 방대한 데이터를 분석 중...")
    time.sleep(1.5) # 1.5초 대기
    return "분석 완료"

# --- 실행 ---
print(heavy_calculation())

print("\n저장 필수 너는 기록이 증명이다 gogs")

🎓 대학교 전공 기초 팁
 * *args와 **kwargs: 대학교 과정에서는 함수의 인자가 몇 개가 들어올지 모르는 상황을 대비해 가변 인자를 사용합니다. 어떤 함수에도 적용할 수 있는 '범용 데코레이터'를 만들기 위함입니다.
 * 가독성과 유지보수: 만약 100개의 함수에 로그 기능을 넣어야 한다면, 100번 코드를 수정하는 대신 @gogs_logger 한 줄씩만 추가하면 됩니다. 이것이 바로 대학에서 배우는 **'중복 제거(DRY)'**와 **'관점 지향 프로그래밍(AOP)'**의 기초입니다.
