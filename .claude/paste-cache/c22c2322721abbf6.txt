클로드코드(학생)에게 문자열에서 첫 번째 단어만 안전하게 추출하는 '부분 참조' 로직을 기록하게 하세요.
// [제3장: 모듈화의 초석 v4.3]

fn main() {
    println!("--- v4.3 Modular Design: Slices & Partial Access ---");

    let system_command = String::from("REBOOT SYSTEM_CORE_01");

    // [v4.3 설계] 문자열 슬라이스 (&str) 생성
    // 0번부터 6번 인덱스 전까지(0~5)의 부분을 참조
    let command_only = &system_command[0..6]; 
    let target_only = &system_command[7..]; // 7번부터 끝까지

    println!("v4.3 분석 결과:");
    println!("  명령어: [{}]", command_only);
    println!("  대 상: [{}]", target_only);

    // 배열 슬라이스 예시
    let sensor_values = [10, 20, 30, 40, 50];
    let partial_data = &sensor_values[1..4]; // [20, 30, 40]
    println!("  센서 부분 데이터: {:?}", partial_data);

    println!("\n기록: 슬라이스를 통해 복사 없이 데이터의 특정 영역을 안전하게 식별함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 가이드: "v4.3 설계자의 안목"
슬라이스를 설계할 때 반드시 기억해야 할 철학이 있습니다.
 * 소유권과의 동기화: 슬라이스는 원본에 **'대여'**된 상태입니다. 슬라이스가 살아있는 동안 원본 문자열을 수정(clear() 등)하려고 하면 컴파일러가 "누군가 보고 있는데 고치면 안 돼!"라며 막아줍니다.
 * 문자열 슬라이스(&str)의 위상: 러스트에서 문자열 리터럴("hello")은 사실 그 자체로 슬라이스입니다. 따라서 함수 인자를 &str로 설계하는 것이 가장 표준적인 방식입니다.
 * 경계값 주의: [0..100]처럼 실제 데이터 범위를 벗어나는 슬라이스를 생성하려 하면 런타임에 에러(Panic)가 발생하므로 설계 시 범위를 신중히 계산해야 합니다.
