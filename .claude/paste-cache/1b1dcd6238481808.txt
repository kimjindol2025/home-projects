선언적 매크로(macro_rules!)를 통해 패턴 매칭의 맛을 보셨다면, 이제 매크로의 '최종 병기'를 꺼낼 차례입니다. [제12장: 매크로 — v13.1 절차적 매크로(Procedural Macros): 코드 분석과 주입] 단계로 진입합니다.
선언적 매크로가 단순한 '치환'이라면, 절차적 매크로는 **'러스트 코드를 입력받아 러스트 함수로 분석한 뒤, 완전히 새로운 코드를 출력하는 프로그램'**입니다. 이것은 사실상 컴파일러 내부에서 돌아가는 또 하나의 미니 컴파일러입니다.
🛠️ 제12장: 매크로 (v13.1) — [절차적 매크로의 삼각 편대]
1. 설계 목표: "코드를 데이터처럼 주무르기"
 * TokenStream: 러스트 코드가 쪼개진 토큰들의 스트림입니다. 절차적 매크로의 입출력 형식입니다.
 * syn (Parsing): 복잡한 TokenStream을 분석하여 구조체 이름, 필드, 타입 등을 의미 있는 **데이터 구조(AST)**로 바꿔주는 도구입니다.
 * quote (Generation): 우리가 생성하고 싶은 러스트 코드를 마치 템플릿처럼 작성하면 다시 TokenStream으로 바꿔주는 도구입니다.
2. 설계자의 관점: "컴파일 타임의 마법사"
절차적 매크로는 별도의 크레이트(라이브러리)로 작성되어야 하며, 컴파일러가 우리 코드를 빌드하기 전에 먼저 실행됩니다. 설계자는 이를 통해 **'반복되는 인터페이스 구현'**이나 **'복잡한 데이터 직렬화'**를 자동화합니다.
🏗️ gogs 저장소 v13.1 설계: 자동 메타데이터 주입기 (Derive Macro)
구조체에 자동으로 "제작자 정보"와 "기록 시간"을 출력하는 메서드를 주입하는 #[derive(GogsInfo)] 매크로의 내부 논리를 설계합니다.
// [제12장: 매크로 v13.1 - 절차적 매크로 설계 의사코드]

// 주의: 절차적 매크로는 별도의 proc-macro = true 크레이트에서 작성해야 합니다.
extern crate proc_macro;
use proc_macro::TokenStream;
use quote::quote;
use syn;

#[proc_macro_derive(GogsInfo)]
pub fn gogs_info_derive(input: TokenStream) -> TokenStream {
    // 1. [Parsing] 입력받은 코드를 분석 가능한 구조(AST)로 파싱
    let ast = syn::parse(input).expect("파싱 실패");

    // 2. [Extraction] 구조체의 이름을 추출
    let name = &ast.ident;

    // 3. [Generation] 추가할 코드를 생성 (quote! 매크로 사용)
    let gen = quote! {
        impl #name {
            fn print_gogs_info(&self) {
                println!("--- Meta Info for {} ---", stringify!(#name));
                println!("저장 필수 너는 기록이 증명이다 gogs");
                println!("Generated at: 2026-02-23");
            }
        }
    };

    // 4. [Output] 생성된 코드를 다시 TokenStream으로 변환하여 반환
    gen.into()
}

🎓 선생님의 세부 가이드 (13.1): "강력한 힘에는 책임이 따른다"
 * 빌드 시간의 증가: 절차적 매크로는 컴파일 중에 별도의 프로그램을 실행하는 것이므로, 너무 많이 사용하면 빌드 속도가 느려집니다. 꼭 필요한 곳에만 사용하십시오.
 * 디버깅의 어려움: 매크로가 생성한 코드는 소스 파일에 보이지 않습니다. cargo-expand 도구를 사용하여 매크로가 실제로 어떤 코드를 뱉어냈는지 확인하는 습관을 들여야 합니다.
 * 컴파일러 프론트엔드 예습: syn을 사용하여 코드를 파싱하는 과정은 우리가 14단계에서 만들 컴파일러의 프론트엔드 로직과 거의 흡사합니다. 이 과정을 즐기며 익히십시오.
> 저장 필수 너는 기록이 증명이다 gogs
> 
이제 당신은 단순히 코드를 짜는 사람이 아니라, 코드를 분석하고 재구성하는 시스템 설계자의 반열에 올랐습니다.
