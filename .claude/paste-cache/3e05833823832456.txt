🎓 Zig 전공 101: 1-5. 구조체(Structs)와 메서드
Zig의 구조체는 단순히 데이터를 담는 바구니가 아닙니다. 데이터의 배치(Layout)를 결정하고, 관련 함수를 내부에 포함하여 객체지향적인 설계를 가능하게 합니다.
1. 구조체 정의와 초기화
구조체는 struct 키워드로 정의하며, 모든 필드는 이름과 타입을 가져야 합니다.
const std = @import("std");

const User = struct {
    id: u32,
    name: []const u8,
    is_active: bool = true, // 기본값 설정 가능
};

pub fn main() void {
    // 구조체 인스턴스 생성
    const user1 = User{
        .id = 1,
        .name = "Zig-Scholar",
    };
    
    std.debug.print("User: {s} (ID: {d})\n", .{user1.name, user1.id});
}

2. 구조체 메서드 (Methods)
Zig에서 메서드는 구조체 블록 안에 정의된 함수일 뿐입니다. 첫 번째 매개변수로 자기 자신(self)을 받으면 메서드처럼 동작합니다.
 * self: User: 값을 복사해서 전달 (Read-only 권장)
 * self: *User: 포인터를 전달하여 내부 값을 수정 가능
<!-- end list -->
const Rectangle = struct {
    width: f32,
    height: f32,

    // 면적 계산 메서드
    pub fn area(self: Rectangle) f32 {
        return self.width * self.height;
    }

    // 크기 변경 메서드 (포인터 사용)
    pub fn scale(self: *Rectangle, factor: f32) void {
        self.width *= factor;
        self.height *= factor;
    }
};

3. 네임스페이스로서의 구조체
Zig의 구조체는 관련 함수들을 묶어주는 네임스페이스 역할도 합니다. self를 받지 않는 함수는 정적 메서드(Static Method)처럼 사용됩니다. 보통 생성자 역할을 하는 init 함수를 이렇게 만듭니다.
const Point = struct {
    x: i32,
    y: i32,

    pub fn init(x: i32, y: i32) Point {
        return Point{ .x = x, .y = y };
    }
};

// 사용: const p = Point.init(10, 20);

🔬 전공 심화: 필드 정렬과 메모리 (Packed & Extern)
대학원 수준의 설계에서는 메모리 레이아웃도 신경 써야 합니다.
 * struct: 컴파일러가 성능 최적화를 위해 필드 순서를 임의로 바꿀 수 있습니다.
 * packed struct: 비트 단위까지 필드 순서를 엄격히 지킵니다. (하드웨어 제어 시 필수)
 * extern struct: C언어의 구조체 레이아웃과 호환되도록 보장합니다.
📦 익명 구조체 (Anonymous Structs)
함수에 여러 인자를 한꺼번에 전달하거나, 임시 데이터를 묶을 때 유용합니다.
std.debug.print에서 사용하는 .{ .name = "value" } 형태가 바로 익명 구조체입니다.
