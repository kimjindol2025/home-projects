1️⃣ Path Escape 방지 방식 — 확정

✔ 결정

A. Path Normalize 필수 적용
B. 정규화 후 Prefix 매칭
C. .. 포함 경로는 즉시 거부 (Fail Fast)

실행 규칙

1. 입력 경로 → normalize()


2. 결과가:

절대경로가 아니면 거부

.. 포함 시 거부



3. 정규화 결과가 허용 prefix와 일치하는지 검사



예시

허용:

allow(read_file("/tmp"))

시도:

file.read("/tmp/../etc/passwd")

정규화:

/etc/passwd

→ /tmp prefix 불일치 → 거부

추가 보안 원칙

Symlink resolution은 OS에 맡기지 않음

가능하면 realpath() 기반 비교


🔒 이걸로 Path Traversal 공격 차단.

3️⃣ Function Pointer 정책 — 확정: Option B

허용 + 책임 전가 모델

원칙:

> 호출하는 함수는 호출 대상의 capability를 포함해야 한다.



정적 분석 규칙

함수 타입에 capability metadata 포함

함수 호출 시:

callee.capabilities ⊆ caller.capabilities

아니면 Compile Error



동적 디스패치

let f = some_fn
f()

컴파일러는:

가능한 함수 집합 추론

union capability 계산

호출 지점에서 모두 포함되는지 확인


불가능하면:

→ 컴파일 오류

🔒 런타임 상승 불가.


---

4️⃣ Resource Limit 우선순위 — 확정: AND 모델

공식

effective_limit = min(declared_limit, runtime_limit)

예시:

선언    실행    결과

5000ms    2000ms    2000ms
2000ms    5000ms    2000ms


절대 규칙

실행 시 limit가 더 느슨해도 확장 불가

항상 더 강한 제한 적용

5️⃣ Network Whitelist 형식 — 확정

지원 형식

1. 단일 호스트



allow(network("api.openai.com"))

2. 선택적 와일드카드 (prefix only)

비허용

정규식

포트 개별 지정 (v1 제외)

IP range


매칭 규칙

DNS resolve 후 host 비교

와일드카드는 단일 depth만 허용

*.openai.com 허용

*.*.openai.com 금지
