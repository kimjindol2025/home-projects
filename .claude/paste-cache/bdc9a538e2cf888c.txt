2.2.1 loop와 match를 결합한 상태 제어 (State Control)
이 패턴은 나중에 언어 인터프리터를 만들 때 '명령어 해석기'의 기초가 됩니다.
// src/main.rs 에 복사하여 기초로 사용
fn main() {
    let mut step_count = 0;

    // gogs 저장소의 첫 번째 가동 로그
    println!("--- Gogs Project: Level 1-2 Startup ---");

    let final_status = loop {
        step_count += 1;

        // [2.1.3] match를 통한 정교한 상태 분기
        match step_count {
            5 => println!("중간 점검: 5단계 통과"),
            10 => {
                println!("목표 달성: 10단계 완료");
                break "SUCCESS"; // 루프 탈출 및 결과 반환
            }
            _ => {
                // 아무것도 하지 않음 (Placeholder)
            }
        }

        if step_count > 100 { break "ERROR"; } // 안전 장치
    };

    println!("최종 빌드 상태: {}", final_status);
}

🛠️ 2.3 클로드코드 전용 '에러 방지' 가이드라인
클로드코드에게 코드를 짜라고 시킬 때, 다음 3가지 원칙을 gogs 저장소 상단에 명시하세요.
 * Immutability First: 모든 변수는 let으로 시작하고, 정말 필요한 경우에만 let mut을 사용한다.
 * Explicit Types: 러스트의 타입 추론에만 의존하지 말고, 핵심 변수에는 반드시 타입을 명시한다. (예: let count: i32 = 0;)
 * No Dead Code: 사용하지 않는 루프나 조건문은 즉시 제거하여 기록의 순수성을 유지한다.
