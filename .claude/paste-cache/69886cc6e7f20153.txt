🎓 Go-언어 전공 101: 1-5. 배열(Array)과 슬라이스(Slice)
1. 배열 (Array): 고정된 크기의 기초
배열은 메모리에 연속적으로 할당된 고정 크기의 데이터 집합입니다.
 * 특징: 선언 시 크기를 정해야 하며, 한 번 정한 크기는 바꿀 수 없습니다.
 * 학술적 관점: 배열의 크기 또한 '타입'의 일부입니다. 즉, [3]int와 [5]int는 서로 다른 타입으로 간주됩니다.
<!-- end list -->
var grades [3]int = [3]int{90, 85, 95}

2. 슬라이스 (Slice): 동적 배열의 유연성
실무에서 배열보다 훠씬 많이 쓰이는 도구입니다. 크기가 가변적이며, 내부적으로는 배열을 가리키는 '포인터' 구조를 가집니다.
// 슬라이스 선언 (크기를 지정하지 않음)
languages := []string{"Go", "Python", "C++"}

// 요소 추가
languages = append(languages, "Rust") 

3. 슬라이스의 내부 구조 (The Slice Header)
대학원 수준의 시스템 프로그래밍을 하려면 슬라이스가 내부적으로 어떻게 구성되는지 알아야 합니다. 슬라이스는 다음 3가지 필드를 가진 구조체와 같습니다.
 * Pointer: 실제 데이터가 담긴 배열의 시작 주소
 * Length (len): 현재 슬라이스에 담긴 요소의 개수
 * Capacity (cap): 시작 주소부터 버킹 배열(Backing Array)의 끝까지 담을 수 있는 최대 개수
🔬 전공 심화: 복사 vs 참조 (Pass by Value)
Go에서 모든 인자는 **'값에 의한 전달(Pass by Value)'**입니다.
 * 배열을 함수에 넘기면 배열 전체가 복사되어 메모리 낭비가 심해집니다.
 * 반면, 슬라이스를 넘기면 슬라이스 헤더(포인터, 길이, 용량 정보)만 복사되므로 매우 효율적입니다. 실제 데이터는 원본과 같은 배열을 공유합니다.
<!-- end list -->
func updateFirst(s []int) {
    s[0] = 999 // 원본 슬라이스의 값도 바뀝니다! (같은 배열을 참조하므로)
}
