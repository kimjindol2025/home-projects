▶ v2.K.4 — Parallel Policy v1 (MaxParallel + Scheduler 결합)

목표

Policy 레벨에서 병렬 실행 제어를 도입하되,
Kernel은 동시성 판단을 하지 않는다.


---

핵심 원칙

Kernel → 실행 가능 Task 집행만

Policy → 몇 개를 동시에 실행할지 결정

Scheduler → 어떤 Task를 선택할지 결정

Constraint → 병렬과 무관하게 hard stop



---

진행 순서

1. ParallelExecutionPolicy 구현


2. MaxParallel 설정 필드 추가


3. 현재 Running Task 수 추적 구조 구현


4. onTaskStart / onTaskEnd 이벤트 연동


5. Scheduler와 결합하여 N개 선택




---

정책 동작 모델

ReadyTasks → Scheduler 선택 → MaxParallel 만큼 실행
Task 완료 → 슬롯 해제 → 다음 Task 선택


---

구현 요구

MaxParallel = 1 → 순차와 동일 동작

MaxParallel > 1 → 동시 실행 허용

Running 상태 정확 추적

실패 시:

Conservative 모드: 전체 중단

Optimistic 모드: 영향 범위만 차단


Memory append-only 유지 (thread-safe 설계)



---

테스트 기준 (최소 8개)

1. MaxParallel=1 → 순차 실행


2. MaxParallel=2 → 2개 동시 실행


3. 3개 Ready 중 2개만 실행


4. Task 완료 시 슬롯 해제


5. 중간 실패 시 정책별 분기


6. Constraint 위반 시 즉시 전체 중단


7. Scheduler 교체 가능


8. 상태 경합 없음 검증




---

완료 기준

기존 모든 테스트 통과

병렬 전용 테스트 통과

무한 루프 / 경합 없음
