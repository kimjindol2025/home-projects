현재 상태 (v5 / v4-ide 중심)
- tree-walking 인터프리터 기반 + ref counting GC
- 기본 문법 지원 (변수, 함수, if/for/while, list/dict, 파일 I/O, try-except 등)
- 일부 stdlib + REPL 존재
- 하지만 파이썬 대비 표준 라이브러리·DX·성능·모듈 시스템 모두 부족
- 목표: 이 기반을 유지하면서 **Intent 중심 자연어 선언 패러다임**으로 전환

<핵심 목표 철학 – 반드시 지키기>
- 인간 입력 = 고수준 의도 선언 (goal, agent, flow, constraints 등) + 자연어 섞기 가능
- 나머지 90% 이상 = AI(내장 LLM + 런타임)가 자동 생성·최적화·디버깅·배포
- 파이썬의 GIL·의존성·디버깅·보일러플레이트 문제를 근본 해결

<새 언어 이름>
Intent (또는 Agentic, Sage 중 선택 – Intent 추천)

<필수 적용 요소>
1. 문법 재설계: 전통 키워드 최소화 → agent, goal, flow, listen, when, tool, memory, constraint, deploy, explain, using 등 고수준 선언 중심
2. AI primitive 내장: Agent, Model, Tool, Memory, Planner, Intent 타입
3. 런타임 LLM 역할: Claude/GPT-4o 등으로 Intent → 실행 코드 자동 생성 + 자동 수정/최적화
4. 성능 현실화: 기본 parallel 지원, JIT/AOT 옵션, GIL 제거 방향 (Rust 백엔드 일부 도입 고려)
5. 배포: deploy to fly.io / lambda / edge 한 줄 명령
6. 비용·신뢰 관리: 캐싱, 로컬 모델 폴백, 자동 검증, human-in-the-loop 옵션
