[Post-Doc Phase 3] 하드웨어 인지형 최적화 (Hardware-Aware Pass)
우리가 설계할 패스의 핵심 로직은 다음과 같습니다:
 * 분석(Analysis): 현재 루프에서 사용하는 메모리 크기가 하드웨어의 로컬 SRAM(예: 128KB)을 초과하는가?
 * 의사결정(Decision): 초과한다면, SRAM에 쏙 들어갈 수 있는 최대 타일 크기(T)를 수학적으로 계산한다.
 * 변환(Transformation): 계산된 T를 바탕으로 루프를 실제로 쪼갠다(Tiling).
1. C++ 패스 내부 로직 (추상화 코드)
박사님의 연구를 위해 작성된 C++ 패스의 핵심 뼈대입니다.
// 하드웨어 제약 조건을 고려한 타일링 패스
struct AccelTilingPass : public PassWrapper<AccelTilingPass, OperationPass<func::FuncOp>> {
  const int64_t MAX_SRAM = 128 * 1024; // 하드웨어 SRAM 제약: 128KB

  void runOnOperation() override {
    getOperation().walk([&](linalg::MatmulOp op) {
      // 1. 현재 연산의 메모리 사용량 분석
      int64_t currentUsage = calculateMemoryUsage(op);

      if (currentUsage > MAX_SRAM) {
        // 2. 최적의 타일 크기 계산 (수학적 모델링)
        // 예: Usage = (T*T + T*T + T*T) * sizeof(float) <= MAX_SRAM
        int64_t optimalTileSize = std::sqrt(MAX_SRAM / (3 * sizeof(float)));

        // 3. 실제 IR 변환 수행 (Tiling)
        applyTiling(op, optimalTileSize);
        
        llvm::outs() << "최적화 완료: 타일 크기 " << optimalTileSize << " 적용됨.\n";
      }
    });
  }
};

2. 연구 데이터: 성능 예측 모델
박사님은 이 패스를 통해 다음과 같은 성능 향상 데이터를 논문에 제시할 수 있습니다.
| 타일 크기 (T) | SRAM 사용량 | 외부 메모리 접근(DRAM) 횟수 | 예상 지연 시간(Latency) |
|---|---|---|---|
| No Tiling | 4.0 MB (Over) | 매우 높음 | 100ms |
| T=128 | 192 KB (Over) | 높음 | 65ms |
| T=64 (Optimal) | 48 KB (Safe) | 최적화됨 | 12ms |
3. 독립 연구자의 심화 과제: "Double Buffering"
단순히 타일링을 하는 것만으로는 부족합니다. 하드웨어가 연산을 하는 동안 다음 타일 데이터를 미리 가져오는 더블 버퍼링(Double Buffering) 기법을 이 패스에 통합해야 합니다.
 * 박사님의 설계 전략: 1. 메모리 공간을 두 영역(Ping-Pong)으로 나눕니다.
   2. async.token을 활용하여 데이터 전송과 연산을 비동기적으로 스케줄링합니다.
