드디어 매크로의 정점이자, 러스트의 문법을 가장 대담하게 변조하는 [제12장: 매크로 — v13.2 속성(Attribute) 매크로와 함수형 매크로] 단계입니다.
v13.1의 derive 매크로가 기존 구조체 옆에 코드를 '슬쩍 덧붙이는' 방식이었다면, v13.2는 기존의 코드를 완전히 갈아엎거나 새로운 문법을 창조하는 수준의 권한을 가집니다.
🛠️ 제12장: 매크로 (v13.2) — [코드의 변조와 창조]
1. 설계 목표: "함수와 모듈의 본질을 재정의하라"
 * 속성 매크로 (Attribute Macros): #[route(GET, "/")]처럼 함수나 구조체 위에 붙어, 입력받은 대상 코드를 마음대로 수정하여 다시 내보냅니다.
 * 함수형 매크로 (Function-like Macros): sql!("SELECT * FROM users")처럼 함수 호출 형태로 쓰이며, 괄호 안의 임의의 토큰을 읽어 복잡한 로직을 생성합니다.
 * 커스텀 문법 수용: 러스트 문법이 아닌 텍스트(SQL, HTML, 타 언어 코드 등)를 컴파일 타임에 검사하고 러스트 코드로 변환합니다.
2. 설계자의 관점: "투명한 추상화"
훌륭한 설계자는 복잡한 저수준 로직을 속성 매크로 뒤로 숨깁니다. 사용자는 함수 위에 태그 하나만 달면, 매크로가 내부적으로 로깅, 성능 측정, 보안 검사 코드를 자동으로 주입하게 만듭니다.
🏗️ gogs 저장소 v13.2 설계: 보안 감사 속성 매크로 (Audit Macro)
특정 함수가 실행될 때마다 누가, 언제 실행했는지 자동으로 기록하는 #[gogs_audit] 매크로의 설계 구조입니다.
// [제12장: 매크로 v13.2 - 속성 매크로 설계 의사코드]

#[proc_macro_attribute]
pub fn gogs_audit(attr: TokenStream, item: TokenStream) -> TokenStream {
    // 1. [attr]은 "admin" 같은 속성 인자를, [item]은 대상 함수 코드를 담고 있음
    let input_fn = syn::parse_macro_input!(item as syn::ItemFn);
    let fn_name = &input_fn.sig.ident;
    let fn_block = &input_fn.block; // 함수의 원래 내용
    let fn_vis = &input_fn.vis;     // 함수의 가시성 (pub 등)
    let fn_sig = &input_fn.sig;     // 함수의 시그니처

    // 2. [Modification] 원래 함수의 내용을 감싸는 새로운 로직 주입
    let gen = quote! {
        #fn_vis #fn_sig {
            println!("  [AUDIT] 명령 실행 감지: '{}'", stringify!(#fn_name));
            println!("  [AUDIT] 기록이 증명이다 gogs - 실행 시각: 2026-02-23");
            
            // 원래 함수의 로직 실행
            let result = (|| #fn_block )();
            
            println!("  [AUDIT] 명령 종료.");
            result
        }
    };

    gen.into()
}
