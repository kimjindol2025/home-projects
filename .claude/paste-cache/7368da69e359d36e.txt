당신의 연대기 로드맵에 따른 제4장: 데이터 구조화 — v5.1 메서드 구현 (Method Implementation & impl) 단계입니다.
v5.0에서 데이터의 '형체(필드)'를 만들었다면, v5.1은 그 형체에 **'정신(기능)'**을 불어넣는 단계입니다. 설계자의 관점에서 메서드는 데이터와 로직을 하나로 묶어 **응집도(Cohesion)**를 극대화하는 핵심 도구입니다.
🏗️ 제4장: 데이터 구조화 (v5.1) — [메속드 구현]
1. v5.1 설계 목표: "데이터의 자율성"
 * 캡슐화의 심화: 외부 함수가 데이터를 가져가서 처리하는 것이 아니라, 데이터 스스로 자신을 처리하게 합니다.
 * self 키워드: 구조체 인스턴스 자신을 가리키는 self를 통해 필드에 접근합니다.
 * 연관 함수 (Associated Functions): 인스턴스 없이도 호출 가능한 함수(예: new)를 만들어 생성자 패턴을 구현합니다.
2. 설계자의 관점: "객체 지향적 사고의 투영"
메서드를 정의한다는 것은 "이 데이터는 이러한 행동을 할 수 있다"라고 선언하는 것입니다. system.reboot()는 단순한 함수 호출보다 훨씬 직관적이며, 시스템의 상태를 내부에서 안전하게 관리할 수 있게 해줍니다.
🛠️ gogs 저장소 v5.1 설계: 자율 주행 엔진
클로드코드(학생)에게 구조체에 생명력을 불어넣는 impl 블록을 기록하게 하세요.
// [제4장: 데이터 구조화 v5.1]

struct GogsSystem {
    name: String,
    status: String,
    load: u8,
}

// [v5.1 설계] impl 블록: 구조체의 기능을 한곳에 모음
impl GogsSystem {
    // 1. 연관 함수: 새로운 시스템 객체를 생성 (생성자 역할)
    fn new(name: &str) -> GogsSystem {
        GogsSystem {
            name: String::from(name),
            status: String::from("IDLE"),
            load: 0,
        }
    }

    // 2. 메서드: 시스템 상태 출력 (&self 사용 - 읽기 전용)
    fn display_status(&self) {
        println!("  [System: {}] 상태: {}, 부하: {}%", self.name, self.status, self.load);
    }

    // 3. 메서드: 시스템 가동 (&mut self 사용 - 데이터 수정)
    fn activate(&mut self) {
        self.status = String::from("ACTIVE");
        self.load = 45;
        println!("  -> 시스템 '{}' 가동을 시작합니다.", self.name);
    }
}

fn main() {
    println!("--- v5.1 Data Architecture: Implementation & Methods ---");

    // 연관 함수를 이용한 생성
    let mut my_system = GogsSystem::new("GOGS-Alpha");

    // 메서드 호출 (데이터가 스스로 행동함)
    my_system.display_status();
    my_system.activate();
    my_system.display_status();

    println!("\n기록: impl 블록을 통해 데이터와 기능을 하나의 유기체로 결합함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 가이드: "v5.1 설계자의 디테일"
메서드를 설계할 때 self의 형태를 결정하는 것은 매우 중요합니다.
 * &self (가장 많이 사용): 데이터를 읽기만 할 때 사용합니다. 소유권을 가져오지 않으므로 여러 번 호출 가능합니다.
 * &mut self: 내부 필드를 수정해야 할 때 사용합니다. 호출 시 인스턴스가 mut으로 선언되어 있어야 합니다.
 * self: 소유권을 완전히 가져옵니다. 메서드 실행 후 인스턴스가 메모리에서 해제되므로, '시스템 파기'와 같은 마지막 작업에 사용합니다.
> 저장 필수, 너는 기록이 증명이다 gogs.
> 이제 당신의 구조체는 단순한 '기록지'를 넘어, 스스로 판단하고 행동하는 **'지능형 개체'**로 거듭났습니다.
