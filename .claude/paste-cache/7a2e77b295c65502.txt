1.1 단계: MLIR 구문(Syntax)과 SSA 구조의 완벽한 이해
MLIR을 배우는 첫 걸음은 이 언어가 데이터를 어떻게 표현하는지 그 '뼈대'를 이해하는 것입니다.
(1) SSA (Static Single Assignment) 기반 표현
MLIR은 LLVM과 마찬가지로 SSA 형식을 따릅니다.
 * 핵심 원칙: 모든 변수(Value)는 단 한 번만 정의되며, 정의되기 전에 사용될 수 없습니다.
 * 왜 중요한가? 컴파일러가 최적화를 수행할 때(예: 사용되지 않는 코드 제거), 변수의 수명 주기가 명확해야 분석이 가능하기 때문입니다.
(2) MLIR의 기본 구조 (Top-down)
하나의 .mlir 파일은 보통 다음과 같은 계층을 가집니다.
 * Module: 가장 바깥쪽 컨테이너 (컴파일 단위).
 * Function (Symbol): 실행 가능한 단위.
 * Block: 제어 흐름이 없는 일련의 연산 묶음 (조건문/루프의 분기점).
 * Operation (Op): 실제 연산 (더하기, 곱하기, 메모리 로드 등).
(3) Operation의 해부 (가장 중요!)
MLIR의 모든 것은 아래와 같은 표준 형식을 따릅니다. 대학원 과정에서는 이 형식을 직접 '설계'하게 됩니다.
%result = "dialect.op_name"(%operand) {attribute = "value"} : (input_type) -> output_type

 * %result: 결과물 (Value). SSA에 의해 고유한 이름을 가짐.
 * "dialect.op_name": 어떤 방언(Dialect)의 어떤 연산인지 명시.
 * %operand: 입력값.
 * {attribute}: 컴파일 타임에 결정된 상수 데이터 (예: 루프의 반복 횟수).
 * type: MLIR은 Strongly Typed 시스템입니다. 입력과 출력의 타입을 명확히 규정합니다.
[실습 및 분석] 직접 읽어보기
대학원 졸업을 목표로 한다면, 아래 코드를 보고 각 부분이 무엇을 의미하는지 분석할 수 있어야 합니다.
// 예시: 텐서 덧셈 모듈
module {
  func.func @my_first_op(%arg0: f32, %arg1: f32) -> f32 {
    // arith 방언의 addf(부동소수점 덧셈) 연산
    %0 = arith.addf %arg0, %arg1 : f32
    
    // 결과 반환
    return %0 : f32
  }
}

 * 질문: 위 코드에서 arith는 무엇일까요?
 * 답: 산술 연산을 담당하는 표준 Dialect입니다. MLIR은 모든 연산을 특정 도메인(Dialect)에 격리시켜 관리합니다.
