🏛️ 파이썬 대학교 3학년 (v5.2)
주제: "기억의 구조화와 관계형 데이터 모델링"
단순히 텍스트 파일(.txt)에 저장하는 것과 데이터베이스(DB)를 쓰는 것은 차원이 다릅니다. DB는 데이터의 **무결성(Integrity)**을 보장하며, 복잡한 조건으로 데이터를 추출하는 데 최적화되어 있습니다.
1. 핵심 개념: SQL (Structured Query Language)
 * CRUD: Create(생성), Read(읽기), Update(수정), Delete(삭제)의 4대 기본 작업.
 * Schema(스키마): 데이터가 들어갈 방의 규격(표)을 정의하는 설계도.
 * Primary Key(PK): 각 데이터를 유일하게 식별하는 '기록의 증명' 번호.
2. SQLite: 파이썬의 기본 내장 연구소
 * 파이썬은 별도의 설치 없이도 가벼운 관계형 DB인 SQLite를 기본으로 제공합니다. 파일 하나가 곧 데이터베이스가 됩니다.
🛠️ 전공 실무: [v5.2 Database Architecture]
실험 기록을 데이터베이스에 영구적으로 기록하고 조회하는 시스템을 설계해 보겠습니다.
# [Python University v5.2] SQLite3 기반 데이터베이스 관리

import sqlite3

# 1. 데이터베이스 연결 (파일이 없으면 생성됨)
conn = sqlite3.connect('gogs_research.db')
cursor = conn.cursor()

# 2. 테이블 설계 (스키마 생성)
cursor.execute('''
    CREATE TABLE IF NOT EXISTS experiments (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        subject TEXT NOT NULL,
        result REAL,
        timestamp DATETIME DEFAULT CURRENT_TIMESTAMP
    )
''')

# 3. 데이터 기록 (Create)
def save_experiment(subject, result):
    cursor.execute(
        "INSERT INTO experiments (subject, result) VALUES (?, ?)", 
        (subject, result)
    )
    conn.commit()
    print(f"💾 데이터 기록 완료: {subject}")

# 4. 데이터 조회 (Read)
def get_all_records():
    cursor.execute("SELECT * FROM experiments")
    print("\n--- [전체 실험 기록 보고서] ---")
    for row in cursor.fetchall():
        print(f"ID: {row[0]} | 주제: {row[1]} | 결과: {row[2]} | 시각: {row[3]}")

# --- 실행 ---
save_experiment("비동기 엔진 테스트", 98.5)
save_experiment("메타클래스 검증", 100.0)

get_all_records()

# 5. 자원 해제 (v3.3 컨텍스트 매니저를 쓰면 더 좋음)
conn.close()

print("\n저장 필수 너는 기록이 증명이다 gogs")

🎓 대학교 3학년 전공 필수 팁
 * SQL Injection 방어: 위 코드에서 ?를 사용한 이유는 보안 때문입니다. 사용자의 입력을 직접 SQL 문장에 더하는 행위는 데이터베이스 해킹의 빌미가 됩니다. 설계자는 항상 파라미터 바인딩을 사용해야 합니다.
 * ORM(Object-Relational Mapping): 대학 4학년 과정에서는 SQL을 직접 쓰지 않고 파이썬 클래스로 DB를 다루는 SQLAlchemy나 Django ORM 같은 고급 도구를 배울 것입니다. 하지만 기초(SQL)를 모르면 모래성을 쌓는 것과 같습니다.
