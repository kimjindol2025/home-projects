🎓 대학교 4학년 (v6.0): 졸업 작품 및 실전 아키텍처
주제: "통합과 확장, 그리고 신뢰할 수 있는 시스템"
4학년의 첫 번째 관문은 **[v6.1: 고가용성 분산 시스템 설계(High Availability & Scalability)]**입니다. 단순히 '작동하는' 프로그램을 넘어, 수만 명이 동시에 접속해도 죽지 않고 스스로 확장되는 시스템을 설계합니다.
1. 캡스톤 프로젝트의 핵심: 오케스트레이션(Orchestration)
여러 개의 마이크로서비스(v5.1에서 배운 내용)가 서로 유기적으로 움직이도록 지휘하는 법을 배웁니다.
 * 로드 밸런싱(Load Balancing): 밀려드는 트래픽을 여러 서버로 공평하게 분산합니다.
 * 오토 스케일링(Auto Scaling): 접속자가 많아지면 서버가 자동으로 복제되고, 적어지면 스스로 줄어듭니다.
 * 장애 복구(Failover): 한 서버가 죽으면 즉시 다른 서버가 그 업무를 이어받아 시스템의 연속성을 **'증명'**합니다.
2. CI/CD (지속적 통합 및 배포)
설계자가 코드를 수정하면, 자동으로 테스트가 수행되고 실제 서비스에 반영되는 자동화 파이프라인입니다.
> "기록(Log)과 테스트(Test)가 증명되지 않은 코드는 배포될 수 없다."는 원칙을 실현합니다.
> 
🛠️ 졸업 작품 실무 설계: [v6.1 Distributed Task Engine]
수많은 연산 요청을 여러 노드(Node)에 분산시키고, 결과를 취합하는 거대 엔진의 프로토타입을 설계해 보겠습니다.
# [Python University v6.1] 분산 작업 처리 엔진 아키텍처

import uuid
import random

# 1. 작업 단위 (Task)
class Task:
    def __init__(self, data):
        self.id = uuid.uuid4()
        self.data = data
        self.status = "PENDING"

# 2. 작업 노드 (Worker Node) - v4.2 멀티프로세싱 지식 활용
class WorkerNode:
    def __init__(self, node_id):
        self.node_id = node_id
    
    def process(self, task):
        print(f"⚙️ [Node {self.node_id}] 작업 처리 중... (Task ID: {task.id})")
        # 실제로는 복잡한 연산이나 네트워크 통신이 발생함
        task.status = "COMPLETED"
        return task

# 3. 마스터 오케스트레이터 (Master Orchestrator)
class SystemMaster:
    def __init__(self):
        self.workers = [WorkerNode(i) for i in range(3)] # 3개의 가상 노드
        self.task_queue = []

    def receive_task(self, data):
        new_task = Task(data)
        self.task_queue.append(new_task)
        self._dispatch()

    def _dispatch(self):
        # 로드 밸런싱: 가용 가능한 노드에 무작위 혹은 순차 분산
        for task in self.task_queue:
            if task.status == "PENDING":
                worker = random.choice(self.workers)
                worker.process(task)

# 실행 및 시스템 증명
master = SystemMaster()
master.receive_task("Deep Learning Model Train")
master.receive_task("Big Data Analysis")
