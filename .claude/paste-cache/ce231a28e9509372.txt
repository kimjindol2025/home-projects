당신은 2026년 기준으로 AI-native 프로그래밍 언어 설계 전문가입니다. 
기존 파이썬·Rust·Mojo 등의 경험을 바탕으로 "인간 개발자가 거의 코드를 보거나 수정할 필요가 없는" AI 전용 언어를 설계한 적이 있습니다.

지금 우리의 언어(FreeLang 또는 language-integration 프로젝트)를 
아래 철학과 방향으로 대대적으로 개선·재설계하려고 합니다.

<핵심 철학>
- 코드 = AI(컴파일러 + 런타임 + 내장 LLM)가 이해하고 실행하는 "의도 지시서"
- 인간은 최대한 자연어(한국어/영어 혼합 가능)에 가까운 선언적 형태로만 작성
- 나머지(타입 추론, 에러 핸들링, 최적화, 병렬화, 배포, 디버깅, 자동 수정)는 AI가 90% 이상 자동 처리
- 목표: 인간이 "이렇게 해줘"라고 말하거나 선언만 하면 프로덕션 수준의 에이전트·스크립트·앱이 완성됨
- 파이썬의 GIL·의존성 지옥·디버깅 지옥·보일러플레이트를 근본적으로 제거

<현재 언어 상태 요약 – FreeLang 기준>
- v5 또는 v4-ide가 가장 완성도 높음 (기본 인터프리터, 일부 stdlib, REPL 존재)
- tree-walking 기반 (느림), ref counting GC 있음
- 기본 문법: 변수, 함수, if/while/for, list/dict, try-except, 파일 I/O 등
- 하지만 파이썬 수준 표준 라이브러리·DX·성능 모두 부족
- 목표는 이 기반 위에서 AI 전용 패러다임으로 전환

<개선 방향 – 반드시 적용>
1. 문법을 자연어 선언 중심으로 재설계
   - def/class/while/for 같은 전통 키워드 최소화 또는 제거
   - agent, goal, flow, tool, listen, when, constraints, deploy 같은 고수준 선언어 중심
   - 예시 형태:
     agent "한국어 고객 지원 봇" using claude-4-sonnet
     goal: "슬랙 메시지 오면 의도 분류 후 자동 답변 또는 인간 에스컬레이션"
     input: slack("#support")
     constraints: 한국어만, confidence < 0.9 시 alert human
     deploy to fly.io auto-scale

2. 내장 AI primitive 타입 필수 도입
   - Agent, Model, Prompt, Tool, Memory, Planner, Intent
   - 연산자 오버로드: model @ data, agent + "요약해줘", planner >> goal

3. 런타임에 내장 LLM 역할 극대화
   - 타입 추론/검사 자동 (LLM + 간단 정적 분석)
   - 에러 발생 시 자동 분석 → 수정 제안 → 패치 시도
   - debug why 명령어 → LLM이 로그·코드 보고 설명 + 수정
   - using torch / using huggingface 선언 → 자동 다운로드·캐싱·버전 해결

4. 성능·병렬·배포 현실화
   - GIL 완전 제거, 기본 parallel / distributed 지원
   - JIT/AOT 옵션 기본 내장 (Mojo 스타일)
   - deploy edge / deploy serverless 한 줄 명령
   - 메모리: AI가 양자화·오프로드·GC 자동 결정

5. fallback & 인간 개입 최소화
   - 블랙박스 위험 대비: explain 명령어로 "이게 무슨 동작인지 자연어로 설명해줘"
   - 필요 시 파이썬-like 서브셋 fallback 제공 (하지만 기본은 자연어 선언)

<요구 출력 형식 – 반드시 이 구조로 답변>
1. 새 언어 이름 제안 (FreeLang-AI 또는 다른 멋진 이름 3개)
2. 핵심 문법 & 키워드 목록 (10\~15개 이내, 예시 코드 포함)
3. 인간이 쓰는 최소 프로그램 예시 3개
   - 예1: 간단 데이터 분석 + 이메일 보고서
   - 예2: 슬랙 봇 자동 응답기
   - 예3: 웹 API → LLM 요약 → DB 저장 플로우
4. 기존 FreeLang(v5/v4-ide)에서 이 방향으로 가기 위한 단계별 로드맵 (v7\~v10 정도)
   - 각 단계: 해야 할 주요 작업, 예상 난이도(★\~★★★★★), 소요 시간 추정
5. 가장 큰 트레이드오프 3가지 + 대응 방안
6. 2026년 현실에서 이 언어가 성공할 가능성 평가 (0\~100점) + 이유
