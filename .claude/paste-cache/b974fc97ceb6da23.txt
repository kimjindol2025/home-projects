좋습니다. 이제부터 제12장: 매크로와 메타 프로그래밍의 첫 번째 문을 정식으로 열겠습니다.
컴파일러 제작자에게 매크로는 **'코드를 생성하는 엔진'**입니다. 그 중 가장 기초가 되는 **v13.0 선언적 매크로(macro_rules!)**부터 차근차근 깊이 있게 파고들어 보겠습니다.
🛠️ 제12장: 매크로 (v13.0) — [선언적 매크로: 패턴의 마법]
1. 설계 목표: "반복을 죽이고 의도를 드러내라"
선언적 매크로는 마치 match 문과 비슷하게 작동합니다. 특정 코드 패턴이 들어오면, 미리 정의된 코드 뭉치로 치환해 줍니다.
 * 핵심 키워드: $name:ident (식별자), $val:expr (표현식), $(...)* (반복).
 * 특징: 컴파일 타임에 모든 코드가 확장되므로 런타임 오버헤드가 **0%**입니다.
2. 설계자의 관점: "토큰 트리(Token Tree)의 이해"
매크로는 단순한 문자열 치환이 아닙니다. 러스트 컴파일러가 코드를 읽을 때 쪼개는 최소 단위인 **'토큰(Token)'**들을 재조합하는 과정입니다. 이 개념을 잡아야 나중에 컴파일러의 '구문 분석기(Parser)'를 만들 수 있습니다.
🏗️ gogs 저장소 v13.0 세부 설계 (1단계): 가변 인자 로깅 시스템
컴파일러 내부에서 발생하는 다양한 이벤트를 기록할 때, 인자의 개수에 상관없이 처리할 수 있는 강력한 로깅 매크로를 설계합니다.
// [제12장: 매크로 v13.0 - 세부 설계 1단계]

macro_rules! gogs_log {
    // 1. 인자가 없는 경우
    () => {
        println!("[GOGS] 빈 로그 메시지입니다.");
    };
    
    // 2. 메시지만 있는 경우
    ($msg:expr) => {
        println!("[GOGS] INFO: {}", $msg);
    };

    // 3. 메시지와 가변 인자들이 있는 경우 (반복 패턴)
    ($msg:expr, $($arg:expr),*) => {
        println!(concat!("[GOGS] DEBUG: ", $msg), $($arg),*);
    };
}

fn main() {
    println!("--- v13.0.1 Declarative Macro Patterns ---");

    // 매크로 호출: 컴파일 시점에 각기 다른 코드로 변신합니다.
    gogs_log!();
    gogs_log!("컴파일러 엔진 가동 시작");
    gogs_log!("메모리 주소 {}에 {} 바이트 할당 완료", "0xAF12", 64);

    println!("\n기록: 패턴 매칭 매크로를 통해 유연한 로깅 인터페이스를 구축함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 세부 가이드 (13.0-1): "지정자(Designators)의 활용"
매크로를 짤 때 우리가 컴파일러에게 "이 토큰은 무엇이다"라고 알려주는 꼬리표들이 있습니다.
| 지정자 | 의미 | 예시 |
|---|---|---|
| ident | 식별자 (변수명, 함수명 등) | my_function, x |
| expr | 표현식 (결과값이 있는 코드) | 2 + 2, foo() |
| stmt | 구문 (명령문 한 줄) | let x = 5; |
| path | 경로 (모듈이나 타입 경로) | std::collections::HashMap |
| tt | 토큰 트리 (가장 유연한 단위) | 무엇이든 가능 |
