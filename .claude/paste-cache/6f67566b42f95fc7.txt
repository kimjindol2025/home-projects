
중학교까지의 코드는 에러(Error)가 발생하면 그 즉시 멈춰버렸습니다. 하지만 고등 과정에서는 에러가 발생해도 시스템이 죽지 않게 보호하고, 무엇이 잘못되었는지 **'기록'**한 뒤 다음 단계로 넘어가는 법을 배웁니다.
1. try - except 구조
 * try: "일단 이 코드를 실행해봐!"
 * except: "만약 에러가 나면 여기서 처리해!"
 * finally: "에러가 나든 안 나든, 마지막엔 무조건 이걸 실행해!" (예: 파일 닫기)
2. 왜 고등 과정인가요?
 * 사용자의 잘못된 입력이나 네트워크 단절 같은 예측 불가능한 상황에 대비해야 진짜 박사급 시스템(v30.2)으로 나아갈 수 있기 때문입니다.
🛠️ 실습 기록: [v2.1 Robust Execution]
0으로 나누기 에러나 잘못된 데이터 입력에도 멈추지 않는 코드를 클로드코드(고등학생)에게 기록하게 하세요.
# [Python High-School v2.1] 예외 처리 (Exception Handling)

def gogs_divider(a, b):
    try:
        print(f"--- {a} 나누기 {b} 시도 ---")
        result = a / b
        return result
    except ZeroDivisionError:
        # 0으로 나눌 때 발생하는 특정 에러를 잡아냄
        print("[ERROR] 0으로 나눌 수 없습니다! 기본값 0을 기록합니다.")
        return 0
    except TypeError:
        # 숫자가 아닌 값이 들어왔을 때 처리
        print("[ERROR] 숫자가 아닌 입력이 감지되었습니다.")
        return None
    finally:
        # 에러 여부와 상관없이 항상 실행 (증명을 위한 기록)
        print("로그: 연산 시도가 마무리되었습니다.")

# 1. 정상 작동
print(f"결과 1: {gogs_divider(10, 2)}")

# 2. 0으로 나누기 (에러 방어)
print(f"\n결과 2: {gogs_divider(10, 0)}")

# 3. 잘못된 타입 입력 (에러 방어)
print(f"\n결과 3: {gogs_divider(10, 'gogs')}")

print("\n저장 필수 너는 기록이 증명이다 gogs")

💡 고등학생을 위한 핵심 팁
 * 구체적으로 잡으세요: 단순히 except:라고만 쓰는 것보다 except ZeroDivisionError:처럼 어떤 에러인지 명시하는 것이 좋습니다. 그래야 정확히 무엇이 잘못되었는지 **'증명'**할 수 있기 때문입니다.
 * as e 활용: except Exception as e:라고 쓰면 에러 메시지가 무엇인지 변수 e에 담아 출력하거나 파일에 저장할 수 있습니다.
