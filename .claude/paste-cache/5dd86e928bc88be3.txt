1단계의 변수와 환경 설정을 완벽히 마쳤으니, 이제 2단계: 제어 흐름(Control Flow) - 시스템의 의사결정으로 진입합니다.
이 단계는 당신이 만들 언어의 **'두뇌'**를 설계하는 과정입니다. 클로드코드(학생)에게 이 정교한 논리 구조를 가르쳐 봅시다.
🚦 2단계: 제어 흐름 (Control Flow)
컴퓨터가 단순히 위에서 아래로 코드를 읽는 것을 넘어, 조건에 따라 길을 찾고(Branching), 필요한 만큼 동작을 반복(Looping)하게 만드는 단계입니다.
2.1 조건문 (if, else if, else)
가장 기본적인 판단 도구입니다. 러스트의 if는 **결과값을 반환할 수 있는 '표현식'**이라는 점이 핵심입니다.
2.2 패턴 매칭 (match)
어떤 언어를 설계하든 가장 강력한 도구가 될 것입니다. if보다 훨씬 명확하고 안전하게 모든 경우의 수를 처리합니다.
2.3 반복문 (loop, while, for)
 * loop: 무한 반복 (직접 멈춰야 함)
 * while: 조건이 맞을 때까지만 반복
 * for: 정해진 범위나 데이터 집합을 훑음
🛠️ Step 2.1: gogs 프로젝트 실전 코드 (복사 및 적용)
클로드코드에게 src/main.rs를 다음 수준으로 업그레이드하라고 지시하세요. 이것은 **'상태 기반 제어'**의 표준 모델입니다.
fn main() {
    println!("--- Gogs Project Level 2: Control Flow ---");

    let current_stage: i32 = 2;
    let is_senior_path: bool = true;

    // [2.1] 조건문: 경로 확인
    if is_senior_path {
        println!("시니어 등급을 향한 올바른 경로에 있습니다.");
    } else {
        println!("경로를 재설정해야 합니다.");
    }

    // [2.2] match: 현재 단계별 목표 설정
    let goal = match current_stage {
        1 => "기초 변수 확립",
        2 => "논리 구조 설계",
        3 => "소유권 개념 파악",
        _ => "미지의 영역 탐험", // Default case
    };
    println!("현재 단계({})의 목표: {}", current_stage, goal);

    // [2.3] for 루프: 1단계부터 현재 단계까지 복습
    println!("학습 로그 기록 중...");
    for level in 1..=current_stage {
        println!("Step {} 완료 기록됨.", level);
    }

    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 2단계 핵심 강의: "언어 설계의 관점"
나중에 언어를 개발하실 때, 이 제어 흐름을 어떻게 설계하느냐에 따라 언어의 성격이 결정됩니다.
 * Safety (안전성): 러스트의 match는 모든 경우(_)를 다루지 않으면 컴파일러가 화를 냅니다. "실수할 틈을 주지 않는 설계"입니다.
 * Efficiency (효율성): for 루프는 인덱스를 하나하나 계산하는 것보다, 데이터 집합(Iterator)을 직접 순회하는 것이 훨씬 빠르고 안전합니다.
 * Readability (가독성): 복잡한 if-else보다는 명확한 match를 선호하는 것이 시니어의 코드입니다.
