1️⃣ 독립성 수치 재검증 (최우선)

Step 1.1 – 실제 코드 비율 계산

측정 항목:

FreeLang 코드 라인 수

Host 언어 코드 라인 수

빌드 체인 의존성

런타임 의존성

FFI 호출 수


공식:

Independence Score =
  FreeLang LOC /
  (FreeLang LOC + Host Runtime LOC + Required Toolchain LOC)

선언 99.9%가 나오려면:

Host logic ≈ 0

Toolchain dependency ≈ 0


하나라도 남아있으면 99.9% 불가.


---

Step 1.2 – Bootstrap Cycle 테스트

검증 질문:

1. FreeLang으로 FreeLang 컴파일 가능한가?


2. Host 제거 후 빌드 가능한가?


3. 새로운 환경에서 재생성 가능한가?


4. 표준 라이브러리도 FreeLang으로만 구성되어 있는가?



이 중 하나라도 NO면

→ self-hosting 미완성

2️⃣ 엔진 깊이 검증 (Integrity / Memory / JIT)

대상:

kim/freelang-final


---

🔍 Integrity Engine

검증 항목:

수학적 불변식 실제로 증명 가능한가?

증명 실패 시 트랜잭션 롤백 작동하는가?

형식 검증인가, 단순 조건 검사인가?


테스트:

고의적 불변식 위반 주입

랜덤 AST 변조

동시성 조건 위반


실패율 > 0 → 엔진 깊이 부족


---

🔍 Memory Engine

검증 항목:

실제 소유권 모델 존재?

Lifetime 추론 그래프 있는가?

GC vs Static 분석 구분?


테스트:

10,000회 동적 할당/해제 스트레스

강제 Use-After-Free 주입

메모리 파편화 시뮬레이션


Leak 1건이라도 발생하면 “0 선언” 무효.


---

🔍 JIT Engine

검증 항목:

실제 IR 변환 존재?

바이트코드 생성?

런타임 재컴파일 되는가?


“95% 개선” 주장 검증:

동일 workload 100회 평균

Warmup 제거

분산/표준편차 계산


통계 유의성 없으면 과장 판정.

재현성 검증 필수.
