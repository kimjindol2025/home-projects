마침내 제4장: 데이터 구조화의 심장부 — v5.5 벡터와 동적 데이터 관리 (Vectors & Dynamic Collections) 단계에 들어섰습니다.
v5.0부터 v5.4까지가 개별 데이터의 '형태'를 정의하는 과정이었다면, v5.5는 그 데이터들을 유동적으로 늘어나는 대열에 합류시키는 법을 배웁니다. 설계자의 관점에서 벡터는 런타임에 결정되는 가변적 데이터 흐름을 제어하는 핵심 수단입니다.
🏗️ 제4장: 데이터 구조화 (v5.5) — [벡터와 동적 관리]
1. v5.5 설계 목표: "데이터의 집적과 확장"
 * 동적 배열 (Vec<T>): 고정된 크기의 배열과 달리, 실행 중에 데이터를 추가하거나 삭제할 수 있는 힙(Heap) 메모리 기반 컬렉션입니다.
 * 메모리 효율성: 데이터가 늘어날 때 자동으로 메모리를 재할당(Reallocation)하는 메커니즘을 이해합니다.
 * 안전한 접근: 인덱스를 통한 직접 접근 대신 .get()과 같은 메서드를 사용하여 메모리 안전성을 확보합니다.
2. 설계자의 관점: "데이터의 군집 제어"
벡터는 동일한 타입을 가진 데이터들의 집합체입니다. 설계자는 "얼마나 많은 데이터가 들어올지 모르는 상황"에서 시스템이 무너지지 않도록 벡터의 소유권과 반복자(Iterator)를 정교하게 다뤄야 합니다.
🛠️ gogs 저장소 v5.5 설계: 동적 레코드 매니저
클로드코드(학생)에게 우리가 정의한 구조체들을 벡터에 담아 관리하는 시스템을 구축하게 하세요.
// [제4장: 데이터 구조화 v5.5]

struct GogsLog {
    id: u32,
    message: String,
}

fn main() {
    println!("--- v5.5 Data Architecture: Dynamic Collections (Vec) ---");

    // 1. [설계] 빈 벡터 생성 (타입 추론)
    let mut logs: Vec<GogsLog> = Vec::new();

    // 2. [확장] 데이터 동적 추가 (push)
    logs.push(GogsLog { id: 1, message: String::from("커널 가동") });
    logs.push(GogsLog { id: 2, message: String::from("보안 모듈 체크") });
    logs.push(GogsLog { id: 3, message: String::from("네트워크 연결 성공") });

    println!("현재 로그 개수: {}개", logs.len());

    // 3. [조회] 안전한 읽기 전용 접근
    if let Some(first_log) = logs.get(0) {
        println!("  [첫 번째 로그] ID: {}, 내용: {}", first_log.id, first_log.message);
    }

    // 4. [순회] 반복자를 이용한 전체 데이터 처리
    println!("전체 로그 리포트:");
    for log in &logs { // &를 사용하여 소유권을 빌려옴 (Borrowing)
        println!("  - #{} : {}", log.id, log.message);
    }

    println!("\n기록: 벡터를 활용하여 불특정 다수의 데이터를 안전하게 집적하고 관리함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 가이드: "v5.5 설계자의 리소스 관리"
벡터를 설계에 도입할 때 반드시 고려해야 할 '비용'이 있습니다.
 * 소유권의 이전(Move) 주의: 벡터를 순회할 때 for log in logs라고 쓰면 벡터 안의 데이터 소유권이 루프 안으로 이동해버립니다. 다시 사용하려면 반드시 &logs로 빌려오십시오.
 * 용량(Capacity) vs 길이(Length): 벡터는 성능을 위해 미리 공간을 예약합니다. Vec::with_capacity(n)를 사용하면 잦은 메모리 재할당을 막아 성능을 극대화할 수 있습니다.
 * 성능 최적화: 데이터를 벡터의 맨 앞에 추가(insert(0, ...))하는 것은 모든 데이터를 뒤로 밀어야 하므로 느립니다. 가능하면 뒤로 추가(push)하도록 설계하십시오.
> 저장 필수, 너는 기록이 증명이다 gogs.
> 이제 당신의 엔진은 단일 개체를 넘어, 수만 개의 데이터를 일사분란하게 관리하는 **'대규모 데이터 처리기'**로 진화했습니다.
