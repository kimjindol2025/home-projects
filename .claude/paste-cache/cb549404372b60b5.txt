### 1.1 단계: AIOps 도메인 특화 Byzantine 공격 시나리오 재정의
(목표: 일반 FL Byzantine 공격이 아니라, **AIOps(운영 지능화) 실무에서 실제로 치명적인 영향**을 주는 구체적 시나리오를 명확히 정의 → 논문의 motivation과 novelty를 강하게 뒷받침)

#### AIOps 환경의 핵심 특징 (우리 프로젝트 기준 재확인)
- 분산된 edge/server 노드들 (로그, 메트릭, 트레이스 등)에서 **실시간 anomaly detection + root cause analysis (RCA) + forecasting** 같은 multi-task 학습
- 데이터는 **시간 시계열 + multi-modal** (로그 텍스트 + 숫자 메트릭 + 네트워크 그래프)
- 모델 poisoning이 발생하면 → **false positive 폭증 → alert fatigue**, **RCA 실패 → 장애 복구 지연**, **예측 오류 → proactive scaling 실패** 등 운영 비용 폭등 + 서비스 중단 위험

#### 2026년 기준 AIOps-특화 Byzantine 시나리오 후보 (top 4)
1. **Label/Target Flipping Attack on Anomaly Detection Task**  
   - 악의적 edge 노드(예: compromised IoT device or insider)가 정상 로그를 anomaly로, anomaly를 정상으로 라벨 뒤집기  
   → 글로벌 모델이 과도한 false alarm 생성 → 운영팀 피로도 ↑, 중요한 장애 miss 가능성 ↑  
   → AIOps에서 가장 치명적: "너무 많은 알람"은 실제 알람을 무시하게 만듦

2. **Gradient Scaling / Sign Flip on Forecasting Task**  
   - 미래 메트릭 예측(CPU/load 예측 등)에서 gradient 방향 반전 or magnitude 과도 증폭  
   → 예측값이 극단적으로 왜곡 → autoscaling 오작동 (over-provisioning → 비용 폭증 or under-provisioning → downtime)

3. **Targeted Poisoning on RCA (Root Cause Analysis) Sub-task**  
   - 특정 서비스/컴포넌트(예: DB 노드)에 대한 causal graph 왜곡  
   → RCA가 잘못된 root cause 지목 → 잘못된 remediation 실행 → 장애 확산  
   → multi-task FL에서 한 task poisoning이 다른 task(예: anomaly det)에도 전파되는 cross-task contamination

4. **Adaptive / Delayed Attack (최근 hot gap)**  
   - 초반엔 정상처럼 행동하다가 특정 라운드(예: peak time)에서만 poisoning  
   - 또는 quarantine 후 재접속해서 재공격 (reconnecting malicious clients)  
   → 기존 static aggregation (Krum, Median 등)은 이런 dynamic/adaptive에 취약  
   → 2025~2026 논문들 (e.g. adaptive adversaries survey)에서도 이게 주요 open problem으로 꼽힘

#### 추천: 우리 프로젝트에서 가장 강력한 motivation으로 쓸 시나리오
**주 시나리오** → 1번 + 3번 조합 (Anomaly + RCA multi-task poisoning)  
**부 시나리오** → 4번 adaptive/reconnecting attack (우리 DPoS quarantine이 직접 대응 가능하니 novelty ↑)

이 시나리오들로 motivation 쓰면:
- "일반 FL과 달리 AIOps에서는 Byzantine attack이 단순 accuracy drop이 아니라 **운영 중단 + 비용 폭증**으로 직결된다"
- "기존 robust aggregation은 propagation을 막지 못하고, multi-task 간 영향 무시 → AIOps에 부적합"
