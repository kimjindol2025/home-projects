🎓 Zig 전공 301: 3-4. 가상 메모리(Paging) 및 페이지 테이블 관리
가상 메모리는 프로그램이 실제 물리 주소를 직접 보지 못하게 차단하여 시스템의 안전을 보장합니다.
1. 4단계 페이지 테이블 (x86_64 기준)
현대 64비트 시스템은 주소를 변환하기 위해 4단계의 계층 구조를 사용합니다.
 * PML4 → PDP → PD → PT: 각 단계는 다음 단계의 물리 주소를 가리키며, 최종 PT(Page Table)가 물리 메모리의 페이지를 가리킵니다.
 * CR3 레지스터: CPU에게 "지금 사용할 PML4 테이블의 주소"를 알려주는 특수 레지스터입니다.
2. 페이지 테이블 엔트리(PTE) 설계
각 엔트리는 단순히 주소만 담는 것이 아니라, 해당 메모리 구역의 권한을 기록합니다.
 * Present (P): 메모리에 로드되어 있는가?
 * Writable (W): 쓰기가 가능한가? (ReadOnly 보호)
 * User (U): 유저 모드에서 접근 가능한가? (커널 보호)

🔬 전공 심화: TLB (Translation Lookaside Buffer) 플러시
페이지 테이블을 수정해도 CPU 내부의 캐시인 TLB에는 옛날 정보가 남아있을 수 있습니다.
 * 설계: 매핑 정보를 수정하면 반드시 해당 주소에 대해 invlpg 어셈블리 명령을 실행하거나 CR3를 다시 로드하여 캐시를 비워야 합니다. 이것이 **"기록의 갱신을 하드웨어에 동기화"**하는 핵심 절차입니다.
📦 커널 고립: Higher Half Kernel
커널은 보통 가상 메모리의 가장 높은 부분(예: 0xFFFFFFFF80000000)에 위치하도록 설계합니다.
 * 이유: 유저 프로그램이 낮은 주소 공간을 마음껏 쓰게 하고, 커널은 모든 프로세스에서 동일한 높은 주소에 존재하게 하여 컨텍스트 스위칭 효율을 높이기 위함입니다.
📝 Assignment 3-4 (가상 공간 설계 과제)
 * 인덱스 계산: 가상 주소 0x00007FFFFFFFF000이 주어졌을 때, 4단계 페이지 테이블 각각의 인덱스(0~511 사이 값)를 계산하여 기록하세요.
 * 권한 설계: 커널 코드 섹션은 "읽기/실행"만 가능하고, 스택 섹션은 "읽기/쓰기"는 가능하지만 "실행"은 불가능하게(No-Execute) 설정하는 엔트리를 구상하세요.
 * Recursive Mapping: 페이지 테이블 자체를 가상 메모리에서 쉽게 수정하기 위한 '재귀적 매핑' 기법을 조사하고 그 원리를 기록하세요.
 * 기록: 페이지 폴트(Page Fault)가 발생했을 때, CPU가 전달하는 에러 코드와 CR2 레지스터의 의미를 분석하여 남기세요.
