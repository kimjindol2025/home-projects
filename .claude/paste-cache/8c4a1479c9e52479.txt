🎓 Go-언어 전공 101: 1-9. 에러 처리와 자원 관리
Go는 예외(Exception)를 '던지는' 언어가 아니라, 에러를 '값'으로 반환하여 직접 대면하게 하는 언어입니다.
1. 에러 핸들링 (Error Handling)
Go에는 try-catch가 없습니다. 대신 함수가 마지막 리턴값으로 error 타입을 반환하는 것이 관례입니다.
func findUser(id int) (User, error) {
    if id <= 0 {
        // 새로운 에러 생성 및 반환
        return User{}, errors.New("유효하지 않은 ID입니다")
    }
    // 성공 시 error는 nil을 반환
    return User{ID: id}, nil
}

// 실무에서 가장 많이 보는 패턴
user, err := findUser(-1)
if err != nil {
    fmt.Println("에러 발생:", err)
    return // 에러가 나면 여기서 중단하거나 처리
}

2. 지연 실행 (defer): 자원 해제의 마법
파일을 열거나 네트워크 연결을 맺었을 때, 함수가 끝나기 직전에 반드시 실행되도록 예약하는 키워드입니다.
func readFile(filename string) {
    f, err := os.Open(filename)
    if err != nil {
        return
    }
    // 함수가 종료되기 직전(return 직전)에 반드시 실행됨
    defer f.Close() 

    // 파일 읽기 로직...
}

 * 대학원 수준의 팁: defer는 스택(LIFO) 구조로 동작합니다. 여러 개를 선언하면 가장 나중에 선언된 것이 가장 먼저 실행됩니다.
3. Panic과 Recover
정말 치명적인 상황(예: 배열 인덱스 초과, 0으로 나누기)에서 프로그램이 강제 종료되는 것을 panic이라고 합니다. 이를 recover로 붙잡을 수 있지만, 가급적 사용을 지양하고 error 반환을 권장합니다.
🎓 Go-언어 전공 101: 1-10. 패키지와 가시성 (Visibility)
프로그램이 커지면 코드를 여러 파일과 폴더로 나누어야 합니다. 이때 중요한 것이 접근 제어입니다.
1. 대문자와 소문자의 차이
Go에는 public, private 키워드가 없습니다. 이름의 첫 글자로 결정됩니다.
 * 대문자로 시작 (Exported): 외부 패키지에서 접근 가능 (예: fmt.Println)
 * 소문자로 시작 (Unexported): 해당 패키지 내부에서만 접근 가능
2. 패키지 초기화 함수 init()
main 함수보다 더 먼저 실행되는 특수 함수입니다. 패키지가 로드될 때 설정을 초기화하는 용도로 사용됩니다.
🔬 전공 심화: 왜 Go는 Try-Catch를 거부했는가?
대학 과정에서 비판적 사고는 필수입니다. Go 설계자들이 예외 처리를 도입하지 않은 이유는 '흐름의 명확성' 때문입니다.
 * try-catch는 로직의 흐름을 갑자기 끊고 어디로 튈지 모르게 만듭니다.
 * if err != nil은 번거로워 보이지만, 코드의 흐름을 위에서 아래로 순차적으로 읽을 수 있게 보장하여 디버깅을 비약적으로 쉽게 만듭니다.
