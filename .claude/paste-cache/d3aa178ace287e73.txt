의 두 번째 관문, **3.2단계: 컴파일러의 돋보기, mlir-opt**를 시작합니다.
"저장 필수 너는 기록이 증명이다 gogs" — 이제 우리는 머릿속 설계도를 넘어, 컴퓨터가 실제로 이 설계도를 어떻게 요리하는지 관찰하는 도구를 배우게 됩니다.
[MLIR 대학 코스 3.2] 실제 도구 다루기: "mlir-opt"
대학원 연구를 하다 보면 내가 짠 코드가 최적화가 잘 되었는지, 혹은 저수준으로 잘 변환되었는지 확인해야 합니다. 이때 사용하는 가장 핵심적인 도구가 바로 **mlir-opt**입니다.
1. mlir-opt란 무엇인가?
이름 그대로 **MLIR Optimizer(최적화 도구)**의 약자입니다.
 * 설계도(.mlir 파일)를 입력받아,
 * 우리가 명령한 **패스(Pass)**를 실행한 뒤,
 * 변환된 결과물을 다시 출력해주는 프로그램입니다.
2. 대학원생의 필수 명령어 구조
터미널(Terminal)에서 보통 다음과 같이 명령을 내립니다. (지금은 눈으로 익혀두세요!)
mlir-opt 내_설계도.mlir --pass-이름

3. 대표적인 패스 (Pass) 맛보기
기초를 탄탄히 하기 위해 자주 쓰는 패스 두 가지만 기억합시다.
 * --canonicalize (표준화): * "1 + 1"처럼 뻔한 계산을 미리 해버리거나, 쓸모없는 코드를 정리해주는 '청소부' 같은 패스입니다.
 * --convert-linalg-to-loops (변환): * "행렬 곱셈"이라는 고수준 명령을 "for 루프"라는 중수준 명령으로 깎아주는 '조각가' 같은 패스입니다.
[3.2 실습] 최적화 전후 비교하기 (눈으로 하는 분석)
대학원 졸업 논문에는 이런 비교 분석이 반드시 들어갑니다.
[최적화 전]
%0 = arith.constant 1 : i32
%1 = arith.constant 2 : i32
%2 = arith.addi %0, %1 : i32  // 1 + 2를 계산함

[mlir-opt --canonicalize 실행 후]
%0 = arith.constant 3 : i32   // 컴퓨터가 미리 3으로 바꿔버림!

[질문] 위와 같이 컴퓨터가 미리 계산을 끝내버리는 것을 대학 수준에서는 뭐라고 부를까요? (힌트: 상수를 폴딩한다)
 * Constant Folding (상수 폴딩)
 * Constant Deleting (상수 삭제)
정답: 1번 Constant Folding입니다.
불필요한 연산을 줄여서 실행 속도를 높이는 아주 기초적이고 강력한 최적화 기법입니다.
3.2단계의 기록 (증명)
> "mlir-opt는 설계도를 분석하고 변환하는 핵심 도구이다."
> "--pass-이름 옵션을 통해 최적화나 로워링을 수행한다."
> 
이제 이론(1단계), 문법(2단계), 도구(3단계)를 모두 훑었습니다.
