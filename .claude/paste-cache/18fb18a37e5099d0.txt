+# P0-2: API 일관성 강화 계획
+
+**작성일:** 2026-02-27
+**상태:** 분석 완료, 실행 대기
+**우선순위:** P0 (필수)
+**예상 기간:** 1~2주
+
+---
+
+## 문제 정의
+
+FreeLang 라이브러리들 (Blockchain, SQLite, CSV 등)의 API 명명이 **혼재**되어 있어:
+- IDE 자동완성 어려움
+- 신규 개발자의 학습 곡선 높음
+- API 문서화 비일관적
+
+### 현재 API 패턴 분석
+
+#### freelang-v4-blockchain
+
+| 영역 | 현재 패턴 | 상태 | 권장 패턴 |
+|------|---------|------|---------|
+| Wallet 생성 | `Wallet.create()` | ✅ 일관됨 | `Wallet.create()` |
+| Wallet 복원 | `Wallet.fromPrivateKey()` | ✅ 일관됨 | `Wallet.fromPrivateKey()` |
+| Tx 생성 (coinbase) | `createCoinbaseTx()` **함수** | ❌ 혼재 | `Transaction.createCoinbase()` |
+| Tx 빌더 | `new TransactionBuilder()` | ✅ 일관됨 | `new TransactionBuilder()` |
+| Tx 검증 | `validateTx()` **함수** | ❌ 혼재 | `Transaction.validate()` |
+| Block 생성 (genesis) | `createGenesisBlock()` **함수** | ❌ 혼재 | `Block.genesis()` |
+| Block 마이닝 | `mineBlock()` **함수** | ❌ 혼재 | `Block.mine()` 또는 `BlockBuilder.mine()` |
+| Block 검증 | `validateBlock()` **함수** | ❌ 혼재 | `Block.validate()` |
+| Merkle Root | `calcMerkleRoot()` **함수** | ❌ 혼재 | `MerkleTree.calculate()` |
+| Block Hash | `calcBlockHash()` **함수** | ❌ 혼재 | `Block.calculateHash()` |
+
+#### freelang-v4-sqlite-integration
+
+| 영역 | 현재 패턴 | 상태 |
+|------|---------|------|
+| Repository 생성 | `createRepository()` **함수** | ❌ 혼재 |
+| 쿼리 캐싱 | `queryCache()` **함수** | ❌ 혼재 |
+| 캐시 관리 | `CacheManager` 클래스 | ✅ 일관됨 |
+
+---
+
+## 해결 방안
+
+### 규칙 1: 객체 생성은 클래스 메서드
+
+```typescript
+// ❌ Before
+createGenesisBlock(miner)
+createCoinbaseTx(address, amount)
+createRepository(db)
+
+// ✅ After
+Block.genesis(miner)
+Transaction.createCoinbase(address, amount)
+Repository.create(db)
+```
+
+### 규칙 2: 복잡한 생성은 빌더 패턴
+
+```typescript
+// ❌ Before
+mineBlock(previousHash, txList, difficulty)
+
+// ✅ After (방식 1: 빌더)
+new BlockBuilder()
+  .previousHash(genesis.hash)
+  .transactions([tx1, tx2])
+  .mine(2)
+  .build()
+
+// ✅ After (방식 2: 클래스 메서드 + async)
+const block = await Block.mine(previousHash, txList, 2);
+```
+
+### 규칙 3: 검증은 클래스 메서드
+
+```typescript
+// ❌ Before
+validateBlock(block, previousBlock)
+validateTx(tx, publicKey)
+
+// ✅ After
+Block.validate(block, previousBlock)
+Transaction.validate(tx, publicKey)
+
+// 또는
+const validator = new BlockValidator();
+const result = validator.validate(block, previousBlock);
+```
+
+### 규칙 4: 유틸은 정적 클래스 메서드
+
+```typescript
+// ❌ Before
+calcMerkleRoot(txList)
+calcBlockHash(header)
+
+// ✅ After
+MerkleTree.calculate(txList)
+Block.calculateHash(header)
+
+// 또는
+CryptoUtils.calculateMerkleRoot(txList)
+CryptoUtils.calculateBlockHash(header)
+```

+## 마이그레이션 체크리스트
+
+### Phase 1: freelang-v4-blockchain (1주)
+
+#### 필수 변경
+
+- [ ] `createGenesisBlock()` → `Block.genesis()`
+  - 영향 범위: block.ts 정의 + blockchain.ts 사용처 + 48개 테스트
+  - 마이그레이션 난이도: 낮음
+
+- [ ] `createCoinbaseTx()` → `Transaction.createCoinbase()`
+  - 영향 범위: transaction.ts 정의 + mempool.ts 사용처 + 테스트
+  - 마이그레이션 난이도: 낮음
+
+- [ ] `mineBlock()` → `Block.mine()` 또는 `BlockBuilder.mine()`
+  - 영향 범위: block.ts + blockchain.ts + 벤치마크
+  - 마이그레이션 난이도: 중간 (async/await 패턴)
+
+- [ ] `validateBlock()` → `Block.validate()`
+  - 영향 범위: block.ts + blockchain.ts + 테스트
+  - 마이그레이션 난이도: 낮음
+
+- [ ] `validateTx()` → `Transaction.validate()`
+  - 영향 범위: transaction.ts + blockchain.ts + 테스트
+  - 마이그레이션 난이도: 낮음
+
+#### 권장 변경
+
+- [ ] `calcMerkleRoot()` → `MerkleTree.calculate()`
+- [ ] `calcBlockHash()` → `Block.calculateHash()`
+- [ ] `serializeTx()` → `Transaction.serialize()`
+
+#### Deprecation 경고 추가
+
+```typescript
+/**
+ * @deprecated Use Block.genesis() instead
+ */
+export function createGenesisBlock(miner: string): Block {
+  return Block.genesis(miner);
+}
+```
+
+### Phase 2: freelang-v4-sqlite-integration (1주)
+
+- [ ] `createRepository()` → `Repository.create()`
+- [ ] 캐시 관련 함수들 → `CachedRepository` 메서드로
+- [ ] `queryCache()` → `CachedRepository.query()`
+
+### Phase 3: 다른 라이브러리 (1주)
+
+- [ ] freelang-csv
+- [ ] freelang-http
+- [ ] freelang-json-validator
+
+### Phase 4: 문서화 (1주)
+
+- [ ] TypeDoc 자동 생성 설정
+- [ ] API 레퍼런스 작성
+- [ ] 마이그레이션 가이드 작성
