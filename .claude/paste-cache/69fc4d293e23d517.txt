🎓 Zig 전공 101: 1-1. Hello, Zig! (설치와 첫 컴파일)
Zig의 철학은 **"숨겨진 제어 흐름이 없다"**는 것입니다. 모든 메모리 할당과 오류 처리는 명시적이어야 합니다. 이 철학이 프로젝트 구조에 어떻게 녹아드는지 확인해 봅시다.
1. 환경 구축 (Laboratory Setup)
먼저 도구가 준비되어야 연구를 시작할 수 있습니다.
 * 설치: ziglang.org에서 OS에 맞는 바이너리를 다운로드하거나, 패키지 매니저(brew install zig 등)를 사용합니다.
 * 버전 확인: 터미널에서 zig version을 입력하여 설치를 확인합니다. (0.11.0 이상 권장)
2. 프로젝트 초기화 (Project Initialization)
Zig는 자체적인 빌드 시스템을 언어 내부에 포함하고 있습니다.
mkdir zig-study && cd zig-study
zig init

이 명령어를 실행하면 다음과 같은 구조가 생성됩니다:
 * build.zig: Zig 전용 빌드 스크립트. (Makefile이나 CMake를 대체합니다.)
 * src/main.zig: 프로그램의 메인 진입점.
 * src/root.zig: 라이브러리 개발 시 사용되는 루트 파일.
3. Hello World 분석
src/main.zig의 표준적인 형태를 살펴봅시다.
const std = @import("std");

pub fn main() !void {
    // 표준 출력(stdout)을 가져옵니다.
    const stdout = std.io.getStdOut().writer();
    
    // "!"는 오류가 발생할 수 있음을 암시하며, try는 이를 처리합니다.
    try stdout.print("Hello, {s}!\n", .{"Zig Graduate School"});
}

🔬 전공 심화: 컴파일 타임의 마법 (comptime)
Zig의 가장 강력한 특징은 comptime 키워드입니다.
 * 개념: 실행 시점이 아닌 컴파일 시점에 코드를 실행하여 타입을 생성하거나 논리를 결정합니다.
 * 장점: 제네릭(Generics)을 구현할 때 별도의 복잡한 문법 없이 일반 함수처럼 작성할 수 있습니다.
📦 실행 및 테스트 (Execution)
연구 결과물을 확인하는 방법은 간단합니다.
 * 빌드 및 실행: zig build run
   * build.zig에 정의된 대로 컴파일 후 즉시 실행합니다.
 * 테스트 실행: zig build test
   * 코드 내에 작성된 test "설명" { ... } 블록들을 찾아 검증합니다.
