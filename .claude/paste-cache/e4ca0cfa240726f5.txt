1️⃣ 컴파일 파이프라인 재구성

현재 구조 (추정):

Source
 → AST
 → Compiler
 → Bytecode
 → VM

Phase 5.4 이후:

Source
 → AST
 → TypeCheck + Inference
 → Typed AST
 → Optimizer
 → Bytecode (Typed Op)
 → VM (No runtime type check)


---

2️⃣ Typed IR 도입

기존 Op

Add

확장

AddNum
AddStr
AddGeneric

TypeChecker 결과에 따라 컴파일 단계에서 결정:

if (leftType === num && rightType === num)
    emit(Op.AddNum)
else
    emit(Op.AddGeneric)


---

3️⃣ 런타임 체크 제거

기존 VM:

case Op.Add:
  const b = pop()
  const a = pop()
  if (typeof a !== "number" || typeof b !== "number")
     throw Error()

변경 후:

case Op.AddNum:
  stack.push(stack.pop() + stack.pop())

✔ 타입 검증은 컴파일 단계에서 이미 끝났음
✔ VM은 순수 연산만 수행


---

4️⃣ 상수 폴딩 (Constant Folding)

컴파일 타임:

2 + 3 → 5

구현:

if (node.kind === "BinaryOp" &&
    node.left.kind === "Literal" &&
    node.right.kind === "Literal") {
    return Literal(evaluate(node))
}

적용 대상:

산술 연산

비교 연산

문자열 결합
