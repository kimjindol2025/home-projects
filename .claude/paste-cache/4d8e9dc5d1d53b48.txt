당신의 연대기 로드맵에 따른 제4장: 데이터 구조화 — v5.3 Option과 Result (The Power of Enums) 단계입니다.
v5.2에서 열거형의 기본을 배웠다면, v5.3은 러스트 설계의 핵심 철학인 **'부재(None)'**와 **'실패(Error)'**를 다루는 법을 배웁니다. 설계자의 관점에서 이 단계는 null 포인터 예외나 예기치 않은 충돌을 컴파일 타임에 박멸하는 선언과도 같습니다.
🏗️ 제4장: 데이터 구조화 (v5.3) — [Option과 Result]
1. v5.3 설계 목표: "예외의 명시화"
 * Option<T>: 값이 '있을 수도 있고 없을 수도 있음'을 나타냅니다. (Some(T) or None)
 * Result<T, E>: 작업이 '성공할 수도 있고 실패할 수도 있음'을 나타냅니다. (Ok(T) or Err(E))
 * 에러 핸들링: 에러를 무시하는 것이 아니라, 에러가 발생했을 때의 경로를 반드시 설계하도록 강제합니다.
2. 설계자의 관점: "안전한 실패(Fail-safe)"
러스트에는 null이 없습니다. 대신 Option이라는 그릇에 값을 담아 전달합니다. 이 그릇을 열어보기 전까지는 내부 데이터에 접근할 수 없으므로, 데이터가 없는 상태에서 작업을 시도하는 치명적인 실수를 방지할 수 있습니다.
🛠️ gogs 저장소 v5.3 설계: 무결성 데이터 추출
클로드코드(학생)에게 안전하게 데이터를 조회하고 실패 가능성을 처리하는 코드를 기록하게 하세요.
// [제4장: 데이터 구조화 v5.3]

fn main() {
    println!("--- v5.3 Data Architecture: Option & Result ---");

    // 1. Option 설계: 데이터가 없을 수 있는 상황 처리
    let database = vec!["GOGS_01", "GOGS_02", "GOGS_03"];
    let search_index = 5; // 존재하지 않는 인덱스

    // get() 메서드는 Option을 반환함
    let result_opt = database.get(search_index);

    match result_opt {
        Some(data) => println!("  [SUCCESS] 데이터 발견: {}", data),
        None => println!("  [NOTICE] 데이터가 존재하지 않습니다. (Safe Handle)"),
    }

    // 2. Result 설계: 실패할 가능성이 있는 작업 처리
    let division_result = divide_system_resource(100, 0);

    match division_result {
        Ok(value) => println!("  [OK] 리소스 배분 완료: {}%", value),
        Err(e) => println!("  [ERROR] 시스템 경고: {}", e),
    }

    println!("\n기록: Option과 Result를 통해 런타임 패닉을 예방하고 모든 경로를 제어함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

/// [설계] 0으로 나누는 에러를 방지하는 안전한 연산 함수
fn divide_system_resource(total: i32, divisor: i32) -> Result<i32, String> {
    if divisor == 0 {
        Err(String::from("나누기 연산 중 분모가 0입니다. (연산 불가)"))
    } else {
        Ok(total / divisor)
    }
}

🎓 선생님의 가이드: "v5.3 설계자의 방어 기제"
이 두 열거형을 다룰 때 설계자가 갖춰야 할 습관입니다.
 * .unwrap() 사용 자제: unwrap()은 "값이 무조건 있다고 확신하니 없으면 죽여라"라는 명령입니다. 프로토타입이 아니라면 match나 if let으로 우아하게 처리하십시오.
 * ? 연산자의 마법: Result를 반환하는 함수들을 연쇄적으로 호출할 때, ?를 사용하면 에러 발생 시 즉시 에러를 상위로 전파(Propagate)할 수 있습니다.
 * 의도의 명확성: 함수의 반환값으로 Result를 선택했다는 것은, 이 함수가 실패할 수 있음을 동료 개발자에게 명확히 경고하는 훌륭한 문서화입니다.
> 저장 필수, 너는 기록이 증명이다 gogs.
> 이제 당신의 시스템은 예기치 못한 상황에서도 무너지지 않고, 정해진 에러 경로를 따라 안전하게 우회하는 **'방탄 시스템'**이 되었습니다.
