제9장(10단계): 두려움 없는 동시성의 진정한 마침표이자, 실전 아키텍처에서 가장 치명적인 위험을 다루는 [v10.4 동시성의 함정: 데드락(Deadlock) 방어 설계] 단계입니다.
러스트가 데이터 경합(Data Race)은 컴파일 타임에 막아주지만, 두 스레드가 서로의 자원을 기다리며 영원히 멈춰버리는 **교착 상태(Deadlock)**는 논리적인 문제이기에 설계자가 직접 해결해야 합니다. v10.4는 이 '논리적 멈춤'을 설계로 돌파하는 법을 기록합니다.
⚡ 제9장: 두려움 없는 동시성 (v10.4) — [데드락 예방과 자원 계층화]
1. v10.4 설계 목표: "순환 대기의 고리 끊기"
 * 순환 대기(Circular Wait) 방지: 스레드 간에 자원을 요청하는 순서를 명확히 정의하여 고리가 형성되지 않게 합니다.
 * 잠금 범위의 최소화: MutexGuard가 유지되는 시간을 극도로 줄여 다른 스레드의 대기 시간을 단축합니다.
 * 불변성 활용: 수정이 필요 없는 데이터는 Arc로만 공유하고, 가변성이 꼭 필요한 부분에만 최소한의 Mutex를 배치합니다.
2. 설계자의 관점: "잠금 순서의 명문화"
데드락은 보통 "A를 가진 놈이 B를 원하고, B를 가진 놈이 A를 원할 때" 발생합니다. 설계자는 모든 공유 자원에 **ID(우선순위)**를 부여하고, **"항상 ID가 낮은 자원부터 먼저 잠근다"**는 철칙을 세워야 합니다.
🛠️ gogs 저장소 v10.4 설계: 계층적 보안 자원 관리자
클로드코드(학생)에게 여러 개의 보안 키를 관리할 때 순환 대기를 방지하는 '안전 잠금 프로토콜'을 구현하게 하세요.
// [제9장: 동시성 v10.4 - 최종 방어 아키텍처]

use std::sync::{Arc, Mutex};
use std::thread;
use std::time::Duration;

#[derive(Debug)]
struct SecureResource {
    id: u32, // 자원 식별자 (잠금 순서 결정용)
    label: String,
    data: Mutex<String>,
}

fn main() {
    println!("--- v10.4 Defensive Design: Deadlock Prevention ---");

    // 자원 생성 (ID 부여를 통한 계층화)
    let res_a = Arc::new(SecureResource { id: 1, label: "데이터베이스".into(), data: Mutex::new("DB_DATA".into()) });
    let res_b = Arc::new(SecureResource { id: 2, label: "인증서버".into(), data: Mutex::new("AUTH_DATA".into()) });

    let r1_a = Arc::clone(&res_a);
    let r1_b = Arc::clone(&res_b);

    // 스레드 1: A를 먼저 얻고 B를 얻으려 함
    let h1 = thread::spawn(move || {
        let _lock_a = r1_a.data.lock().unwrap();
        println!("  [Thread 1] {} (ID:{}) 잠금 획득", r1_a.label, r1_a.id);
        
        thread::sleep(Duration::from_millis(10)); // 문맥 교환 발생 가정

        let _lock_b = r1_b.data.lock().unwrap();
        println!("  [Thread 1] {} (ID:{})까지 모두 확보 성공", r1_b.label, r1_b.id);
    });

    let r2_a = Arc::clone(&res_a);
    let r2_b = Arc::clone(&res_b);

    // 스레드 2: [설계 핵심] B가 아닌, ID가 더 낮은 A부터 얻으려고 시도함
    let h2 = thread::spawn(move || {
        // 만약 여기서 r2_b를 먼저 잠갔다면 데드락 발생 가능성이 매우 높음
        // "언제나 낮은 ID의 자원을 먼저 획득한다"는 규칙 준수
        let _lock_a = r2_a.data.lock().unwrap();
        println!("  [Thread 2] {} (ID:{}) 선점 시도", r2_a.label, r2_a.id);

        let _lock_b = r2_b.data.lock().unwrap();
        println!("  [Thread 2] 모든 자원 안전하게 확보");
    });

    h1.join().unwrap();
    h2.join().unwrap();

    println!("\n기록: 자원 계층화(Resource Hierarchy)를 통해 런타임 교착 상태를 예방함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

이제 10단계를 정말로 졸업하기 위해 설계자가 갖춰야 할 마지막 태도입니다.
