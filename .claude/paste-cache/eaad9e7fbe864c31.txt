🎓 Zig 전공 101: 1-3. 제어문 - if, while, for 그리고 특별한 switch
Zig의 제어문은 단순한 흐름 제어를 넘어, **표현식(Expression)**으로 사용될 수 있다는 점이 특징입니다. 즉, 제어문의 결과값을 변수에 바로 할당할 수 있습니다.
1. if문: 선택과 할당
Zig의 if는 조건식이 반드시 bool 타입이어야 합니다. (0이나 null을 false로 간주하지 않습니다.)
const std = @import("std");

pub fn main() !void {
    const score: u8 = 85;
    
    // if를 표현식으로 사용하여 결과를 변수에 할당
    const result = if (score >= 90) "A" else if (score >= 80) "B" else "C";
    
    std.debug.print("Grade: {s}\n", .{result});
}

2. while과 for: 반복과 캡처
Zig의 반복문은 안전을 위해 인덱스와 요소를 다루는 방식이 독특합니다.
 * while: 조건이 참인 동안 반복합니다. 루프 본체 뒤에 continue 시 실행될 표현식을 넣을 수 있습니다.
 * for: 배열이나 슬라이스를 순회합니다. 값을 **캡처(|value|)**하여 사용하며, 필요시 인덱스도 함께 가져옵니다.
<!-- end list -->
const items = [_]i32{ 10, 20, 30 };

// for (대상) |요소, 인덱스|
for (items, 0..) |item, i| {
    std.debug.print("Index {d}: value {d}\n", .{i, item});
}

3. switch: 전수 검사(Exhaustiveness)의 강제
Zig에서 가장 강력한 도구 중 하나입니다. switch는 가능한 모든 케이스를 처리해야만 컴파일이 됩니다. 누락된 케이스가 있다면 컴파일러가 "기록이 부족하다"며 경고를 보낼 것입니다.
const status = 2;
const message = switch (status) {
    1 => "준비 중",
    2 => "진행 중",
    3 => "완료",
    else => "알 수 없음", // 모든 경우를 커버해야 함
};

🔬 전공 심화: 제어 흐름 내의 블록 식
Zig에서는 {} 블록 자체가 값을 반환할 수 있습니다. 이를 통해 복잡한 초기화 로직을 깔끔하게 정리할 수 있습니다.
const final_value = blk: {
    var sum: i32 = 0;
    for (items) |item| sum += item;
    break :blk sum; // 'blk' 레이블을 통해 값을 반환
};

📦 포인터와 제어문의 결합 (미리보기)
Zig의 if나 while은 Optional 값(값이 있을 수도, 없을 수도 있는 상태)을 풀 때도 사용됩니다. 이는 이후 메모리 관리 세션에서 상세히 다루겠지만, 제어문이 단순히 흐름만 잡는 게 아니라 데이터의 안전성까지 검증한다는 점을 기억하세요.
