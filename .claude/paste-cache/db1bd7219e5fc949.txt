2. 에러 유니온 타입 (Error Union Type)
Zig에는 try-catch 같은 복잡한 예외 처리 메커니즘이 없습니다. 대신 에러를 값으로 취급합니다.
 * error 열거형: 발생할 수 있는 에러 목록을 정의합니다.
 * ! 기호: 반환 타입 앞에 !를 붙이면 "이 함수는 에러를 반환할 수도 있고, 정상적인 값을 반환할 수도 있다"는 뜻입니다.
<!-- end list -->
const MathError = error{
    DivisionByZero,
};

fn divide(a: i32, b: i32) !i32 {
    if (b == 0) return MathError.DivisionByZero;
    return a / b;
}

3. 에러 처리의 3총사: try, catch, if
에러가 발생할 수 있는 함수를 호출할 때는 반드시 처리를 명시해야 합니다.
 * try: 에러가 발생하면 즉시 호출한 곳으로 에러를 던집니다. (가장 많이 사용됨)
 * catch: 에러가 발생했을 때 기본값을 제공합니다.
 * if (함수) |값|: 에러가 없을 때만 블록을 실행합니다.
<!-- end list -->
pub fn main() !void {
    const result = divide(10, 2) catch 0; // 에러 시 0 반환
    
    // try는 에러 발생 시 main 함수를 즉시 종료하고 에러를 리턴함
    const success = try divide(20, 5); 
    
    std.debug.print("Result: {d}\n", .{success});
}

🔬 전공 심화: 지연 실행 (defer)
함수가 종료되기 직전에 반드시 실행되어야 하는 코드가 있다면 defer를 사용합니다. 파일 닫기나 메모리 해제처럼 **"뒷정리"**를 기록하는 데 필수적입니다.
fn doSomething() !void {
    std.debug.print("1. 리소스 확보\n", .{});
    defer std.debug.print("3. 리소스 해제 (자동 실행)\n", .{});
    
    std.debug.print("2. 작업 수행\n", .{});
    // 함수가 끝나기 직전에 defer 문이 실행됨
}
