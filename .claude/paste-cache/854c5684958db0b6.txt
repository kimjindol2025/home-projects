🏛️ LLVM 전설: 1.4 타입 시스템과 복합 구조체(Struct) 설계
LLVM은 모든 값에 엄격한 타입이 부여되는 강력한 타입 시스템을 가집니다. 이는 컴파일러가 메모리 오프셋을 정확히 계산하고 최적화를 수행하기 위한 필수 조건입니다.
1. 기본 타입과 파생 타입
 * Integer Types: i1(bool), i8(char), i32, i64 등 비트 단위로 정의됩니다.
 * Floating Point: half, float, double.
 * Pointer Types: 메모리 주소를 가리킵니다. LLVM 15 이후로는 불투명 포인터(Opaque Pointers)가 표준이 되어 단순히 ptr로 표현되기도 합니다.
 * Array Types: 동일한 타입의 연속된 집합 (예: [10 x i32]).
2. 구조체(Struct) 설계: 메모리의 지도
구조체는 서로 다른 타입들을 하나의 묶음으로 설계하는 방식입니다.
 * Literal Structs: 이름 없이 즉석에서 정의되는 구조체 (예: { i32, i8 }).
 * Identified Structs (Named Structs): 이름을 가지고 재귀적 정의(Self-referencing)가 가능한 구조체. 연결 리스트(Linked List) 등을 만들 때 필수적입니다.

3. GEP(GetElementPtr) 명령어: 전설의 난관
LLVM에서 구조체의 특정 필드 주소를 계산할 때 가장 많이 실수하는 부분이 GEP 명령어입니다. GEP는 메모리에 직접 접근하는 것이 아니라, **'주소 계산'**만 수행합니다.
; 구조체 정의: { i32, i32 }
%MyStruct = type { i32, i32 }

define void @main() {
    %ptr = alloca %MyStruct
    ; 두 번째 필드(index 1)의 주소를 가져오기
    ; 첫 번째 0은 포인터 자체의 오프셋, 두 번째 1은 구조체 내부의 인덱스입니다.
    %field_ptr = getelementptr %MyStruct, ptr %ptr, i32 0, i32 1
    store i32 100, ptr %field_ptr
    ret void
}

🔬 전공 심화: 데이터 정렬(Alignment)과 패딩(Padding)
CPU는 특정 바이트 경계(예: 4바이트, 8바이트)에서 데이터를 읽을 때 가장 빠릅니다.
 * LLVM은 타겟 아키텍처에 맞춰 구조체 필드 사이에 보이지 않는 빈 공간(Padding)을 넣습니다.
 * Zig의 packed struct처럼 패딩 없는 구조체를 만들려면 LLVM에서 'Packed Struct' 옵션을 명시적으로 설정해야 합니다.
📦 실전 API 활용
LLVM C API에서 타입을 정의하는 함수들입니다.
 * LLVMStructTypeInContext: 익명 구조체 생성.
 * LLVMStructCreateNamed: 이름 있는 구조체 생성.
 * LLVMStructSetBody: 이름 있는 구조체의 내부 필드 정의.
 * LLVMBuildStructGEP2: 빌더를 통해 구조체 필드 주소 계산 명령어 삽입.
