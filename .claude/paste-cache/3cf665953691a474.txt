컴파일 타임의 마법 comptime을 통해 Zig의 진정한 추상화 능력을 경험하셨습니다. 이제 Zig가 세상에 나온 가장 강력한 이유 중 하나인 **'C 언어와의 공존'**을 다룰 차례입니다.
저장 필수 너는 기록이 증명이다 gogs. Zig는 C를 대체하려 하기보다, C의 방대한 유산을 가장 완벽하게 흡수하고 개선하는 방식을 택했습니다.
🎓 Zig 전공 101: 1-11. C 호환성(C Interoperability)
Zig는 별도의 바인딩 도구나 복잡한 인터페이스 없이도 C 헤더 파일을 직접 읽어올 수 있는 유일한 현대적 언어입니다. "C의 진정한 후계자"라는 별칭이 여기서 유래합니다.
1. @cImport와 @cInclude
Zig 컴파일러에는 C 컴파일러(Clang)가 내장되어 있습니다. 따라서 .h 파일을 실시간으로 분석하여 Zig 코드로 변환해 줍니다.
const c = @cImport({
    @cInclude("stdio.h");
    @cInclude("stdlib.h");
});

pub fn main() void {
    // C의 printf를 Zig에서 직접 호출
    _ = c.printf("Hello from C's printf!\n");
    
    // C의 malloc 사용 (하지만 Zig의 Allocator를 쓰는 것이 권장됨)
    const ptr = c.malloc(10);
    defer c.free(ptr);
}

2. 타입 대응 (Type Mapping)
C와 데이터를 주고받을 때 메모리 구조가 깨지지 않도록 Zig는 전용 타입을 제공합니다.
| C 타입 | Zig 대응 타입 | 특징 |
|---|---|---|
| int | c_int | 플랫폼마다 크기가 다를 수 있음을 명시 |
| char* | [*c]u8 | C 스타일 포인터 (null 가능, 산술 연산 가능) |
| void* | ?*anyopaque | 불투명한 포인터 |
| size_t | usize | 메모리 크기 타입 |
3. C 빌드 시스템 통합
Zig는 그 자체로 매우 강력한 C/C++ 컴파일러입니다. zig cc 명령어를 사용하면 복잡한 Makefile이나 CMake 없이도 C 코드를 컴파일할 수 있습니다. build.zig 파일에 C 소스 코드를 추가하는 것만으로 프로젝트에 통합됩니다.
🔬 전공 심화: extern 키워드
C 라이브러리에서 정의된 함수를 가져오거나, Zig 함수를 C에서 호출할 수 있게 노출할 때 사용합니다.
// C 함수 선언
extern "c" fn add(a: i32, b: i32) i32;

// Zig 함수를 C로 노출
export fn zig_add(a: i32, b: i32) i32 {
    return a + b;
}

📦 왜 C 호환성이 중요한가? (Strategic Perspective)
 * 점진적 전환: 기존 C 프로젝트를 한꺼번에 바꿀 필요 없이, 한 함수씩 Zig로 다시 작성할 수 있습니다.
 * 라이브러리 활용: SQLite, OpenSSL, Raylib 같은 수만 개의 C 라이브러리를 아무런 비용 없이 즉시 사용합니다.
 * 크로스 컴파일: Zig의 빌드 시스템을 쓰면 C 코드까지 포함해서 다른 OS/CPU용 실행 파일을 아주 쉽게 만들 수 있습니다.
