파이썬 학습의 네 번째 관문, [v1.4: 객체의 마법 - 매직 메소드(Magic Methods)와 프로토콜] 단원을 시작합니다.
Gogs-Lang이나 Rust에서는 연산자 오버로딩을 위해 엄격한 Trait를 구현해야 했습니다. 하지만 파이썬의 대학 과정(새로운 사고)에서는 **더블 언더스코어(__)**로 시작하고 끝나는 이른바 **'던더(Dunder) 메소드'**를 정의하는 것만으로 객체에 특수한 능력을 부여할 수 있습니다.
🐍 Python v1.4: 객체 지향의 마법(Dunder)
제1.4단원: [언어의 문법에 침투하는 객체]
1. 탐구(대학): "객체는 어떻게 연산자와 대화하는가?"
 * 새로운 사고: a + b를 실행하면 파이썬 내부에서는 a.__add__(b)가 호출됩니다. len(a)는 a.__len__()을 호출하죠.
 * 이처럼 특정 메소드를 구현함으로써, 우리가 만든 클래스가 파이썬 내장 타입(리스트, 숫자 등)처럼 자연스럽게 동작하게 만드는 것을 **'파이썬 프로토콜'**이라고 합니다.
2. 증명(대학원): "인터페이스의 일관성"
 * 직접 설계한 클래스가 파이썬의 내장 함수들과 완벽히 호환됨을 보임으로써, **'덕 타이핑(Duck Typing)'**의 실체를 입증합니다. (오리처럼 걷고 오리처럼 운다면, 그것은 오리다!)
🛠️ Python-Gogs 저장소: [v1.4 Smart Vector]
클로드코드(연구원)에게 수학적 벡터 연산을 파이썬 문법으로 우아하게 처리하는 클래스를 기록하게 하세요.
# [Python v1.4: Magic Methods and Operator Overloading]

class GogsVector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        """객체를 문자열로 표현 (개발자용 기록)"""
        return f"GogsVector({self.x}, {self.y})"

    def __add__(self, other):
        """'+' 연산자 정의"""
        if isinstance(other, GogsVector):
            return GogsVector(self.x + other.x, self.y + other.y)
        return NotImplemented

    def __len__(self):
        """len() 함수에 반응 - 여기서는 벡터의 길이를 정수로 반환"""
        return int((self.x**2 + self.y**2)**0.5)

    def __getitem__(self, index):
        """인덱싱(v[0]) 가능하게 만들기"""
        coords = (self.x, self.y)
        return coords[index]

# 실증 실험
v1 = GogsVector(3, 4)
v2 = GogsVector(1, 2)

print("--- v1.4 매직 메소드 실증 실험 ---")
print(f"  [ADD] v1 + v2 = {v1 + v2}")  # __add__ 호출
print(f"  [LEN] 벡터 v1의 크기: {len(v1)}")  # __len__ 호출
print(f"  [INDEX] v1의 x좌표: {v1[0]}")  # __getitem__ 호출

print("\n결론: 매직 메소드를 통해 사용자 정의 객체가 언어 표준 문법에 통합됨.")
print("저장 필수 너는 기록이 증명이다 gogs")
