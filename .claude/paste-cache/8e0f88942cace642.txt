당신은 방금 전 '오메가 프로토콜'이라는 불가능한 미션을 통해 러스트의 절대적 금기를 정면으로 마주했습니다. 설계자가 언어의 한계를 명확히 인지하는 것만큼 중요한 시험은 없습니다.
하지만, 연대기의 정식 절차에 따라 당신의 실력을 **'작동하는 코드'**로 증명해야 하는 **[제4장: 데이터 구조화 — 통합 기말고사 (v5.9)]**를 정식으로 발령합니다. 이번 시험은 거짓이 아닌, 지금까지 배운 모든 구조화 기술을 총동원하여 **'실제로 완벽하게 돌아가는 시스템'**을 구축하는 것입니다.
🏆 제4장 정식 기말고사: "차세대 은하 연합 통신 노드" 설계
당신은 은하계 곳곳에 흩어진 통신 노드들을 관리하는 중앙 제어 엔진을 설계해야 합니다. 아래의 요구사항을 하나의 프로그램 안에 녹여내십시오.
[기술 요구사항 - 5계층 설계]
 * 데이터 정의 (v5.0, v5.2, v5.8.2):
   * SignalType 열거형: Beacon, Data(String), Emergency(u8) 상태를 가집니다.
   * NodeID 뉴타입: u32를 감싸는 구조체로 정의하여 ID의 엄격함을 유지합니다.
   * Node 구조체: NodeID, location(String), last_signal(SignalType) 필드를 가집니다.
 * 행동 구현 (v5.1):
   * Node 구조체에 impl 블록을 생성하고, 새로운 노드를 만드는 new 함수와 신호를 갱신하는 update_signal 메서드를 만드십시오.
 * 동적 관리 (v5.5, v5.7):
   * GalaxyNetwork 구조체를 만드십시오. 내부에 HashMap<u32, Node>를 사용하여 노드들을 ID별로 저장하고 관리합니다.
 * 안전한 처리 (v5.3, v5.4):
   * 특정 ID의 노드를 찾아 그 신호가 Emergency일 경우에만 특별 경고를 출력하는 로직을 match와 Option을 활용해 구현하십시오.
 * 범용성 확보 (v5.8.1):
   * (보너스) 어떤 타입의 부가 데이터든 담을 수 있는 Packet<T> 제네릭 구조체를 하나 정의하여 시스템에 통합하십시오.
🛠️ 클로드코드(학생)에게 내릴 최종 지시문
이 메시지를 복사하여 클로드코드에게 전달하고, 생성된 코드가 당신의 설계 철학에 부합하는지 감독하십시오.
> "클로드코드, 제4장 통합 기말고사를 집행한다.
>  * NodeID 뉴타입과 SignalType 열거형을 정의하라.
>  * Node 구조체와 이를 관리할 HashMap 기반의 GalaxyNetwork를 구축하라.
>  * 노드를 등록하고, 안전하게 검색하며, 패턴 매칭으로 비상 상황을 처리하는 로직을 포함하라.
>  * 모든 코드는 소유권 원칙을 준수하며, 불필요한 복사 없이 참조를 효율적으로 사용해야 한다.
>  * 마지막에 '저장 필수 너는 기록이 증명이다 gogs'를 출력하여 제4장 연대기를 마무리하라."
> 
🎓 선생님의 최종 채점 기준
 * 유효성: 코드가 컴파일 오류 없이 완벽하게 작동하는가?
 * 안전성: unwrap() 대신 match나 if let을 사용하여 잠재적 패닉을 방지했는가?
 * 구조화: 데이터와 로직이 struct와 impl로 깔끔하게 분리되어 있는가?
> 저장 필수 너는 기록이 증명이다 gogs
