Kim님의 Z-Lang 설계 철학이 반영된 CodeGenerator.cpp의 실제 동작을 시뮬레이션하는 두 번째 단계입니다. 예제 1(산술 연산)이 메모리 접근(alloca, load, store)의 기초였다면, 예제 2는 **제어 흐름(Control Flow)**이라는 컴파일러의 지능을 증명하는 단계입니다.
제시해주신 가이드를 바탕으로, **Z-Lang의 실시간성(Zero-Latency)**을 극대화한 최적화된 형태의 LLVM IR을 작성하겠습니다.
🏛️ 예제 2: 조건문(IF-ELSE) LLVM IR 변환 결과
; Z-Lang 소스: fn max(a: i64, b: i64) -> i64 { if a > b { return a; } else { return b; } }
define i64 @max(i64 %a, i64 %b) {
entry:
  ; --- Step 1: 조건 평가 (Comparison) ---
  ; 파라미터 %a와 %b를 직접 비교 (Signed Greater Than)
  ; 결과값 %cond는 i1 (boolean) 타입으로 생성됨
  %cond = icmp sgt i64 %a, %b

  ; --- Step 2: 조건 분기 (Conditional Branch) ---
  ; %cond가 1(true)이면 then 블록으로, 0(false)이면 else 블록으로 점프
  br i1 %cond, label %then, label %else

then:
  ; --- Step 3: 참인 경우 (a > b) ---
  ; 추가적인 메모리 로드 없이 파라미터 %a를 즉시 반환
  ret i64 %a

else:
  ; --- Step 4: 거짓인 경우 (a <= b) ---
  ; 추가적인 메모리 로드 없이 파라미터 %b를 즉시 반환
  ret i64 %b
}

🔍 전설의 검증 리포트 (Architectural Review)
 * 레지스터 기반 최적화: 가이드에서 강조하신 대로 alloca를 사용하지 않고 파라미터 레지스터(%a, %b)를 직접 활용했습니다. 이는 메모리 대역폭을 아끼고 최적의 실행 속도를 보장합니다.
 * Basic Block의 독립성: then과 else 블록이 명확히 구분되어 있으며, 각 블록이 ret(종결자 명령어)로 완결성을 갖춥니다.
 * 예측 가능성(WCET): 분기(Branch) 구조가 단순하여 CPU의 분기 예측기(Branch Predictor)가 학습하기 용이하며, 실시간 시스템에서 실행 시간을 수학적으로 계산하기 매우 유리한 구조입니다.
