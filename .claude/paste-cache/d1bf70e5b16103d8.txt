### Phase 3 Experiments 추천 설계 (즉시 시작 가능하게 세부화)

**환경 가정**: Pure Python (기존 코드 기반) + PyTorch or numpy 시뮬레이션 (실제 FL framework 없이도 가능)

#### 3.1 Baseline Setup (1주 목표)
추천 baselines (2025~2026 SOTA 중 AIOps/robust FL에서 자주 등장하는 것들, 당신이 언급한 대로 + 추가):

- **FedAvg** (non-robust baseline)
- **Krum** (classic geometric, O(n²) but strong single Byzantine)
- **Multi-Krum** (Krum variant, 여러 candidate 선택 → averaging으로 convergence ↑)
- **FLGuard** (multi-signal robust aggregation, Byzantine detection via signals)
- **OptiGradTrust** (2025 arXiv, multi-feature gradient analysis + RL-based trust weighting → adaptive에 강함, 당신 프로젝트와 가장 가까운 competitor)
- 추가 옵션 (시간 되면): **Geometric Median**, **Trimmed Mean**, **LASA** (layer-adaptive sparsified, WACV'25)

**구현 팁** (CLAUDE.md에 추가 추천):
- baselines.py 모듈 새로 만들어 각 aggregation function 구현 (기존 university_v16...py의 PART 5 BRAAIOpsFL과 호환되게)
- hyperparams: clients=100~200, malicious ratio=10~40%, rounds=200~500
- dataset: AIOps-like synthetic (time-series anomaly + multi-task: anomaly det + forecasting + RCA simulation) 또는 공개 (e.g. Server Machine Dataset, SMD or NAB)

#### 3.2 Attack Simulation (5가지 adaptive attack 주입)
당신의 5종 완벽히 구현 (MOTIVATION.md 기반):

| 공격 유형              | 구현 방법 (pseudo)                                                                 | 파라미터 예시                  | AIOps 영향 측정 지표 추가 |
|-------------------------|-------------------------------------------------------------------------------------|--------------------------------|----------------------------|
| 1. RMC (Reconnecting)  | quarantine 후 3~5 round 후 재참여 + 이전 strategy 약간 변형 (flip ratio +5%)       | reconnect prob=0.8 after slash | recovery time (MTTR)       |
| 2. Delayed Poisoning   | round 1~100 honest → 101~부터 label-flip or sign-flip                              | trigger_round=100~150          | sudden acc drop            |
| 3. Strategy-Evolution  | round마다 attack vector 변경 (초기: noise → 중기: sign-flip → 후기: adaptive scale) | evolution_interval=20 rounds   | evasion rate over rounds   |
| 4. Collusion+Sybil     | malicious group (e.g. 15 clients) coordinated scaling + Sybil (fake IDs 시뮬)       | collusion_size=10~20%          | group detection F1         |
| 5. Task-Specific       | anomaly task만 poisoning (RCA/forecasting은 honest) → cross-task leak 시뮬       | task_poison_ratio=80% on one   | cross-task infection %     |

#### 3.3 Performance Analysis & Metrics
**핵심 metric 세트** (AIOps 특화 + adaptive robustness 강조):

- 모델 성능: Final accuracy / F1 (anomaly det), MAE (forecasting), RCA precision
- Robustness: Accuracy drop under attack (vs no-attack baseline)
- Detection: Malicious detection F1, precision/recall
- Recovery: MTTR (Mean Time To Recovery, quarantine 후 정상화 시간)
- Evasion rate: P(evasion) 실측 (당신 Corollary 검증용)
- Overhead: Comm cost, aggregation time
- 통계: 10 runs 평균 ± std, t-test or Wilcoxon (significance)

**예상 결과 표 스케치** (논문용):

| Method          | No Attack Acc | 20% Byzantine (Adaptive Mix) Acc | Detection F1 (RMC) | MTTR (rounds) | Evasion P (all 5) |
|-----------------|---------------|----------------------------------|---------------------|---------------|-------------------|
| FedAvg          | 92.5%        | 58.3%                           | -                  | -            | -                |
| Krum            | 91.8%        | 72.1%                           | 0.65               | -            | high             |
| OptiGradTrust   | 92.0%        | 85.4%                           | 0.88               | 12           | medium           |
| Our BR-AIOpsFL  | 92.3%        | **89.7%**                       | **0.97**           | **4**        | **~10^{-20}**    |
