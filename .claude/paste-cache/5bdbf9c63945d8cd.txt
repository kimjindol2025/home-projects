📋 [v5.1-Audit] 파이썬 통합 문법 정밀 진단
다음 코드는 '실험 데이터 스트리밍 시스템'의 일부입니다. 총 3곳의 주요 문법적/논리적 오류가 숨겨져 있습니다. 설계자의 눈으로 이를 찾아내어 교정해 보세요.
import asyncio
import time

# [진단 1] 데코레이터 설계
def gogs_timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs) # <- 이 지점을 주목하세요
        print(f"소요 시간: {time.time() - start:.4f}s")
        return result
    return wrapper

class GogsDataSystem:
    def __init__(self, data_stream):
        self.data_stream = data_stream
        self.processed_count = 0

    # [진단 2] 비동기와 데코레이터의 결합
    @gogs_timer
    async def analyze_node(self, node_id):
        await asyncio.sleep(0.5) # 분석 시뮬레이션
        self.processed_count += 1
        return f"Node-{node_id} Complete"

# [진단 3] 제너레이터 활용 (v3.2)
def data_generator(limit):
    for i in range(limit):
        yield i
        # yield 이후에 i += 1을 명시하지 않아도 작동할까요?

# --- 실행부 ---
async def main():
    nodes = data_generator(5)
    system = GogsDataSystem(nodes)
    
    # 여기서 각 노드를 비동기로 처리하려고 합니다.
    tasks = [system.analyze_node(n) for n in nodes]
    results = await asyncio.gather(*tasks)
    print(results)

if __name__ == "__main__":
    asyncio.run(main())

🔍 점검 포인트 (설계자님을 위한 힌트)
 * 비동기 함수를 일반 데코레이터로 감싸면 어떻게 될까요?
   * gogs_timer 내부의 func(*args, **kwargs)는 코루틴 객체만 반환하고 바로 종료될까요, 아니면 실행이 끝날 때까지 기다려줄까요? (힌트: await의 부재)
 * time.time()은 비동기 환경에서 정확할까요?
   * 여러 작업이 번갈아 가며 멈추는 비동기 환경에서 순수하게 한 함수의 실행 시간만 잴 수 있을까요?
 * 제너레이터의 소모성:
   * tasks 리스트 컴프리헨션에서 nodes(제너레이터)를 한 번 순회하고 나면, nodes 내부에는 무엇이 남아있을까요?
