### Claude Code를 활용한 추천 전략 (2026 기준 최적화)
Claude Code는 **대형 연구+코드 프로젝트**에 딱 맞아요. 특히 당신처럼 **문서(md) + Python 코드 + 이론 증명**이 섞인 multi-phase 프로젝트에서 강력합니다. 아래는 지금 바로 적용 가능한 팁 + 다음 단계 플랜입니다.

1. **CLAUDE.md를 프로젝트 "뇌(brain)"로 업그레이드** (즉시 해야 할 1순위)
   - 프로젝트 루트에 CLAUDE.md 파일 하나 만들어서 모든 지식 중앙화.
   - 내용 예시 (당신 프로젝트에 맞춰 copy-paste 추천):

     ```markdown
     # BR-AIOpsFL Project Constitution (v16 Byzantine-Resilient FL)

     ## Project Overview
     - Goal: AIOps-specific Byzantine-resilient Federated Learning with DPoS quarantine + trajectory detection + task-adaptive aggregation
     - Key novelty: 5 adaptive attack resilience (RMC, Delayed, Strategy-Evolution, Collusion+Sybil, Task-Specific)
     - Tech stack: Python 3.10+, PyTorch (FL sim), numpy/pandas, scikit-learn (Isolation Forest), on-chain sim (web3.py or custom)

     ## Directory Structure
     docs/          : MOTIVATION.md, LITERATURE_REVIEW.md, ALGORITHM_DESIGN.md, THEORETICAL_FOUNDATION.md
     code/          : university_v16_BR.py (core), test_v16_BR.py
     experiments/   : datasets, attack injectors, baselines (Krum, etc.)
     results/       : plots, tables

     ## Coding Conventions
     - Use type hints everywhere
     - Docstrings: Google style
     - Tests: pytest, 100% coverage on critical paths
     - Commit msg: "Phase X: short desc [test pass]"

     ## Workflow Rules
     - Always Plan first: /plan "implement next step from Phase 3"
     - Use sub-agents for parallel tasks (e.g. attack sim vs baseline impl)
     - After code gen: run tests automatically via hook
     - Reference files: MOTIVATION.md for attacks, THEORETICAL_FOUNDATION.md for proofs

     ## Adaptive Attack Knowledge
     1. RMC: Reconnecting → DPoS slash + trajectory history block
     ...
     (5가지 전부 요약 + Corollary 참조)

     ## Next Phase (Phase 3 Experiments)
     - Baselines: Krum, Multi-Krum, FLGuard, OptiGradTrust
     - Attacks: 5 adaptive 시나리오 구현
     - Metrics: acc drop, recovery time, F1 on detection, comm overhead
     ```

   → Claude Code 실행 시 (`claude` 명령어) 이 파일 자동 로드 → 매번 Phase/context 설명 반복 안 해도 됨. (이미 13KB 넘는 monorepo에서도 잘 돼요)

2. **Phase 3 Experiments를 Claude Code로 가속화하는 방법**
   - **Plan Mode** 적극 활용: `/plan`으로 "5 adaptive attack simulator + baseline comparison 설계" 먼저 짜게 함 → 상세 step-by-step plan 나오면 하나씩 `/execute` 또는 직접 implement 요청.
   - Sub-agents: 병렬로
     - Agent 1: attack injector (RMC, Delayed 등 5개)
     - Agent 2: baseline impl (Krum 등)
     - Agent 3: metrics logger & plot generator (matplotlib)
   - Hooks 설정: code gen 후 자동 `pytest` 돌리게 (post-write hook)
   - Git integration: Claude가 직접 commit/push 가능 (안전하게 review 후)

3. **Claude.ai 웹 + Claude Code 병행 추천 (하이브리드)**
   - 웹 Claude (Projects 기능): MOTIVATION/LIT REVIEW/THEORY md 파일들 Projects에 업로드 → RAG처럼 참조
   - Claude Code (terminal): 실제 코드/테스트/실험 실행
   - 문서 ↔ 코드 sync: md 수정 시 Claude Code에 "update constitution from MOTIVATION.md" 요청
