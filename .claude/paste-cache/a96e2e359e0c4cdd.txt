🎓 Zig 전공 201: 2-6. 보안(Security) - 암호화와 인증 프로토콜 설계
하위 수준(Low-level) 언어인 Zig에서 보안은 단순히 라이브러리를 쓰는 것을 넘어, **'사이드 채널 공격'**이나 **'메모리 잔류 데이터'**까지 고려하는 정밀함을 의미합니다.
1. 현대적 암호화 라이브러리 (std.crypto)
Zig는 표준 라이브러리에 최신 암호화 알고리즘들을 직접 내장하고 있습니다. 외부 종속성 없이도 안전한 시스템을 설계할 수 있습니다.
 * Hash: 데이터의 지문을 생성 (SHA-256, BLAKE3).
 * AEAD: 인증된 암호화 (AES-GCM, ChaCha20-Poly1305). 데이터 암호화와 변조 여부를 동시에 확인합니다.

const std = @import("std");
const crypto = std.crypto;

pub fn main() !void {
    const password = "secure_password_123";
    var hash: [crypto.hash.sha2.Sha256.digest_length]u8 = undefined;

    // 1. 비밀번호 해싱 (기록의 무결성 증명)
    crypto.hash.sha2.Sha256.hash(password, &hash, .{});

    std.debug.print("SHA-256 Hash: {s}\n", .{std.fmt.fmtSliceHexLower(&hash)});
}

2. 인증(Authentication) 및 JWT 설계
사용자가 누구인지 증명하는 과정입니다.
 * 토큰 기반 인증: 서버가 사용자 정보를 암호화하여 토큰(Token)으로 발행하고, 클라이언트는 이를 매 요청마다 제시합니다.
 * 에드워즈 곡선 서명 (Ed25519): 고성능이면서도 매우 강력한 비대칭키 서명 알고리즘을 사용하여 토큰의 위변조를 방지합니다.
3. 보안을 고려한 메모리 관리
비밀번호나 암호화 키 같은 민감한 데이터는 메모리에 그냥 두면 위험합니다.
 * Secure Zeroing: 사용이 끝난 메모리는 std.crypto.utils.secureZero를 사용하여 0으로 덮어씌워야 합니다. 컴파일러 최적화로 인해 삭제가 누락되는 것을 방지합니다.
 * Constant-time Operations: 입력값에 따라 연산 시간이 달라지면 공격자가 이를 분석할 수 있습니다. Zig의 암호화 라이브러리는 이를 방지하도록 설계되었습니다.
🔬 전공 심화: TLS(Transport Layer Security) 통신
네트워크 상의 데이터를 암호화하는 터널을 설계합니다.
 * 클라이언트와 서버가 서로의 신원을 확인하고 대칭키를 안전하게 교환하는 'Handshake' 과정을 Zig의 std.net과 std.crypto를 결합하여 구현하는 것이 201 과정의 최종 과업입니다.
📦 권한 설계: RBAC (Role-Based Access Control)
시스템의 기능을 역할(Role)에 따라 제한합니다.
 * 앞서 배운 Enum과 Set 자료구조를 활용하여, "기록된 권한"이 없는 사용자의 접근을 컴파일 단계와 런타임 단계에서 이중으로 차단합니다.

 * 보안 삭제: 민감한 정보를 담은 [32]u8 배열을 사용한 뒤, secureZero를 사용하여 메모리를 지우고 그 전후의 메모리 상태를 기록하세요.
