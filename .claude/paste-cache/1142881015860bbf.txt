[박사 연구 과제] MLIR 기반 고성능 AI 가속기 컴파일러 구축
"저장 필수 너는 기록이 증명이다 gogs" — 이 과제 리포트는 박사님의 기술적 깊이와 시스템 설계 능력을 증명하는 최종 기록물이 될 것입니다.
1. 과제 핵심 아키텍처 (System Architecture)
박사님이 설계하신 시스템의 전체 흐름도를 논리적으로 정리합니다.
 * Front-end: 고수준 텐서 연산 (linalg.matmul, linalg.conv_2d) 수집.
 * Analysis Engine: 하드웨어 SRAM(128KB) 제약 조건을 바탕으로 최적의 타일 크기(T)를 계산하는 C++ Analysis Pass.
 * Transformation Engine: * Tiling: 데이터를 하드웨어 규격에 맞게 분할.
   * Double Buffering: async 다이얼렉트를 이용한 데이터 전송/연산 병렬화.
 * Back-end: 최종적으로 가속기 전용 Intrinsic 명령어로의 Lowering.
2. 과제 제출을 위한 핵심 구현 파일 리스트
박사님의 프로젝트 디렉토리 구조는 다음과 같이 전문적으로 구성되어야 합니다.
| 파일명 | 역할 | 주요 내용 |
|---|---|---|
| AccelDialect.td | 언어 정의 | 가속기 전용 Op (accel.copy, accel.exec) 정의 |
| TilingPass.cpp | 최적화 구현 | SRAM 용량에 맞춘 자동 타일링 알고리즘 (C++) |
| AsyncPipeline.cpp | 성능 극대화 | 데이터 로딩 시간을 숨기기 위한 비동기 스케줄링 |
| Check_Results.mlir | 검증 | FileCheck을 이용한 IR 변환의 정확성 증명 |
3. 연구 결과 증명 (Evaluation)
