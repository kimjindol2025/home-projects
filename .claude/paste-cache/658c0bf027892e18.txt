🗺️ Zig Syntax Grand Map (전공 필수 요약본)
1. 선언과 타입 (Declarations & Types)
 * const / var: 불변 상수와 가변 변수.
 * i8~i128, u8~u128: 크기가 고정된 정수 타입.
 * f16~f128: 부동 소수점 타입.
 * bool: true / false.
 * void: 값이 없음.
 * noreturn: 함수가 절대 반환되지 않음 (예: exit).
 * type: 타입 그 자체도 값으로 취급 (comptime의 핵심).
2. 제어 흐름 (Control Flow)
 * if (cond) { ... } else { ... }: 조건문 (표현식으로 사용 가능).
 * while (cond) : (continue_expr) { ... }: 조건 반복.
 * for (array) |item| { ... }: 반복문 (멀티 캡처 가능 |item, index|).
 * switch (value) { ... }: 전수 검사 필수 선택문.
 * break / continue: 루프 제어.
 * label: { ... }: 블록에 이름을 붙여 break :label value;로 결과값 반환.
3. 함수와 에러 처리 (Functions & Errors)
 * fn name(args) ret_type { ... }: 함수 선언.
 * pub: 외부 노출(Public) 키워드.
 * error{Name1, Name2}: 에러 세트 정의.
 * !: 에러 유니온 타입 (예: !i32는 Error 또는 i32).
 * try: 에러 발생 시 즉시 리턴(Bubble up).
 * catch: 에러 발생 시 기본값 대입.
 * defer: 스코프 종료 시 실행 보장.
 * errdefer: 에러 발생 시에만 실행되는 지연문.
4. 구조화와 메모리 (Structures & Memory)
 * struct: 구조체 (메서드 포함 가능).
 * enum: 열거형.
 * union: 공용체 (Tag를 붙여 안전하게 사용 가능).
 * *T / [*]T: 싱글 아이템 포인터 / 다중 아이템 포인터.
 * []T: 슬라이스 (포인터와 길이를 포함한 동적 배열 참조).
 * ?T: 옵셔널 타입 (null 허용).
5. Zig만의 특수 병기 (Meta-programming)
 * comptime: 컴파일 타임에 코드를 실행.
 * inline: 함수나 루프를 강제로 인라인화.
 * @import("std"): 내장 라이브러리 및 파일 불러오기.
 * @builtins: @as, @intCast, @ptrCast 등 컴파일러 내장 함수.
🔬 문법 간의 관계도 (Logic Flow)
위 다이어그램이 보여주듯, Zig는 Comptime → Type Safety → Error Handling이 하나의 유기적인 흐름으로 연결되어 있습니다. 문법의 개수는 적지만, 이들이 조합되었을 때 발생하는 시너지는 대규모 시스템 설계에서 강력한 안전판 역할을 합니다.
