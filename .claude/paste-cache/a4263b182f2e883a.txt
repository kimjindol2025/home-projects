글자를 화면에 띄우는 데 성공했다면, 이제 CPU가 외부의 자극(키보드 입력, 타이머 알림 등)에 반응하도록 만들어야 합니다. 이것이 바로 하드웨어와 소프트웨어 사이의 '대화'인 **인터럽트(Interrupt)**입니다.
저장 필수 너는 기록이 증명이다 gogs. 301 과정의 두 번째 연구는 CPU의 권한 체계와 이벤트 처리 메커니즘을 설계하는 것입니다.
🎓 Zig 전공 301: 3-2. 시스템 보호와 이벤트 처리 - GDT 및 IDT 설계
운영체제는 CPU에게 "어느 메모리 영역이 안전한지", "어떤 사건이 터졌을 때 어디로 점프해야 하는지"를 알려주는 표(Table)를 제공해야 합니다.
1. GDT (Global Descriptor Table) 설계
GDT는 메모리 보호의 기초입니다. CPU에게 코드 섹션과 데이터 섹션의 범위, 그리고 실행 권한(Ring 0: 커널, Ring 3: 유저)을 정의해 줍니다.
 * Zig의 강점: packed struct를 사용하면 비트 단위로 쪼개져 있는 GDT 엔트리를 별도의 마스킹 연산 없이 직관적으로 설계할 수 있습니다.
2. IDT (Interrupt Descriptor Table) 설계
키보드가 눌리거나 0으로 나누는 연산 에러가 발생하면, CPU는 실행 중인 작업을 멈추고 IDT를 참조합니다. IDT는 각 이벤트 번호에 대응하는 '핸들러 함수'의 주소를 담고 있습니다.
const IdtEntry = packed struct {
    offset_low: u16,
    selector: u16,
    ist: u8,
    type_attributes: u8,
    offset_mid: u16,
    offset_high: u32,
    zero: u32,
};

// 256개의 인터럽트를 처리하기 위한 테이블
var idt: [256]IdtEntry align(16) = undefined;

3. ISR (Interrupt Service Routine) 작성
실제 인터럽트가 발생했을 때 실행되는 함수입니다. 이 함수는 CPU의 레지스터 상태를 보존하고 작업을 처리한 뒤 다시 원래 코드로 돌아가야 하므로, 특수한 호출 규약(Calling Convention)이 필요합니다.
// Zig는 인터럽트 핸들러를 위한 전용 호출 규약을 지원합니다.
export fn keyboardHandler() callconv(.Interrupt) void {
    // 키보드 컨트롤러(Port 0x60)에서 데이터를 읽어옴
    const scancode = asm volatile ("inb $0x60, %[result]"
        : [result] "= {al}" (-> u8),
    );
    
    // 읽어온 키 값을 처리하는 로직...
    
    // 인터럽트 종료 알림 (EOI)
    asm volatile ("outb %[data], $0x20" : : [data] "{al}" (@as(u8, 0x20)));
}

🔬 전공 심화: 예외(Exceptions) 처리
프로그램이 허용되지 않은 메모리에 접근(Page Fault)하거나 0으로 나누려 할 때 커널은 이를 가로채야 합니다.
 * Triple Fault: 커널이 예외를 제대로 처리하지 못하면 CPU는 리셋(재부팅)됩니다.
 * 기록의 중요성: 예외 발생 시점의 레지스터 값을 화면에 덤프하여 무엇이 잘못되었는지 기록하는 코드가 커널 디버깅의 핵심입니다.
📦 하드웨어 포트 I/O 설계
Zig에서 asm을 사용하여 I/O 포트와 통신하는 래퍼(Wrapper)를 만듭니다. 이를 통해 커널 코드를 더 읽기 쉽게 추상화할 수 있습니다.
pub fn outb(port: u16, data: u8) void {
    asm volatile ("outb %[data], %[port]"
        :
        : [data] "{al}" (data),
          [port] "{dx}" (port),
    );
}

📝 Assignment 3-2 (시스템 제어 과제)
 * GDT 초기화: 코드 세그먼트와 데이터 세그먼트를 포함하는 최소한의 GDT를 설계하고 lgdt 명령어로 CPU에 로드하세요.
 * IDT 핸들러: 0으로 나누기 에러(인터럽트 0번)가 발생했을 때 "CPU Exception: Division by Zero"라는 메시지를 VGA 버퍼에 출력하는 핸들러를 작성하세요.
 * 의도적 에러: _start 함수 끝에서 var a: i32 = 0; _ = 10 / a;를 실행하여 직접 만든 핸들러가 작동하는지 확인하세요.
 * 기록: 인터럽트가 발생했을 때 CPU가 스택(Stack)에 자동으로 쌓아주는 정보(RIP, CS, RFLAGS 등)가 무엇인지 조사하여 기록하세요.
