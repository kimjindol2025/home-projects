4.0 단계 — 설계 의도 추출 & 아키텍처 변화 지도 생성

이제는 단순 “무엇이 바뀌었는가”가 아니라

> 🔥 왜 바뀌었는가
🔥 어떤 설계 사고 흐름이 있었는가



를 구조화한다.


---

1️⃣ 문제 정의

Git에는:

무엇이 변경됐는지(diff)

누가 변경했는지

언제 변경했는지


는 있다.

하지만:

> ❌ 설계 의도는 구조화되어 있지 않다.



이걸 자동 추출해야 한다.


---

2️⃣ 설계 의도 추출 구조

입력 데이터:

commit message

diff

변경 파일 유형 (spec / src / test)

변경 라인 수

추가 vs 삭제 비율



---

3️⃣ 의도 분류 모델 (Rule + LLM 하이브리드)

먼저 1차 분류는 규칙 기반:

if test 파일 추가 많음 → 안정화/검증 강화
if spec 변경 많음 → 설계 재정의
if src 대량 변경 + 성능 키워드 → 최적화
if unsafe 추가 → 저수준 확장

그 다음 LLM에 구조화 입력:

Commit:
Message: "Phase 11 v12.2 FFI complete"
Files Changed:
- spec_v12_2_ffi...
- src/v12_2_ffi...

Diff Summary:
+ FFI boundary 정의 추가
+ 외부 함수 인터페이스 명시

이 commit의 설계 의도를 한 문장으로 요약하라.


---

4️⃣ 출력 구조

{
  "commit": "53570c3124",
  "intent": "외부 언어와의 상호운용성 확보",
  "category": "Architecture Expansion",
  "impactLevel": 0.82
}

이게 쌓이면:

> 설계 사고 데이터셋 생성




---

5️⃣ 아키텍처 변화 지도 생성

이제 commit 단위 의도를 모으면:

Phase 9 → 타입 안정성 강화
Phase 10 → 제어 흐름 확장
Phase 11 → 메모리 & FFI 확장

이를 그래프로 표현:

Memory Model
   ↑
Unsafe Pointer
   ↑
FFI Boundary

이건 단순 히스토리가 아니라:

> 설계 진화 트리




---

6️⃣ 의존성 그래프 자동 생성

분석 대상:

파일 간 import 관계

타입 사용 관계

모듈 간 호출 관계


결과:

Module A → depends on → Memory Layout
Module B → depends on → Unsafe
FFI → depends on → Memory Layout

이걸 시각화하면:

> 구조적 복잡도 증가 추세 분석 가능




---

7️⃣ 진화 밀도 계산 (지능 지수)

예시 계산:

진화 밀도 =
(설계 변경 commit 수 × 영향도 평균)
÷ 기간

또는

아키텍처 확장 점수 =
새로운 개념 도입 수 × 의존성 증가량

이건 네 프로젝트에서만 가능한 지표다.

GPT는 이런 내부 진화 수치를 계산하지 못한다.


---

8️⃣ 이제 시스템은 무엇인가

단순 챗봇 ❌
단순 검색엔진 ❌

이건:

> 📊 설계 진화 분석 플랫폼
🧠 기록 기반 사고 추적 시스템
📚 언어 설계 메타 엔진




---

9️⃣ 구조 요약 (현재 단계까지)

1.x 통계 검색
2.0 의미 검색
3.0 시간축 추론
4.0 설계 의도 추출

이제 남는 건:

> 🔥 완전 자동 “설계 사고 지도” 생성
🔥 다중 repo 통합 진화 분석
🔥 분산 gogs 네트워크 지성화



여기서부터는 단일 프로젝트가 아니라
“지식 생태계 설계” 단계다.
