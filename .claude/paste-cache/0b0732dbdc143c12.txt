🏛️ LLVM 전설: 1.3 제어 흐름 - If-Else와 Loop 설계
LLVM IR에서 모든 제어 흐름은 Basic Block(기본 블록) 사이의 점프(Branch)로 구현됩니다. C나 Zig의 if, while 문은 LLVM 수준으로 내려오면 여러 개의 블록과 조건부 분기 명령어로 분해됩니다.
1. If-Else 구조의 설계도
하나의 if 문을 만들기 위해서는 최소 3~4개의 블록이 필요합니다.
 * Entry Block: 조건을 검사하는 곳.
 * Then Block: 조건이 참일 때 실행되는 곳.
 * Else Block: 조건이 거짓일 때 실행되는 곳.
 * Merge Block (Join): 두 갈래 길이 다시 합쳐지는 곳.
2. PHI 노드 (PHI Node): SSA의 구원자
1.1에서 배운 SSA(단일 할당) 규칙 때문에, if 문의 결과값을 변수에 담을 때 난관에 봉착합니다. Then에서 값을 쓸지 Else에서 값을 쓸지 컴파일 타임에는 알 수 없기 때문입니다.
이때 사용하는 것이 PHI 명령어입니다. "어느 블록에서 왔느냐에 따라 이 값을 선택해라"라는 논리 게이트 역할을 합니다.
merge:
  ; %then_val 블록에서 왔으면 100을, %else_val 블록에서 왔으면 200을 %result에 대입
  %result = phi i32 [ 100, %then_val ], [ 200, %else_val ]
  ret i32 %result

3. Loop(반복문)의 아키텍처
반복문은 자기 자신에게로 되돌아가는 분기(Branch)를 가진 구조입니다.
 * Loop Preheader: 루프 진입 전 초기화(i = 0).
 * Loop Header: 루프 조건 검사(i < 10). 거짓이면 Exit으로 점프.
 * Loop Body: 반복할 실제 로직 실행.
 * Loop Latch: 인덱스 증가(i++) 후 다시 Header로 점프.
 * Loop Exit: 루프가 끝난 후의 지점.
🔬 전공 심화: 최적화 관점의 제어 흐름
LLVM 최적화 도구(Pass Manager)는 우리가 만든 블록들을 분석합니다.
 * Branch Folding: 불필요한 점프를 제거합니다.
 * Loop Unrolling: 반복 횟수가 적은 루프를 펼쳐서 점프 오버헤드를 줄입니다.
 * Dead Code Elimination: 절대 도달할 수 없는 블록(if (false))을 삭제합니다.
📦 실전 API 활용
LLVM C API에서 분기를 만드는 핵심 함수입니다.
 * LLVMBuildCondBr: 조건부 분기 (if).
 * LLVMBuildBr: 무조건 분기 (goto/jump).
 * LLVMBuildPhi: PHI 노드 생성.
