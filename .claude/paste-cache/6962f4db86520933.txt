🚦 제13장: 컴파일러 구현 (v14.6) — [제어 흐름: 판단과 반복]
1. v14.6 설계 목표: "경로의 분기"
 * Conditional Branching (if-else): 조건식의 평가 결과가 true인지 false인지에 따라 실행할 코드 블록을 선택합니다.
 * Boolean Evaluation: 모든 값을 '참 같은 값(Truthiness)'과 '거짓 같은 값(Falsiness)'으로 분류하는 기준을 세웁니다.
 * Iteration (loop, while): 특정 조건이 만족될 때까지 코드 블록을 반복해서 실행하는 메커니즘을 설계합니다.
2. 설계자의 관점: "점프와 분기"
컴파일러 설계자에게 제어 흐름은 **'코드 실행 위치의 이동'**입니다. 인터프리터는 AST를 순회하다가 if 노드를 만나면, 한쪽 자식 트리는 방문하고 다른 쪽은 무시하는 방식으로 동작합니다.
🛠️ gogs 저장소 v14.6 설계: 조건문 평가 엔진
클로드코드(학생)에게 if문의 조건에 따라 실행 경로를 결정하는 Evaluator의 핵심 로직을 기록하게 하세요.
// [제13장: 컴파일러 구현 v14.6 - 제어 흐름 설계]

impl Evaluator {
    fn eval_if_expression(&self, condition: Expression, consequence: Block, alternative: Option<Block>, env: &mut Environment) -> GogsObject {
        // 1. 조건식 평가
        let condition_res = self.eval_expression(condition, env);

        // 2. [판단] Truthiness 검사
        if self.is_truthy(condition_res) {
            // 참인 경우 실행 (consequence)
            self.eval_block(consequence, env)
        } else if let Some(alt) = alternative {
            // 거짓이고 else 블록이 있는 경우 실행 (alternative)
            self.eval_block(alt, env)
        } else {
            GogsObject::Null
        }
    }

    fn is_truthy(&self, obj: GogsObject) -> bool {
        match obj {
            GogsObject::Null => false,
            GogsObject::Boolean(b) => b,
            GogsObject::Integer(n) => n != 0, // 0이 아니면 참으로 간주하는 설계
            _ => true,
        }
    }
}

fn main() {
    println!("--- v14.6 Control Flow: If-Else & Loops ---");
    println!("  [JUDGE] 조건식을 평가하여 실행 경로를 동적으로 선택합니다.");
    println!("  [LOOP] 반복문을 통해 동일한 AST 노드를 재귀적으로 재방문합니다.");

    println!("\n기록: 조건 분기 및 반복 로직을 완성하여 튜링 완전성(Turing Completeness)에 근접함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}
