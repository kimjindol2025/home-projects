🎓 Go-언어 전공 101: 1-4. 함수 (Functions)
함수는 독립적인 로직의 단위입니다. Go에서는 함수를 정의할 때 func 키워드를 사용하며, 매개변수와 반환 값의 타입을 명확히 규정합니다.
1. 기본 구조와 매개변수
Go의 함수는 타입을 이름 뒤에 씁니다. 이는 읽기 순서(왼쪽에서 오른쪽)를 자연스럽게 만들기 위함입니다.
func add(a int, b int) int {
    return a + b
}

// 같은 타입일 경우 생략 가능 (심화 문법)
func multiply(a, b int) int {
    return a * b
}

2. 다중 반환값 (Multiple Return Values)
Go의 가장 강력한 특징 중 하나는 함수가 여러 개의 값을 반환할 수 있다는 점입니다. 보통 '결과값'과 '에러 객체'를 함께 반환하는 패턴이 표준입니다.
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("0으로 나눌 수 없습니다")
    }
    return a / b, nil
}

 * 대학원 수준의 통찰: C 언어처럼 포인터를 넘기거나, Java처럼 Exception을 던지는 대신, Go는 에러를 '값'으로 취급하여 명시적으로 처리하게 강제합니다. 이는 시스템의 예측 가능성을 극대화합니다.
3. 가변 인자 함수 (Variadic Functions)
전달받을 매개변수의 개수가 정해지지 않았을 때 사용합니다. ... (Ellipsis) 기호를 활용합니다.
func sumAll(nums ...int) int {
    total := 0
    for _, num := range nums {
        total += num
    }
    return total
}

🔬 전공 심화: 일급 함수 (First-class Functions)
Go에서 함수는 **'일급 객체'**입니다. 즉, 함수를 변수에 담을 수도 있고, 다른 함수의 인자로 넘길 수도 있으며, 함수에서 함수를 반환할 수도 있습니다.
 * 익명 함수 (Anonymous Functions): 이름 없이 정의하고 즉시 실행하거나 변수에 할당합니다.
 * 클로저 (Closures): 함수 외부의 변수를 참조하고 수정할 수 있는 함수 상태를 유지합니다. 이는 고도의 추상화 설계 시 필수적인 개념입니다.
