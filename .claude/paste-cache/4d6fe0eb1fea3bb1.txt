### 1.1 최종 정리: AIOps 도메인 특화 Byzantine 공격 시나리오
(논문 Introduction이나 Motivation 섹션에 바로 넣을 수 있는 수준으로 작성)

**주요 시나리오 1 (Primary: 가장 강조 추천)**  
**Label/Target Flipping + Cross-Task Propagation Attack on Multi-Task AIOps**  
- 악의적 edge 노드(compromised 서버/디바이스)가 anomaly detection task에서 정상 로그를 anomaly로, anomaly를 정상으로 뒤집어 라벨링하거나 gradient를 조작  
- 동시에 RCA (Root Cause Analysis) task의 causal graph나 correlation을 왜곡 (e.g. 특정 컴포넌트에 fault를 잘못 귀속)  
- 결과: 글로벌 모델이 **false positive 폭증** → alert fatigue 유발, 운영팀이 중요한 장애를 miss  
  + **잘못된 root cause 지목** → 오진단된 remediation 실행 (e.g. 불필요한 restart or scaling) → 장애 확산 및 downtime 연장  
- AIOps 특성상 multi-task 간 coupling이 강해 한 task의 poisoning이 forecasting이나 다른 task로 전파 → 전체 시스템 신뢰성 붕괴  
- 실무 영향: 비용 폭증 (over-alert + 잘못된 auto-remediation) + 서비스 가용성 저하

**주요 시나리오 2 (Secondary: novelty 강조용)**  
**Adaptive / Reconnecting Byzantine Attack (Delayed or Recon Attack)**  
- 초반 라운드에서는 정상처럼 행동 (honest gradient 제출) → 탐지 피함  
- 특정 조건(peak traffic, critical maintenance window)에서만 poisoning 실행  
- quarantine/detection 후 재접속 → stake 복구 후 재공격 (reconnecting malicious clients)  
- 기존 robust aggregation (Krum, GeoMedian 등)은 **단일 라운드 snapshot**만 봐서 이런 temporal/adaptive 패턴 못 잡음  
- AIOps에서 특히 위험: 실시간 운영 환경에서 "잠복 후 대규모 공격" → sudden false alarm storm or 예측 실패 → cascading failure 유발

**왜 이 두 시나리오가 AIOps에 가장 적합한가?**  
- AIOps는 accuracy drop 자체보다 **운영 영향(Alert fatigue, Downtime, Cost explosion)**이 핵심 → 일반 FL Byzantine 논문(주로 MNIST/CIFAR accuracy 중심)과 차별화  
- Multi-task + time-series 특성 → cross-task contamination + temporal trajectory poisoning이 핵심 gap  
- 2025~2026 최근 논문들(Adaptive Adversaries survey 2025, BR-MTRL 2025, trajectory anomaly detection 관련)에서도 **adaptive/reconnecting**과 **multi-task resilience**이 hot open problem으로 꼽히고 있음 → 우리 DPoS quarantine + trajectory detection + task-wise adaptive agg가 딱 맞아떨어짐

**논문 Motivation 단락 예시 (바로 copy-paste 가능)**  
"Federated learning in AIOps environments enables collaborative model training across distributed edge nodes for tasks such as anomaly detection, root cause analysis (RCA), and forecasting, using heterogeneous time-series and log data. However, Byzantine nodes can launch sophisticated attacks, including label flipping on anomaly detection that triggers excessive false positives and alert fatigue, or targeted poisoning on RCA leading to erroneous remediation and cascading failures. Furthermore, adaptive adversaries that behave honestly initially and poison only during critical periods—or reconnect after quarantine—remain largely unaddressed by existing robust aggregation methods, which focus on single-round snapshots and ignore cross-task propagation and temporal patterns. This work addresses these AIOps-specific threats by proposing ARES-FL, an end-to-end Byzantine-resilient framework with real-time trajectory detection, DPoS-linked dynamic quarantine, and task/layer-adaptive aggregation."
