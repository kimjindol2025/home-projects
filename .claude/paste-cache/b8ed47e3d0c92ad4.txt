메모리 할당의 명시성을 통해 자원 관리의 정점에 도달하셨습니다. 이제 Zig의 가장 독보적인 기능이자, 현대 프로그래밍 언어 설계의 혁신이라 불리는 **'컴파일 타임 메타 프로그래밍'**의 세계로 들어갑니다.
저장 필수 너는 기록이 증명이다 gogs. Zig는 실행 시점(Runtime)에 할 일을 컴파일 시점(Compile-time)으로 앞당겨 성능과 안전성을 동시에 잡습니다.
🎓 Zig 전공 101: 1-10. Comptime - 컴파일 타임의 마법
comptime 키워드는 Zig 컴파일러에게 **"이 코드는 컴파일 중에 실행해!"**라고 명령하는 도구입니다. 이를 통해 별도의 매크로나 복잡한 제네릭 문법 없이도 강력한 추상화를 구현할 수 있습니다.
1. comptime 변수와 함수
컴파일 타임에 결정되는 값은 실행 파일의 크기를 줄이고 속도를 극대화합니다.
const std = @import("std");

fn fibonacci(comptime n: u32) u32 {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

pub fn main() void {
    // 결과값 55는 런타임 연산 없이 컴파일 시점에 상수로 박힙니다.
    const result = comptime fibonacci(10);
    std.debug.print("Fibonacci(10): {d}\n", .{result});
}

2. 제네릭(Generics)의 실체
Zig에는 <T> 같은 특수 문법이 없습니다. 대신 타입을 인자로 받는 함수가 그 역할을 대신합니다.
 * 원리: 함수 인자에 comptime T: type을 선언하면, 컴파일러는 호출 시 사용된 타입별로 특화된 함수를 자동으로 생성(Monomorphization)합니다.
<!-- end list -->
fn List(comptime T: type) type {
    return struct {
        items: []T,
        len: usize,
    };
}

pub fn main() void {
    const IntList = List(i32); // i32를 위한 구조체가 생성됨
    var my_list = IntList{ .items = undefined, .len = 0 };
}

3. 컴파일 타임 리플렉션 (Reflection)
Zig는 자신의 구조를 스스로 들여다볼 수 있습니다. 예를 들어, 어떤 구조체에 특정 이름의 필드가 있는지 컴파일 타임에 확인하고 로직을 분기할 수 있습니다.
🔬 전공 심화: inline 루프와 조건문
comptime 블록 안에서 for나 if를 사용하면 컴파일러는 루프를 완전히 풀어서(Unrolling) 실행 코드를 생성합니다.
const types = [_]type{ i32, f32, bool };

inline for (types) |T| {
    // 각 타입에 대한 코드가 컴파일 시점에 각각 생성됨
    const value: T = undefined;
    std.debug.print("Type: {s}\n", .{@typeName(T)});
}

📦 왜 comptime인가? (Architect's Perspective)
 * 성능: 복잡한 계산을 빌드 단계에서 끝내버립니다.
 * 안전: 잘못된 타입이나 값의 조합을 프로그램이 실행되기도 전에 잡아냅니다.
 * 단순성: 언어의 문법을 늘리지 않고도 메타 프로그래밍이 가능합니다.
