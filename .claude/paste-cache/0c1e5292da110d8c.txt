좋다.
Kernel / Policy 분리 고정으로 간다.
확장 중단. 구조 먼저 잠근다.


---

▶ 2.K — Kernel / Policy 분리 설계 고정

목표

Agent Kernel을 전략 불변(core) 으로 고정하고
모든 실행 전략을 Policy 계층으로 외부화


---

1️⃣ Kernel 책임 (변하지 않는 것)

상태 머신

DAG 실행 엔진

Task 생명주기 관리

Memory append-only 저장

Constraint 강제

이벤트 발생 (onTaskStart / onTaskEnd / onFail)


❗ Kernel은 “결정”하지 않는다.
❗ Kernel은 “집행”만 한다.


---

2️⃣ Policy 책임 (전략 영역)

Retry 여부

Replan 허용 여부

실패 전파 범위

MaxParallel 값

Scheduler 전략 (FIFO / Priority / Cost-based)

Abort vs Continue 결정


❗ Policy는 판단한다.
❗ Kernel은 실행한다.


---

3️⃣ 구조 변경 작업

(1) Execution Loop에서 판단 제거

기존:

if retry_allowed:
    retry()

변경:

decision = policy.onFailure(context)
kernel.apply(decision)


---

(2) Replanner를 Policy 내부로 이동

Replanner는 Kernel 구성요소가 아니라
Policy 전략의 일부가 된다.


---

(3) ReflectionResult 제거 → PolicyDecision 통합

기존:

Continue

Retry

Abort


변경:

enum PolicyDecision {
    Proceed
    Retry(task_id)
    Replan(plan_delta)
    Abort
}

Kernel은 Decision을 집행만 한다.


---

4️⃣ 인터페이스 확정

interface AgentPolicy {
    onTaskSuccess(context) -> PolicyDecision
    onTaskFailure(context) -> PolicyDecision
    onConstraintViolation(context) -> PolicyDecision
}

Kernel은 Policy를 주입받는다.


---

5️⃣ 고정 완료 조건

Kernel 코드에 retry / replan 판단 로직 없음

Policy 없이 Kernel 실행 불가

Policy 교체 시 Kernel 수정 0줄
