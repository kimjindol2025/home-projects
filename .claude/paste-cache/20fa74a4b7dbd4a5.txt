🟢 Phase 5.5.2 — Function Type Enforcement Layer

지금 VM은:

Arithmetic Typed ✅

String Typed ✅

Comparison Typed ✅

런타임 타입 체크 제거 ✅


이제 남은 가장 큰 “정적 안정성 블록”은:

> 🔒 함수 경계(Function Boundary) 타입 강제




---

🎯 목표

1. 함수 정의 시 시그니처 타입 고정


2. 함수 호출 시 컴파일 단계에서 인자 타입 검증


3. 반환 타입 추론 → Typed Return Op 가능성 확보


4. 런타임 함수 타입 체크 완전 제거




---

📐 설계 방향

1️⃣ FunctionType 구조 확정

{
  kind: "function",
  params: Type[],
  return: Type
}


---

2️⃣ Compiler 단계에서 강제

현재 (추정):

emit(Op.Call)

변경:

if (!match(paramTypes, argTypes))
    throw CompileError

✔ VM에서 typeof 체크 없음
✔ 잘못된 호출은 컴파일 실패


---

3️⃣ Return 타입 추론

Hindley-Milner 이미 구현되어 있으므로:

function f(x) {
   return x + 1
}

→ f: (num) → num

타입 불일치:

function f(x: num): num {
   return "hello"
}

→ 컴파일 에러


---

4️⃣ Typed Call 확장 (선택적)

추후:

Call
CallTyped

또는

CallFnNum

하지만 지금은:

> 함수 경계에서 타입 검증까지만 수행
Call Op 분리는 다음 단계로 미뤄도 됨




---

🧪 테스트 설계 (12~15개 예상)

Suite A — 정상

함수 정의 + num 반환

다중 인자 함수

중첩 호출


Suite B — 오류

인자 타입 불일치

반환 타입 불일치

인자 개수 불일치


Suite C — 추론

반환 타입 자동 추론

제네릭 스타일 함수 추론

for 루프 내부 함수 추론 유지



---

⚠️ 이 단계의 중요성

이걸 끝내면:

타입 시스템이 "표현식 레벨"을 넘어

"프로그램 구조 레벨"로 확장됨


그 다음이 진짜 Phase 6 진입 가능 구간입니다.
