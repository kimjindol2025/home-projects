메타클래스는 클래스의 행동을 정의하는 클래스입니다. 이를 통해 설계자가 만든 클래스들이 특정 규칙을 지키고 있는지 검사하거나, 클래스가 생성되는 순간 기능을 주입할 수 있습니다.
1. 모든 것의 뿌리: type
 * 파이썬에서 type()은 객체의 타입을 알려주기도 하지만, 새로운 클래스를 동적으로 생성하는 기본 메타클래스이기도 합니다.
2. 왜 사용하나요? (증명을 위한 엄격함)
 * 표준 준수 강제: "우리 연구실의 모든 로봇 클래스는 반드시 battery_check라는 메서드를 가져야 한다"와 같은 규칙을 강제할 수 있습니다.
 * 자동 등록: 클래스가 정의되는 순간 시스템에 자동으로 등록되게 만들 수 있습니다.
🛠️ 전공 실습: [v3.4 The Creator of Classes]
설계자가 만든 클래스가 특정 이름을 포함해야만 생성을 허용하는 엄격한 메타클래스를 설계해 보겠습니다.
# [Python University v3.4] 메타클래스 설계 (The Meta-Level)

# 1. 메타클래스 정의 (type을 상속받아야 함)
class GogsValidationMeta(type):
    def __new__(mcs, name, bases, attrs):
        # name: 생성될 클래스의 이름
        # bases: 상속받는 부모 클래스들
        # attrs: 클래스에 정의된 속성과 메서드들
        
        print(f"--- [META] 클래스 '{name}' 생성 검전 시작 ---")
        
        # 규칙: 클래스 이름은 반드시 'Gogs'로 시작해야 함
        if not name.startswith("Gogs"):
            raise TypeError(f"규정 위반: 클래스 이름 '{name}'은 'Gogs'로 시작해야 합니다.")
            
        return super().__new__(mcs, name, bases, attrs)

# 2. 메타클래스를 적용한 기본 클래스 생성
class GogsBase(metaclass=GogsValidationMeta):
    pass

# 3. 올바른 설계
class GogsRobot(GogsBase):
    def work(self):
        print("로봇 가동")

# 4. 잘못된 설계 (주석을 해제하면 에러가 발생하여 프로그램이 실행되지 않음)
