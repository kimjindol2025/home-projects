🏗️ Stage 3.1: 파이프라인 대통합 (The Great Integration)
흩어져 있던 Lexer, Parser, Semantic Analyzer, Codegen을 하나의 유기체로 묶습니다.
1. 빌드 엔진 구축 (CMakeLists.txt)
시스템에 설치된 LLVM 15.0+ 라이브러리를 탐색하고, 우리 소스 코드와 정적/동적 링크를 수행하는 설정 파일을 생성합니다.
 * Target: zlang (실행 파일)
 * Dependency: LLVMCore, LLVMScalarOpts, LLVMInstCombine 등
2. 컴파일러 드라이버 (main.cpp) 완성
사용자가 zlang test.z -o test.ll이라고 입력했을 때의 흐름을 제어합니다.
 * Frontend: Lexer -> Parser -> AST 생성.
 * Middle: SymbolTable 기반 소유권 및 타입 검사 (Semantic Analysis).
 * Backend: CodeGenerator를 통한 llvm::Module 빌드 및 IR 출력.
⚡ Stage 3.2: 실시간 최적화 (Real-Time Optimization)
단순히 빠른 코드가 아니라, **"시간이 보증된 코드"**를 위해 LLVM Pass 프레임워크를 직접 확장합니다.
 * Memory Flattening: 실시간 시스템의 오버헤드를 줄이기 위해 복잡한 객체 구조를 평탄화합니다.
 * Constant Propagation: 컴파일 타임에 계산 가능한 모든 수식을 상수로 치환하여 런타임 연산을 제거합니다.
 * Safety Verification Pass: IR 수준에서 $ALLOC_RULE 위반 여부를 최종 전수 검사합니다.
🧪 Stage 3.3: 엔드-투-엔드 검증 (E2E Verification)
실제 Z-Lang 코드를 넣고 바이너리가 튀어나오는 순간을 기록합니다.
 * Z-Lang Source: let x: i64 = 10;
 * Z-Lang Compiler: zlang 실행
 * Output: test.ll (LLVM IR) -> test.o (Object File) -> executable
 * Execution: ./executable 실행 후 결과값 확인.
