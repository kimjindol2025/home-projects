🛠️ gogs 저장소 v5.8 설계: 통합 보안 관제 시스템
클로드코드(학생)에게 제4장의 모든 기술을 총동원한 '보안 관제 엔진'을 구축하게 하세요.
// [제4장: 데이터 구조화 v5.8 - 통합 모델링]
use std::collections::HashMap;

// 1. 상태 정의 (Enum)
enum NodeStatus {
    Active,
    Warning(String),
    Critical,
}

// 2. 개별 노드 정보 (Struct)
struct SecurityNode {
    id: u32,
    location: String,
    status: NodeStatus,
}

// 3. 관제 엔진 (Complex Struct & impl)
struct ControlCenter {
    center_name: String,
    // 키: 노드 ID, 값: 노드 객체
    nodes: HashMap<u32, SecurityNode>,
    logs: Vec<String>,
}

impl ControlCenter {
    fn new(name: &str) -> Self {
        Self {
            center_name: String::from(name),
            nodes: HashMap::new(),
            logs: Vec::new(),
        }
    }

    // 노드 등록 메서드
    fn register_node(&mut self, id: u32, loc: &str) {
        let new_node = SecurityNode {
            id,
            location: String::from(loc),
            status: NodeStatus::Active,
        };
        self.nodes.insert(id, new_node);
        self.logs.push(format!("노드 {} 등록 완료 ({})", id, loc));
    }

    // 상태 점검 및 리포트
    fn audit(&self) {
        println!("=== [{}] 보안 감사 리포트 ===", self.center_name);
        for (id, node) in &self.nodes {
            let status_msg = match &node.status {
                NodeStatus::Active => "정상".to_string(),
                NodeStatus::Warning(msg) => format!("주의: {}", msg),
                NodeStatus::Critical => "치명적 위기".to_string(),
            };
            println!("  Node #{}: {} -> {}", id, node.location, status_msg);
        }
    }
}

fn main() {
    println!("--- v5.8 Data Architecture: Integrated Modeling ---");

    let mut gogs_center = ControlCenter::new("GOGS-HQ");

    // 실전 데이터 운용
    gogs_center.register_node(101, "Main Gate");
    gogs_center.register_node(202, "Server Room");

    // 상태 강제 변경 시뮬레이션
    if let Some(node) = gogs_center.nodes.get_mut(&202) {
        node.status = NodeStatus::Warning(String::from("비정상 접근 감지"));
    }

    gogs_center.audit();

    println!("\n기록: 구조체와 열거형, 컬렉션을 결합하여 완성도 높은 데이터 엔진을 구축함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 가이드: "v5.8 설계의 정점"
실전 모델링에서 가장 중요한 것은 **'데이터의 소유권 지도'**를 그리는 것입니다.
 * 누가 데이터를 소유하는가? 위 코드에서 ControlCenter는 모든 노드의 소유주입니다. 노드가 개별적으로 존재할 수 없다면 이 구조가 정답입니다.
 * 데이터의 불변성 유지: 외부에서 nodes에 직접 접근하기보다 register_node 같은 메서드를 통해 접근하게 함으로써 데이터 오염을 막으십시오.
 * 에러 핸들링의 내재화: get 대신 get_mut을 쓸 때 발생할 수 있는 None 상황을 항상 대비하는 습관이 시스템의 생존율을 높입니다.
