대학 과정에서 웹 서버를 만든다는 것은 단순히 페이지를 띄우는 것이 아니라, 요청(Request)과 응답(Response)의 흐름, 그리고 효율적인 핸들러 구조를 설계하는 것을 의미합니다.
🎓 Go-언어 전공 101: 1-11. 표준 라이브러리를 활용한 웹 서버
Go는 별도의 프레임워크 없이 표준 라이브러리인 net/http만으로도 강력한 성능의 서버를 만들 수 있습니다.
1. 기본 서버 구조
가장 먼저 'Hello World' 서버를 구축하며 구조를 파악합니다.
package main

import (
    "fmt"
    "net/http"
)

func main() {
    // 1. 핸들러 등록 (URL 경로와 실행될 함수 연결)
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprint(w, "대학원 Go 서버에 오신 것을 환영합니다.")
    })

    fmt.Println("서버가 8080 포트에서 시작되었습니다...")
    
    // 2. 서버 실행 및 리스닝
    http.ListenAndServe(":8080", nil)
}

2. 구조체와 메서드를 활용한 핸들러 (심화)
앞서 배운 Struct와 Method를 활용하면 서버 상태(데이터베이스 연결 등)를 유지하는 고도의 핸들러를 설계할 수 있습니다.
type apiHandler struct {
    dbName string
}

// 인터페이스 조건을 충족하는 메서드 설계
func (h *apiHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "[%s] 데이터베이스를 사용하는 API 응답입니다.", h.dbName)
}

func main() {
    handler := &apiHandler{dbName: "PostgreSQL"}
    http.Handle("/api", handler)
    http.ListenAndServe(":8080", nil)
}

🔬 전공 심화: Go 서버가 빠른 이유 (Concurrency inside)
웹 서버의 ListenAndServe 함수 내부를 들여다보면 왜 Go가 백엔드에서 강력한지 알 수 있습니다.
 * 자동 고루틴 할당: Go의 표준 서버는 접속하는 모든 요청(Request)마다 자동으로 새로운 고루틴을 생성하여 처리합니다.
 * 비차단 I/O: 수천 명의 사용자가 동시에 접속해도, 각 요청이 고루틴 단위로 분리되어 실행되므로 한 명의 작업이 늦어진다고 해서 전체 서버가 멈추지 않습니다.
 * 효율적 자원 관리: OS 쓰레드를 직접 생성하지 않기 때문에 메모리 오버헤드가 매우 적습니다.
