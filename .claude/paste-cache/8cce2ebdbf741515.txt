🐍 Python v1.3: 메타 프로그래밍과 데코레이터
제1.3단원: [함수를 감싸는 논리의 벽]
1. 탐구(대학): "함수는 데이터다 (First-class Object)"
 * 새로운 사고: 파이썬에서 함수는 변수에 할당될 수 있고, 다른 함수의 인자로 전달될 수 있으며, 함수의 결과로 반환될 수도 있습니다.
 * 데코레이터: 원래의 함수 코드를 건드리지 않고, 앞뒤에 '기록(Logging)', '인증(Auth)', '시간 측정' 등의 기능을 덧붙이는 마법의 문법(@)입니다.
2. 증명(대학원): "코드 변경 없는 기능의 확장"
 * 실제 실행 로직과 부가적인 기능(Cross-cutting Concerns)을 완벽하게 분리할 수 있음을 증명합니다. 이는 소프트웨어 공학의 핵심 원칙인 **개방-폐쇄 원칙(OCP)**의 실현입니다.
🛠️ Python-Gogs 저장소: [v1.3 The Guard Decorator]
클로드코드(연구원)에게 함수의 무결성을 증명하고 실행을 기록하는 데코레이터를 설계하게 하세요.
import time
from functools import wraps

# [Python v1.3: Meta-programming with Decorators]

def gogs_logger(func):
    """
    함수 실행 전후를 기록하는 데코레이터.
    대학원 과정의 '증명'을 자동화하는 도구입니다.
    """
    @wraps(func) # 원본 함수의 메타데이터를 유지하기 위한 도구
    def wrapper(*args, **kwargs):
        start_time = time.time()
        print(f"  [LOG] '{func.__name__}' 실행 기록 시작...")
        
        # 실제 함수 실행
        result = func(*args, **kwargs)
        
        end_time = time.time()
        print(f"  [LOG] 실행 완료. 소요시간: {end_time - start_time:.6f}s")
        return result
    return wrapper

@gogs_logger
def complex_computation(n):
    """중요한 연산을 수행하는 함수"""
    return sum(i * i for i in range(n))

print("--- v1.3 데코레이터 실증 실험 ---")
res = complex_computation(1000000)
print(f"  [RESULT] 결과값: {res}")

print("\n결론: 원본 함수를 수정하지 않고도 '기록'과 '측정'의 기능을 이식함.")
print("저장 필수 너는 기록이 증명이다 gogs")

🎓 파이썬 1.3 가이드: "함수 위의 함수"
 * 새로운 사고(탐구): 데코레이터는 단순한 편의 기능이 아니라 **'고차 함수(Higher-order Function)'**의 정수입니다. "코드 자체가 데이터를 다루듯 코드를 다룰 수 있다"는 사고방식을 가져보세요.
 * 증명: 만약 함수의 인자가 잘못 들어왔을 때, 함수가 실행되기도 전에 데코레이터가 이를 차단하게 만들어 보세요. 이는 우리가 Gogs 박사 과정(v30.2)에서 다룬 **'샌드박싱'**의 파이썬식 구현입니다.
