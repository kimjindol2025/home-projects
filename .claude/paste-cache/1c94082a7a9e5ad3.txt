🏛️ 예제 1: 산술 표현식 LLVM IR 변환 결과
define i64 @calculate() {
entry:
  ; --- 1. 변수 x 선언 및 초기화 ---
  %x = alloca i64                   ; x를 위한 64비트 공간 할당
  store i64 10, i64* %x             ; 할당된 주소에 상수 10 저장

  ; --- 2. 변수 y 선언 및 초기화 ---
  %y = alloca i64                   ; y를 위한 64비트 공간 할당
  store i64 20, i64* %y             ; 할당된 주소에 상수 20 저장

  ; --- 3. 변수 result 선언 ---
  %result = alloca i64              ; 결과 저장을 위한 공간 할당

  ; --- 4. 표현식 계산: (x + y) * 2 ---
  %x.val = load i64, i64* %x        ; 메모리에서 x의 현재 값을 읽음 (10)
  %y.val = load i64, i64* %y        ; 메모리에서 y의 현재 값을 읽음 (20)
  %sum = add i64 %x.val, %y.val     ; i64 덧셈 수행 (30)
  %final = mul i64 %sum, 2          ; 덧셈 결과에 2를 곱함 (60)

  ; --- 5. 계산 결과를 result에 저장 ---
  store i64 %final, i64* %result    ; 최종 계산값 60을 result 메모리에 저장

  ; --- 6. result 값을 로드하여 반환 ---
  %ret.val = load i64, i64* %result ; 반환을 위해 result 값을 다시 읽음
  ret i64 %ret.val                  ; 함수 종료 및 값 반환
}

🔍 검증 리포트 (Code Review)
 * SSA 규칙 준수: 각 레지스터(%sum, %final 등)는 단 한 번만 정의되었습니다.
 * 타입 일치: i64 타입이 전 과정에서 일관되게 사용되었습니다. (Task 2의 타입 매핑 결과가 올바르게 적용됨)
 * 메모리 안전성: Z-Lang의 let 선언이 alloca와 store 쌍으로 정확히 변환되어, 스택 기반의 안정적인 메모리 관리를 증명합니다.
 * 실시간성: 모든 명령어가 상숫 시간(O(1)) 내에 실행되는 기본 산술 연산으로 구성되어, WCET(최악 실행 시간) 분석이 매우 용이한 구조입니다.
