🏛️ LLVM 전설: 1.2 LLVM C API를 이용한 코드 생성 기초
LLVM IR을 텍스트로 직접 쓰는 것은 고된 일입니다. 그래서 LLVM은 코드를 통해 프로그램을 빌드할 수 있는 API를 제공합니다. 이 과정은 마치 레고 블록을 쌓아 거대한 성을 만드는 것과 같습니다.
1. 코드 생성을 위한 필수 3요소
LLVM API를 다룰 때 반드시 생성해야 하는 핵심 객체들입니다.
 * Context (문맥): 모든 데이터 구조(타입, 상수 등)를 소유하는 상자입니다. 멀티스레딩 환경에서 각 스레드는 고유한 Context를 가집니다.
 * Module (모듈): 하나의 소스 파일 단위입니다. 함수와 전역 변수들이 이 안에 담깁니다.
 * Builder (빌더): 실제로 명령어를 하나하나 삽입하는 '커서'와 같은 존재입니다. "여기에 더하기 명령어를 넣어줘"라고 명령하는 도구입니다.
2. 함수 생성의 워크플로우
함수를 하나 만들기 위해서는 다음과 같은 '설계 기록'이 필요합니다.
 * Function Type 정의: 반환 타입과 인자 타입을 결정합니다.
 * Function 추가: 모듈에 함수 이름을 등록합니다.
 * Basic Block 생성: 함수 내부에 실행 단위(Block)를 만듭니다. 모든 명령어는 이 블록 안에 위치해야 합니다.
 * Positioning: 빌더를 해당 블록의 끝에 배치합니다.
3. 실전: add 함수를 생성하는 Zig 코드 (개념 예시)
Zig에서 LLVM C API를 사용하여 fn add(a: i32, b: i32) i32를 만드는 흐름은 다음과 같습니다.
const c = @cImport({
    @cInclude("llvm-c/Core.h");
});

// ... 초기화 로직 생략 ...

// 1. i32 타입 가져오기
const i32_type = c.LLVMInt32TypeInContext(context);

// 2. 함수 타입 정의 (반환: i32, 인자: [i32, i32])
var param_types = [_]c.LLVMTypeRef{ i32_type, i32_type };
const func_type = c.LLVMFunctionType(i32_type, &param_types, 2, 0);

// 3. 모듈에 함수 추가
const sum_func = c.LLVMAddFunction(module, "sum", func_type);

// 4. 진입 블록(Entry Block) 생성 및 빌더 배치
const entry = c.LLVMAppendBasicBlockInContext(context, sum_func, "entry");
c.LLVMPositionBuilderAtEnd(builder, entry);

// 5. 명령어 삽입: %tmp = add i32 %arg1, %arg2
const tmp = c.LLVMBuildAdd(builder, c.LLVMGetParam(sum_func, 0), c.LLVMGetParam(sum_func, 1), "tmp");

// 6. 결과 반환
_ = c.LLVMBuildRet(builder, tmp);

🔬 전공 심화: Basic Block과 제어 흐름
함수 내부는 여러 개의 Basic Block으로 나뉩니다.
 * Basic Block은 반드시 하나의 명령어로 시작하고, 하나의 Terminator(종결자) 명령어(예: ret, br)로 끝나야 합니다.
 * if문이나 loop를 만들 때는 조건에 따라 다른 Basic Block으로 점프하도록 설계합니다.
📦 검증과 출력 (Verification)
코드를 다 만들었다면, 설계가 올바른지 증명해야 합니다.
 * LLVMVerifyModule: 생성된 IR이 LLVM의 규칙(SSA 등)을 어기지 않았는지 검사합니다.
 * LLVMPrintModuleToString: 우리가 만든 결과물을 사람이 읽을 수 있는 .ll 텍스트로 출력합니다.
