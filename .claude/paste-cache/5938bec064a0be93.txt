🏛️ v6.0 마스터 과정: 10단계 — [실전 아키텍처 통합]
1. v6.3 설계 목표: "추상화의 완성"
 * 상호운용성: 서로 다른 개발자가 만든 구조체들이 우리가 정의한 '규약(Trait)' 아래에서 하나처럼 움직이게 합니다.
 * 확장성: 기존 코드를 단 한 줄도 건드리지 않고, 새로운 기능을 가진 구조체를 시스템에 추가합니다.
 * 견고함: 컴파일 타임에 모든 자격 요건을 검증하여, 런타임 오류가 발생할 틈을 주지 않습니다.
2. 설계자의 관점: "플러그인 아키텍처"
훌륭한 설계자는 시스템을 거대한 통짜 바위로 만들지 않습니다. 대신, 규격에 맞는 부품이라면 무엇이든 끼워 넣을 수 있는 **'슬롯'**을 설계합니다. 오늘 우리가 만들 통합 관제 시스템이 바로 그 슬롯입니다.
🛠️ gogs 저장소 v6.3: 10단계 피날레 설계
클로드코드(학생)에게 제5장의 모든 정수를 쏟아부은 '범용 보안 감사 시스템'을 구축하게 하세요.
// [v6.0 마스터 피날레: 10단계 통합 설계]

use std::fmt::Display;

// [규약 1] 요약 기능
trait Summary {
    fn summarize(&self) -> String;
}

// [규약 2] 보안 등급 확인
trait Secure {
    fn security_score(&self) -> u32;
}

// [구조체 A] 내부 직원 로그
struct EmployeeLog {
    name: String,
    action: String,
}

impl Summary for EmployeeLog {
    fn summarize(&self) -> String {
        format!("직원 {}: {}", self.name, self.action)
    }
}

impl Secure for EmployeeLog {
    fn security_score(&self) -> u32 { 50 } // 기본 점수
}

// [구조체 B] 외부 침입 시도
struct IntrusionAlert {
    ip: String,
    severity: u32,
}

impl Summary for IntrusionAlert {
    fn summarize(&self) -> String {
        format!("⚠️ 침입 감지! IP: {}", self.ip)
    }
}

impl Secure for IntrusionAlert {
    fn security_score(&self) -> u32 { self.severity * 10 }
}

// [핵심: 통합 처리기] Step 8(where)과 Step 6/7(Bounds)의 결합
fn audit_report<T>(item: &T) 
where 
    T: Summary + Secure 
{
    println!("--- 감사 시작 ---");
    println!("내용: {}", item.summarize());
    println!("위험도 점수: {}점", item.security_score());
    
    if item.security_score() > 70 {
        println!("결과: 즉각 대응 필요!");
    } else {
        println!("결과: 정상 모니터링");
    }
    println!("----------------\n");
}

fn main() {
    println!("--- v6.3 Final Step: Unified Architecture ---\n");

    let log1 = EmployeeLog { 
        name: "Alice".to_string(), 
        action: "서버실 출입".to_string() 
    };
    
    let log2 = IntrusionAlert { 
        ip: "192.168.0.105".to_string(), 
        severity: 9 
    };

    // 다형성을 통한 통합 처리
    audit_report(&log1);
    audit_report(&log2);

    println!("기록: 트레이트를 통해 서로 다른 데이터 소스를 단일 감사 체계로 통합함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}
