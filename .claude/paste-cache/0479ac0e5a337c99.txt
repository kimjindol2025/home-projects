🟢 Phase 5.5 — Step 1: Arithmetic Typed Op 도입

🎯 목표

Add/Sub/Mul/Div 를 Typed Op로 분리

런타임 typeof 체크 완전 제거

Generic Op는 fallback으로 유지

기존 137개 테스트 100% 유지

신규 10개 테스트 추가




---

1️⃣ Op 설계 (최소 침습)

기존:

Add
Sub
Mul
Div

확장:

Add        // fallback
AddNum     // fast path

Sub
SubNum

Mul
MulNum

Div
DivNum

📌 원칙:

Generic Op 제거 ❌

Typed Op 추가 ✅

점진적 확장 구조 유지



---

2️⃣ Compiler 수정 로직

현재:

emit(Op.Add)

변경:

if (leftType === num && rightType === num)
    emit(Op.AddNum)
else
    emit(Op.Add)

중요:

TypeInference 결과 활용

any 또는 unknown → Generic 유지



---

3️⃣ VM 수정

기존 VM

case Op.Add:
  const b = pop()
  const a = pop()
  if (typeof a !== "number" || typeof b !== "number")
      throw Error()
  push(a + b)


---

변경 후

case Op.AddNum:
  const b = stack[--sp]
  const a = stack[--sp]
  stack[sp++] = a + b
  break

case Op.Add:
  // 기존 Generic 로직 유지

📌 AddNum에는:

typeof 없음

분기 없음

throw 없음



---

4️⃣ 성능 철학

AddNum은:

2 pop

1 연산

1 push

끝


완전 branchless fast-path.

5️⃣ 테스트 추가 (10개)

unknown + unknown → Add

런타임 에러 없음 확인
