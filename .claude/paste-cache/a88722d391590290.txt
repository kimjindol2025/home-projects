이제 우리는 고수준 언어의 안락함을 벗어나, Go 컴파일러가 생성하는 기계어 레벨에서 성능을 쥐어짜는 법을 배웁니다. Go는 독특하게도 'Plan 9'이라는 고유의 어셈블리 형식을 사용하며, 이는 하드웨어와 Go 런타임 사이의 가교 역할을 합니다.
저장 필수 너는 기록이 증명이다 gogs. CPU의 심장 박동을 기록합니다.
🎓 Go-언어 전공 401: 4-2. Plan 9 어셈블리와 컴파일러 최적화
1. Go 어셈블리 (Plan 9)
Go는 플랫폼마다 다른 어셈블리(x86, ARM 등)를 직접 쓰지 않고, 이를 추상화한 Plan 9 어셈블리를 사용합니다.
 * 가상 레지스터: FP(Frame Pointer), PC(Program Counter), SB(Static Base), SP(Stack Pointer) 등 4개의 가상 레지스터를 이해하는 것이 핵심입니다.
 * 성능의 정점: 표준 라이브러리의 암호화 알고리즘(crypto/sha256 등)이나 수학 연산은 속도를 위해 이 어셈블리로 직접 작성되어 있습니다.
2. 인라이닝 (Inlining) 최적화
함수 호출은 비용입니다. (스택 프레임 생성, 레지스터 저장 등) 컴파일러는 작은 함수들을 호출부로 직접 복사해서 넣는 인라이닝을 수행합니다.
 * 최적화 기법: 함수가 너무 크면 인라이닝이 되지 않습니다. 박사 과정 엔지니어는 함수를 '인라이닝이 가능할 만큼' 작게 쪼개거나, 컴파일러 힌트를 분석하여 성능을 높입니다.
3. 고차원 최적화 도구 (SSA: Static Single Assignment)
Go 컴파일러는 코드를 기계어로 바꾸기 전, SSA라는 중간 단계에서 수많은 최적화(죽은 코드 제거, 상수 폴딩 등)를 거칩니다.
# 내 코드가 어떻게 최적화되는지 HTML로 확인하는 법
GOSSAFUNC=myFunctionName go build main.go

이 명령어를 실행하면 컴파일러가 각 단계별로 코드를 어떻게 변형하여 성능을 올렸는지 시각적으로 보여줍니다.
🔬 전공 심화: SIMD (Single Instruction, Multiple Data) 활용
CPU의 특수 명령어를 사용하여 한 번에 여러 개의 데이터를 처리하는 기술입니다.
 * Julia가 수치 계산에서 빠른 이유 중 하나도 이 SIMD를 잘 쓰기 때문입니다.
 * Go에서는 직접적인 SIMD 지원이 부족할 때, 어셈블리를 직접 작성하여 CPU의 성능을 100% 끌어냅니다.
🏗️ 박사 과정의 기술: Bounds Check Elimination (BCE)
Go는 배열 인덱스에 접근할 때마다 범위를 벗어나는지 검사합니다. 이는 안전하지만 느립니다.
func fastSum(s []int) int {
    _ = s[3] // 미리 인덱스 3을 체크하면, 아래 루프에서 0~3까지의 체크가 생략됩니다 (BCE)
    return s[0] + s[1] + s[2] + s[3]
}

컴파일러의 지능을 이용하여 불필요한 '안전 검사'를 제거함으로써 성능을 극대화하는 기법입니다.
📝 Assignment 4-2 (하드웨어 최적화 과제)
 * 어셈블리 분석: 간단한 덧셈 함수를 만들고 go tool compile -S 명령어를 사용하여 생성된 Plan 9 어셈블리 코드를 한 줄씩 해석해 보세요.
 * 인라인 확인: go build -gcflags="-m -m" 명령어를 사용하여 특정 함수가 왜 인라이닝 되었는지(또는 거부되었는지) 컴파일러의 판단 근거를 분석해 보세요.
