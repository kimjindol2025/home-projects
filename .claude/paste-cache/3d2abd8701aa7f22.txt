지금까지는 하나의 프로그램(Monolithic) 안에서 모든 것을 처리했습니다. 하지만 구글이나 넷플릭스 같은 거대 시스템은 수만 대의 서버가 각자의 역할을 수행하며 거대한 하나의 유기체처럼 동작합니다.
1. 핵심 개념: Monolithic vs Microservices
 * Monolithic (모놀리식): 모든 기능(로그인, 결제, 검색)이 하나의 거대한 몸통에 들어있는 구조. 하나가 고장 나면 전체가 멈춥니다.
 * Microservices (마이크로서비스): 각 기능을 독립된 작은 서비스로 쪼개어 네트워크(Socket)로 대화하게 만드는 구조. 유연하고 확장이 쉽습니다.
2. 분산 시스템의 난제: CAP 이론
분산 시스템을 설계할 때, 설계자님은 다음 세 가지 중 두 가지만 완벽하게 가질 수 있다는 법칙을 마주하게 됩니다.
 * Consistency (일관성): 모든 사용자가 같은 시간에 같은 데이터를 봐야 함.
 * Availability (가용성): 일부 서버가 죽어도 서비스는 계속 돌아가야 함.
 * Partition Tolerance (분할 내구성): 네트워크 연결이 끊겨도 시스템이 유지되어야 함.
🛠️ 전공 실무: [v5.1 분산 메시지 브로커 설계 기초]
서로 다른 프로세스나 서버들이 직접 대화(Socket)하는 것은 위험할 수 있습니다. 중간에 **'우체국(Message Broker)'**을 두어 데이터를 안전하게 전달하는 구조를 설계해 보겠습니다.
# [Python University v5.1] 분산 시스템을 위한 메시지 큐 시뮬레이션

import queue
import threading
import time

# 1. 메시지 브로커 (중앙 우체국)
class MessageBroker:
    def __init__(self):
        self.queue = queue.Queue()

    def publish(self, message):
        print(f"📬 [Broker] 메시지 접수: {message}")
        self.queue.put(message)

    def subscribe(self):
        return self.queue.get()

# 2. 마이크로서비스 A: 주문 서비스 (Producer)
def order_service(broker):
    for i in range(3):
        msg = f"주문번호-{i+100}"
        broker.publish(msg)
        time.sleep(1)

# 3. 마이크로서비스 B: 결제 서비스 (Consumer)
def payment_service(broker):
    while True:
        order = broker.subscribe()
        print(f"💳 [Payment] {order} 결제 처리 중...")
        time.sleep(2)
        print(f"✅ [Payment] {order} 결제 완료!")

# 메인 시스템 설계
if __name__ == "__main__":
    broker = MessageBroker()

    # 각 서비스를 독립된 스레드(혹은 프로세스)로 실행
    p1 = threading.Thread(target=order_service, args=(broker,), daemon=True)
    p2 = threading.Thread(target=payment_service, args=(broker,), daemon=True)

    p1.start()
    p2.start()

    p1.join()
    print("\n저장 필수 너는 기록이 증명이다 gogs")
    print("🚀 모든 주문이 브로커에 전달되었습니다.")

🎓 대학교 3학년 전공 심화 팁
 * Loose Coupling (느슨한 결합): 서비스들이 서로의 내부 사정을 모른 채 메시지만 주고받게 설계하는 것이 핵심입니다. 그래야 하나를 수정해도 전체가 무너지지 않습니다.
 * Scalability (확장성): 결제 서비스가 느려지면? 결제 서버만 10대로 늘리면 됩니다. 이것이 3학년 과정에서 배울 클라우드 컴퓨팅의 핵심 원리입니다.
