드디어 **'언어 사용자'**의 영역을 졸업하고, **'언어 제작자'**의 영역으로 진입하는 역사적인 순간입니다. **[제11장: 언세이프와 메모리 주권 — v12.0 컴파일러를 위한 로우 레벨 설계]**를 시작합니다.
컴파일러는 추상화된 코드를 실제 메모리의 바이트와 주소로 번역하는 도구입니다. 이를 설계하기 위해서는 러스트가 우리를 보호하기 위해 쳐두었던 '안전 장치'를 잠시 걷어내고, 그 내부의 날것(Raw)을 다루는 법을 배워야 합니다.
🏗️ 제11장: 언세이프와 메모리 (v12.0) — [메모리 주권 선언]
1. v12.0 설계 목표: "추상화의 장벽 허물기"
 * Unsafe Rust: 컴파일러가 안전성을 보장할 수 없는 영역임을 명시하고, 설계자가 직접 모든 책임을 지고 메모리를 제어합니다.
 * 로우 포인터(Raw Pointers): *const T와 *mut T. 수명(Lifetime)이나 소유권 규칙을 무시하고 메모리 주소에 직접 접근합니다.
 * FFI (Foreign Function Interface): C 언어 등 다른 언어로 작성된 라이브러리와 통신하기 위한 필수 관문을 설계합니다.
2. 설계자의 관점: "절대적 자유와 절대적 책임"
컴파일러 제작자는 메모리를 '거대한 바이트의 바다'로 봅니다. 설계자는 특정 주소에 데이터를 쓰고, 그 주소를 다시 해석(Cast)하여 의미를 부여합니다. 이 과정에서 발생하는 **세그멘테이션 폴트(Segfault)**나 **정의되지 않은 동작(UB)**을 막는 것이 이제부터 여러분의 몫입니다.
🛠️ gogs 저장소 v12.0 설계: 가상 메모리 관리자 (Memory Manager)
클로드코드(학생)에게 안전한 참조자를 로우 포인터로 분해하고, 다시 조립하는 '컴파일러급 메모리 조작'을 기록하게 하세요.
// [제11장: 언세이프와 메모리 v12.0 - 컴파일러 기초 설계]

fn main() {
    println!("--- v12.0 The Core: Breaking the Safety Barrier ---");

    let mut data = 100_u32;
    
    // 1. [설계] 안전한 참조자를 로우 포인터로 변환
    // 로우 포인터는 생성하는 것 자체는 안전하지만, '역참조'는 위험합니다.
    let address_ptr: *mut u32 = &mut data as *mut u32;

    println!("  [INFO] 데이터의 실제 메모리 주소: {:p}", address_ptr);

    // 2. [위험 구역] 컴파일러는 여기서 메모리를 직접 수정합니다.
    unsafe {
        // 로우 포인터를 통해 값을 직접 변경
        *address_ptr = 200;
        
        // 유효하지 않은 주소 접근 시 시스템이 즉시 충돌하므로 극도의 주의가 필요함
        println!("  [UNSAFE] 주소 {:p}의 값을 200으로 강제 변경함.", address_ptr);
    }

    // 3. [검증] 안전 영역으로 돌아와 결과 확인
    println!("  [RESULT] 최종 데이터 값: {}", data);

    println!("\n기록: 로우 포인터를 활용하여 컴파일러 수준의 메모리 직접 제어권을 확보함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 세부 가이드 (12.0): "언세이프를 대하는 자세"
컴파일러 제작자가 unsafe를 사용하는 이유는 실력을 뽐내기 위함이 아닙니다.
 * 필요악의 수용: 컴퓨터 하드웨어나 OS 커널, 혹은 다른 언어와 대화할 때 '안전한 러스트'만으로는 한계가 있습니다. 그때 unsafe는 강력한 탈출구가 됩니다.
 * 격리(Encapsulation): 유능한 설계자는 unsafe 코드를 아주 작은 함수 내부에 가두고, 외부에는 '안전한 인터페이스'만 노출합니다. 이것이 러스트 표준 라이브러리가 만들어진 방식입니다.
 * UB(Undefined Behavior) 경계: 초기화되지 않은 메모리를 읽거나, 이미 해제된 메모리에 접근하는 행위는 컴파일러 설계에서 가장 경계해야 할 죄악입니다.
