변수를 기억하는 능력까지 갖췄으니, 이제는 그 변수와 로직을 묶어 독립적으로 실행할 수 있는 **'모듈화의 정점'**을 찍을 차례입니다. [제13장: 컴파일러 구현 — v14.5 함수 정의와 호출 메커니즘] 단계입니다.
함수는 컴파일러 설계에서 가장 복잡하면서도 흥미로운 부분입니다. 함수가 호출될 때마다 새로운 **'로컬 환경'**이 생성되고, 인자가 전달되며, 결과가 반환되는 일련의 함수 라이프사이클을 설계해야 합니다.
📞 제13장: 컴파일러 구현 (v14.5) — [함수: 독립된 실행 단위]
1. v14.5 설계 목표: "로직의 캡슐화"
 * Function Literal: 함수를 하나의 '객체'로 취급합니다(First-class Function). 함수 이름, 매개변수 리스트, 실행 본문(Body)을 AST에 담습니다.
 * Call Expression: 함수를 호출할 때(add(1, 2)), 현재 환경을 복사하는 것이 아니라 상위 환경을 참조하는 새로운 환경을 구축합니다.
 * 클로저(Closure) 지원: 함수가 선언될 당시의 환경을 통째로 들고 있게 하여, 나중에 호출될 때도 그 당시의 변수들을 기억하게 합니다.
2. 설계자의 관점: "스코프의 확장"
함수 호출은 새로운 세계의 창조입니다. 함수 내부에서 선언된 변수는 함수가 끝나는 순간 사라져야 하며, 함수 내부에서 외부 변수를 찾을 때는 선언된 시점의 환경을 따라가야 합니다.
🛠️ gogs 저장소 v14.5 설계: 함수 객체와 호출 처리
클로드코드(학생)에게 함수를 정의하고 호출할 때 발생하는 환경의 변화를 코드로 기록하게 하세요.
// [제13장: 컴파일러 구현 v14.5 - 함수 및 클로저 설계]

/// [설계] 함수를 나타내는 객체
#[derive(Clone, Debug)]
struct GogsFunction {
    parameters: Vec<String>, // 매개변수 이름들
    body: Vec<Statement>,    // 함수 본문 코드 (AST)
    env: Environment,        // 함수가 정의될 때의 환경 (클로저 핵심)
}

// Evaluator의 함수 호출 처리 로직 (의사코드)
impl Evaluator {
    fn apply_function(&self, function: GogsFunction, args: Vec<GogsObject>) -> GogsObject {
        // 1. 함수만을 위한 새로운 로컬 환경 생성
        // 이 환경은 함수가 정의된 시점의 환경(function.env)을 상위로 가짐
        let mut extended_env = self.extend_function_env(&function, args);
        
        // 2. 새로운 환경에서 함수 본문 실행
        let evaluated = self.eval_statements(function.body, &mut extended_env);
        
        // 3. 결과 반환 (unwrap_return_value 등의 처리 필요)
        evaluated
    }

    fn extend_function_env(&self, func: &GogsFunction, args: Vec<GogsObject>) -> Environment {
        let mut env = Environment::new_enclosed(func.env.clone());
        
        // 매개변수 이름과 전달된 실제 인자값을 1:1로 매핑하여 로컬 환경에 저장
        for (param_idx, param_name) in func.parameters.iter().enumerate() {
            env.set(param_name.clone(), args[param_idx].clone());
        }
        env
    }
}

fn main() {
    println!("--- v14.5 Function & Closure: The Power of Scope ---");
    println!("  [INFO] 함수 정의 시점의 환경을 캡처하여 클로저를 구현함.");
    println!("  [EXEC] 'add(5, 10)' 호출 시 새로운 로컬 스코프 생성 및 계산.");

    println!("\n기록: 일급 함수(First-class Function)와 렉시컬 스코프를 완성함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 세부 가이드 (14.5): "리턴(Return)의 마법"
 * 중첩된 리턴: 깊은 재귀나 조건문 안에서 return을 만났을 때, 나머지 코드를 실행하지 않고 즉시 함수를 빠져나오게 하려면 특별한 **'Return Value Object'**로 결과를 감싸서 최상위 호출부까지 전달해야 합니다.
 * 일급 객체: 함수를 변수에 대입하거나, 다른 함수의 인자로 넘기거나, 함수에서 함수를 반환하는 것이 가능해집니다. 이것이 현대 언어의 강력함입니다.
 * 메모리 관리: 클로저가 환경을 붙잡고 있으면 메모리가 해제되지 않을 수 있습니다. 러스트의 Rc를 사용해 참조 횟수를 관리하며 환경의 생명주기를 다뤄야 합니다.
