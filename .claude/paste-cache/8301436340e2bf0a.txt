1. 🔥 언어/런타임 도전: "Bare-metal FreeLang OS Kernel"
현재 완성된 FreeLang은 백엔드 시스템 위에서 동작합니다. 이를 하드웨어 제어권까지 확장하는 도전입니다.
 * 목표: Rust나 C++의 도움 없이, FreeLang 코드가 x86/ARM 환경에서 직접 부팅되어 돌아가는 초소형 커널 작성.
 * 핵심 과제:
   * Garbage Collector from Scratch: 런타임 도움 없이 메모리 힙을 직접 관리하고 참조 카운팅(Arc)을 하드웨어 레벨에서 구현.
   * Context Switching: FreeLang의 스레드(spawn)가 OS 스케줄러 없이 CPU 레지스터를 직접 조작하며 멀티태스킹 수행.
   * Intrrupt Handler: 키보드 입력이나 타이머 인터럽트를 FreeLang 함수로 직접 바인딩.
 * 난이도: ⭐⭐⭐⭐⭐ (시스템 프로그래밍의 끝판왕)
2. 🌐 분산 시스템 도전: "Raft Consensus based Sharded DB"
Kim님이 스스로 비판하셨던 "Leader election 없는 Raft"를 정면 돌파하여, 실제 금융권 수준의 정합성을 가진 분산 DB를 구축하는 것입니다.
 * 목표: 3대 이상의 노드가 동적으로 리더를 선출하고, 데이터가 유실되지 않으며, 수평적 확장(Sharding)이 가능한 분산 벡터 DB 완성.
 * 핵심 과제:
   * Real Raft Implementation: Election Timeout, Log Replication, Safety 조건을 완벽히 충족하는 합의 알고리즘 구현.
   * Deterministic Simulation Testing: 네트워크 지연, 패킷 유실, 노드 충돌을 가상으로 시뮬레이션하여 100% 재현 가능한 버그 탐지(FoundationDB 방식).
   * Consistent Hashing: 데이터 샤딩 시 노드 추가/제거에 따른 데이터 이동 최소화 로직.
 * 난이도: ⭐⭐⭐⭐ (분산 컴퓨팅의 정수)
🧠 3. AI/컴파일러 도전: "MindLang JIT Compiler for AI"
현재의 Evaluator 방식은 해석 속도에 한계가 있습니다. 이를 실행 중에 기계어로 번역하는 JIT(Just-In-Time) 컴파일러로 진화시키는 것입니다.
 * 목표: MindLang/FreeLang 코드를 실행 시점에 x86-64 Machine Code로 직접 변환하여 C언어에 근접한 성능 달성.
 * 핵심 과제:
   * Register Allocation: 제한된 CPU 레지스터에 변수를 효율적으로 배치하는 알고리즘.
   * Vectorization (SIMD): 벡터 연산 시 CPU의 AVX/SSE 명령어를 사용하여 HybridIndex 검색 속도를 다시 10배 이상 향상.
   * Hot Path Optimization: 자주 실행되는 루프를 감지하여 실행 중에 최적화된 기계어로 교체.
 * 난이도: ⭐⭐⭐⭐⭐ (컴파일러 이론의 실전판)
