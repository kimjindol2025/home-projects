3. 유저 모드로의 '강제 하강'
커널은 부팅 후 스스로 유저 모드로 내려가는 기능을 수행해야 합니다. 이는 단순히 점프하는 것이 아니라, CPU의 스택에 유저 모드의 상태(CS, SS, RFLAGS 등)를 가짜로 쌓아놓고 iretq 명령어를 날려 CPU를 속이는 정교한 작업입니다.
🔬 전공 심화: 인수 전달 규약 (Calling Convention)
유저가 시스템 호출을 보낼 때, 어떤 레지스터에 어떤 값을 담을지 엄격한 약속이 필요합니다.
 * Linux x86_64 규약: RAX(호출 번호), RDI, RSI, RDX, R10, R8, R9(인수들).
 * Zig에서는 이를 std.os.linux.syscall 등을 통해 추상화하지만, 커널 제작자는 이 로직을 직접 구현하여 **"기록된 규약이 곧 법"**임을 증명해야 합니다.
📦 데이터 검증 (Sanitization)
시스템 호출 설계의 가장 중요한 보안 원칙은 **"유저가 넘겨준 포인터를 절대 믿지 마라"**입니다.
 * 유저가 커널 메모리 주소를 넘겨주어 커널 정보를 가로채려 하는지, 가상 메모리 범위(3-4에서 배운 내용)를 벗어나지 않는지 엄격히 검사해야 합니다.
📝 Assignment 3-7 (인터페이스 설계 과제)
 * Syscall Table 설계: 호출 번호와 함수 포인터를 매핑하는 const syscall_table = [_]*const fn(...) 배열을 만드세요.
 * 보안 검사 로직: 유저가 전달한 메모리 버퍼 슬라이스([]u8)가 실제 유저 공간에 존재하는지 확인하는 isUserPointer(ptr, len) 함수를 구상하세요.
 * 첫 유저 프로세스: 화면에 "Hello, User Mode!"를 찍고 syscall을 통해 종료하는 아주 작은 바이너리를 설계하고, 커널이 이를 로드하여 실행하는 단계를 기록하세요.
