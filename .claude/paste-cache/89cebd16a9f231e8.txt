드디어 올 것이 왔습니다. 1학년 과정의 가장 큰 고비이자, 러스트를 다른 모든 언어와 차별화시키는 독보적인 개념, **[3단계: 소유권(Ownership)]**에 진입합니다.
이 개념을 마스터하면 메모리 관리의 '신'이 됩니다. 가비지 컬렉터(Java, Python 등) 없이도 메모리 누수나 오류가 없는 언어를 설계할 수 있게 됩니다.
💎 3단계: 소유권 (Ownership) - 메모리의 지배자
러스트의 소유권은 딱 세 가지 규칙만 기억하면 됩니다.
 * 각각의 값은 '소유자(Owner)'라고 불리는 변수를 가지고 있다.
 * 한 번에 딱 한 명의 소유자만 존재할 수 있다.
 * 소유자가 범위를 벗어나면(Out of Scope), 값은 즉시 메모리에서 삭제된다.
3.1 이동(Move) vs 복사(Copy)
기본적인 숫자 타입은 복사되지만, 문자열 같은 복잡한 데이터는 **'권력의 이동'**이 일어납니다.
3.2 빌림(Borrowing) & 참조(Reference)
소유권을 완전히 넘기지 않고, 잠시 '빌려주는' 기술입니다. & 기호를 사용합니다.
🛠️ Step 3.1: gogs 프로젝트 실전 코드 (소유권 체화)
클로드코드(학생)에게 이 코드를 src/main.rs에 적용하고 분석하게 하세요. 소유권의 원리를 깨닫지 못하면 러스트는 컴파일조차 되지 않습니다.
fn main() {
    println!("--- Gogs Project Level 3: Ownership ---");

    // [3.1.1] 소유권의 이동 (Move)
    let s1 = String::from("Gogs Data");
    let s2 = s1; // s1의 권력이 s2로 넘어갔습니다.

    // println!("{}", s1); // <- 여기서 에러 발생! s1은 더 이상 권한이 없음.
    println!("소유자 s2: {}", s2);

    // [3.2.1] 빌려주기 (Borrowing)
    let len = calculate_length(&s2); // &를 사용해 잠시 빌려줌
    println!("'{}'의 길이는 {}입니다.", s2, len); // s2는 여전히 사용 가능

    println!("저장 필수 너는 기록이 증명이다 gogs");
}

fn calculate_length(s: &String) -> usize {
    s.len()
} // 여기서 s는 범위를 벗어나지만, '빌린' 것이므로 원본은 삭제되지 않음

🎓 선생님의 3단계 핵심 강의: "메모리 안전의 철학"
나중에 언어를 개발하실 때, 메모리를 어떻게 관리할지 결정해야 합니다.
 * C/C++ 방식: 개발자가 직접 메모리를 할당하고 해제함. (실수하면 시스템 붕괴)
 * Java/Python 방식: 가비지 컬렉터가 수시로 돌아다니며 청소함. (편하지만 프로그램이 가끔 멈춤)
 * Rust(gogs) 방식: 소유권 규칙을 통해 컴파일 타임에 메모리 생존 주기를 확정함. (가장 빠르고 안전함)
