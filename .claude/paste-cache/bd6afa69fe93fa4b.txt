🎓 Go-언어 전공 201: 2-4. 고루틴 생명주기 관리
1. sync.WaitGroup: "모두 끝날 때까지 기다려라"
여러 고루틴이 각자 작업을 수행할 때, 메인 함수가 먼저 종료되어 프로그램이 꺼지는 것을 방지합니다.
var wg sync.WaitGroup

for i := 1; i <= 3; i++ {
    wg.Add(1) // 작업 하나 추가
    go func(id int) {
        defer wg.Done() // 작업 완료 신호 (Add에서 1을 뺌)
        fmt.Printf("고루틴 %d 작업 중...\n", id)
    }(i)
}

wg.Wait() // 카운트가 0이 될 때까지 대기
fmt.Println("모든 작업 완료")

2. Context: "이제 그만하고 철수하라"
대학원 및 실무 아키텍처에서 가장 중요한 개념입니다. Context는 트리 구조로 신호를 전달하여, 상위 작업이 취소되면 하위에 연결된 모든 고루틴에게 취소 신호를 보냅니다.
 * 취소(Cancel): 사용자가 요청을 취소했을 때.
 * 타임아웃(Timeout): API 응답이 너무 늦어질 때 강제로 중단.
<!-- end list -->
ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
defer cancel()

go func(ctx context.Context) {
    select {
    case <-time.After(5 * time.Second):
        fmt.Println("작업 완료")
    case <-ctx.Done(): // 2초가 지나면 여기서 신호를 받음
        fmt.Println("타임아웃 발생: 작업 중단")
    }
}(ctx)

🔬 전공 심화: 고루틴 누수(Goroutine Leak) 방지
대학원 수준의 시스템 검수에서 가장 빈번하게 탈락하는 사유가 바로 고루틴 누수입니다.
 * 원인: 채널에서 데이터를 기다리느라 영원히 종료되지 않는 고루틴이 쌓여 메모리를 고갈시키는 현상.
 * 해결책: 모든 고루틴은 반드시 종료 조건을 가져야 합니다. Context를 인자로 넘겨 ctx.Done()을 체크하거나, 채널 닫기(close)를 명확히 설계해야 합니다.
🏗️ 실무 패턴: Context를 통한 데이터 전달
Context는 신호뿐만 아니라, 요청 단위의 **메타데이터(로그 ID, 인증 토큰 등)**를 하위 함수로 전달하는 용도로도 쓰입니다.
 * context.WithValue(parent, key, value) 를 사용하여 요청의 맥락(Context)을 유지합니다.
