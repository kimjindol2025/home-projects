🟢 Phase 5: 타입 시스템 강화 — 구현 지시서

1️⃣ 목표

정적 타입 검증: 컴파일 시점에 타입 불일치 감지

컴파일 타임 최적화: 타입 정보 활용 최적화

타입 안전성 보장: 런타임 오류 최소화



---

2️⃣ 핵심 구성 요소

1. TypeChecker

AST 기반 타입 검사

주요 기능:

변수 선언 및 참조 타입 확인

함수 인자/반환 타입 검사

연산자 타입 검증 (Add/Sub 등)


오류 발생 시 위치 정보 제공


2. TypeInference

타입 추론 엔진

기능:

타입 미지정 변수 자동 추론

함수 반환 타입 추론

제네릭/템플릿 타입 추론



3. TypeConstraints

타입 제약 조건 수집

기능:

변수와 표현식 사이 제약 생성

제약 충돌 검출

타입 최적화 경로 결정




---

3️⃣ 구현 단계

Step 1 — 기본 타입 시스템

Primitive 타입: num, str, bool

Composite 타입: array[T], object{prop: T}

함수 타입: (arg1: T1, …) -> Tn


Step 2 — TypeChecker 구현

AST 순회하며 타입 검사

연산자, 함수 호출, 변수 참조 확인

테스트 케이스 작성: 정상/비정상 타입 예시


Step 3 — TypeInference 통합

타입 미지정 변수 추론

함수 반환값 자동 추론

테스트 케이스: 추론 정확도 검증


Step 4 — TypeConstraints

제약 조건 생성 및 충돌 검출

타입 안정성 강화

테스트 케이스: 제약 충돌 확인


Step 5 — 컴파일 타임 최적화

타입 정보 활용 최적화

예: Add(num, num) → 직접 산술 연산, 런타임 타입 체크 제거

테스트: 최적화 적용 후 동작 검증



---

4️⃣ 테스트 계획 (총 20개)

Suite 1: Primitive 타입 (6개)

Suite 2: Composite 타입 (4개)
