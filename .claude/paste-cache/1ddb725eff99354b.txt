여러 대의 서비스(Go, Julia 등)가 거대한 지도를 이루었을 때, 외부 사용자가 이 모든 서비스의 주소를 알 필요는 없습니다. 또한 각 서비스마다 로그인 여부를 일일이 확인할 필요도 없죠. 이를 한곳에서 관리하는 것이 API Gateway와 통합 인증입니다.

🎓 Go-언어 전공 301: 3-4. API Gateway와 JWT 인증
1. API Gateway의 역할
API Gateway는 클라이언트와 백엔드 서비스 사이의 단일 진입점입니다.
 * 라우팅: /user 요청은 유저 서비스로, /calc 요청은 Julia 연산 서비스로 전달합니다.
 * 부하 분산: 여러 대의 인스턴스 중 하나로 요청을 배분합니다.
 * 공통 기능: 로깅, 인증, 속도 제한(Rate Limiting)을 일괄 처리합니다.
2. JWT (JSON Web Token) 기반 인증
분산 시스템에서는 '세션'을 쓰기 어렵습니다. 서버가 여러 대라 세션 정보를 공유하기 힘들기 때문이죠. 그래서 상태를 저장하지 않는(Stateless) JWT가 표준으로 쓰입니다.
 * 구조: Header.Payload.Signature
 * 장점: 서버는 DB를 매번 조회하지 않고도 토큰 내의 서명만 확인하여 사용자가 적법한지 즉시 판단할 수 있습니다.
3. Go를 이용한 미들웨어 설계
Go의 강력한 함수형 특성을 이용해 모든 요청을 검사하는 인증 미들웨어를 만듭니다.
func AuthMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        token := r.Header.Get("Authorization")
        if !validateToken(token) { // JWT 검증 로직
            http.Error(w, "Forbidden", http.StatusForbidden)
            return
        }
        next.ServeHTTP(w, r) // 검증 통과 시 다음 단계(실제 서비스)로 진행
    })
}

🔬 전공 심화: 권한 전파 (Token Exchange)
API Gateway가 토큰을 검증한 후, 내부 서비스들에게 "이 사용자는 누구다"라는 정보를 전달해야 합니다.
 * 방법: 인증이 완료된 사용자 정보를 HTTP Header(예: X-User-ID)에 담아 내부 서비스로 전달(Forwarding)합니다.
 * 보안: 내부망 사이의 통신이므로 외부에서는 이 헤더를 조작할 수 없도록 Gateway에서 철저히 차단해야 합니다.
🏗️ 설계의 완성: 속도 제한 (Rate Limiting)
특정 사용자가 시스템을 마비시키지 않도록 초당 요청 횟수를 제한합니다. Go의 golang.org/x/time/rate 패키지는 'Token Bucket' 알고리즘을 사용하여 이를 우아하게 처리합니다. Julia 연산처럼 비용이 비싼 서비스에는 이 제한이 필수적입니다.
