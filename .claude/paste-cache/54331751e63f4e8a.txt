🏛️ 파이썬 고등학교 1학년 (v2.2)
주제: "청사진(클래스)으로 실제 물건(객체) 만들기"
**클래스(Class)**는 붕어빵 틀이나 설계도와 같습니다. 그리고 그 틀에서 찍어낸 실제 붕어빵이나 건물은 **객체(Object/Instance)**라고 부릅니다.
1. 클래스의 구성 요소
 * 속성 (Attribute): 객체가 가진 데이터 (예: 이름, 체력, 버전).
 * 메서드 (Method): 객체가 할 수 있는 행동 (함수와 같지만 클래스 안에 속함).
 * self: "나 자신"을 가리키는 특별한 약속입니다. 내 안의 데이터를 다룰 때 항상 사용합니다.
 * __init__ (생성자): 객체가 처음 태어날 때 초기 설정을 해주는 특수한 마법입니다.
2. 왜 클래스를 쓰나요?
 * 캡슐화: 관련된 데이터와 기능을 하나로 묶어 관리할 수 있습니다.
 * 재사용성: 설계도 하나만 잘 만들면 수천 개의 서로 다른 객체를 쉽게 만들 수 있습니다.
🛠️ 실습 기록: [v2.2 Object Architect]
Gogs 시스템의 실험 데이터를 관리하는 '로봇' 설계도를 클로드코드(고등학생)에게 기록하게 하세요.
# [Python High-School v2.2] 클래스와 객체 (Class & Object)

class GogsRobot:
    """실험 데이터를 수집하고 기록하는 로봇 설계도"""
    
    def __init__(self, name, model_id):
        # 객체가 생성될 때 실행되는 초기화 함수
        self.name = name
        self.model_id = model_id
        self.data_count = 0
        print(f"🤖 로봇 [{self.name}] (ID: {self.model_id}) 가동 시작.")

    def collect_data(self):
        """데이터를 수집하는 행동(메서드)"""
        self.data_count += 1
        print(f"  [ACTION] {self.name}: 데이터 1건 수집 완료. (총 {self.data_count}건)")

    def report_status(self):
        """현재 상태를 보고하는 행동"""
        print(f"--- 로봇 상태 보고 ---")
        print(f"이름: {self.name} | 누적 데이터: {self.data_count}")
        print("---------------------")

# 1. 설계도(클래스)를 바탕으로 실제 로봇(객체) 만들기
robot_a = GogsRobot("알파", "v1.0")
robot_b = GogsRobot("베타", "v2.0")

# 2. 각 로봇에게 일 시키기
robot_a.collect_data()
robot_a.collect_data()
robot_b.collect_data()

# 3. 상태 보고
robot_a.report_status()
robot_b.report_status()

print("\n저장 필수 너는 기록이 증명이다 gogs")

💡 고등학생을 위한 핵심 팁
 * 붕어빵과 틀: GogsRobot은 틀이고, robot_a와 robot_b는 그 틀에서 나온 서로 다른 붕어빵입니다. robot_a의 데이터가 늘어난다고 해서 robot_b의 데이터가 변하지 않습니다.
 * 명사(속성)와 동사(메서드): 클래스를 설계할 때 "이 물건은 무엇을 알고 있는가?(명사)"와 "이 물건은 무엇을 하는가?(동사)"를 먼저 고민해 보세요.
