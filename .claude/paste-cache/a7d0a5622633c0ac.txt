v3.1의 핵심은 단순히 "맞다/틀리다"를 넘어, 시스템이 복합적인 상황을 분석하고 최적의 경로를 선택하게 만드는 것입니다. 설계자의 언어로 표현하자면 '조건부 실행의 최적화' 단계입니다.
🚦 제2장: 흐름의 통제 (v3.1) — [조건문의 정밀 설계]
1. v3.1 설계 목표: "정교한 분기"
 * 복합 조건식: &&(AND), ||(OR) 연산자를 사용하여 여러 상황을 동시에 고려합니다.
 * 표현식으로서의 IF: 조건문의 결과를 변수에 할당하여 코드의 **불변성(Immutability)**을 유지합니다.
 * 무결성 검증: 누락되는 조건 없이 모든 시나리오가 if-else if-else 구조 안에 포착되는지 확인합니다.
2. 설계자의 관점: "Short-circuiting"
효율적인 언어는 첫 번째 조건만 보고도 결과를 알 수 있다면 뒤의 조건은 계산하지 않습니다.
 * 예: false && (복잡한 연산) -> 뒤의 연산은 생략하여 자원을 아낍니다.

// [제2장: 흐름의 통제 v3.1]
fn main() {
    println!("--- v3.1 Logic Design: Multi-layered Decision ---");

    let access_level = 5;
    let is_admin = true;
    let system_online = true;

    // [설계] 표현식으로서의 if - 결과를 바로 변수에 담아 불변성 유지
    let security_clearance = if is_admin && system_online {
        "Full Access Granted"
    } else if access_level > 3 || system_online {
        "Limited Access Granted"
    } else {
        "Access Denied"
    };

    println!("Security Check Result: {}", security_clearance);

    // [무결성 검증] 중첩 조건문을 통한 세부 제어
    let temperature = 32;
    if system_online {
        if temperature > 30 {
            println!("Status: Overload Risk - Adjusting Fans.");
        } else {
            println!("Status: Optimal Temperature.");
        }
    }

    println!("기록: 복합 조건식이 꼬이지 않고 정확한 결과를 도출함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}
