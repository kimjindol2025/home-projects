마침내 제4장: 데이터 구조화 — v5.7 해시맵과 키-값 저장소 (HashMap & Key-Value Storage) 단계에 진입했습니다.
v5.5의 벡터가 '순서'를 기반으로 데이터를 관리했다면, v5.7의 해시맵은 **'의미(Key)'**를 기반으로 데이터를 즉시 찾아내는 고속 검색 시스템입니다. 설계자의 관점에서 해시맵은 복잡한 데이터 사이의 **연관 관계(Mapping)**를 정의하는 핵심 저장소입니다.
🏗️ 제4장: 데이터 구조화 (v5.7) — [해시맵과 검색]
1. v5.7 설계 목표: "즉각적인 데이터 매핑"
 * 키-값 쌍 (Key-Value): 고유한 키를 통해 그에 대응하는 데이터(Value)를 빛의 속도(O(1))로 찾아냅니다.
 * 동적 확장: 데이터의 양에 따라 해시 테이블의 크기를 유연하게 조절합니다.
 * 타입 안전성: 키와 값의 타입을 엄격히 제한하여 잘못된 데이터가 섞이는 것을 방지합니다.
2. 설계자의 관점: "인덱스가 아닌 이름으로 찾기"
사용자 ID로 사용자 정보를 찾거나, 설정 파일의 키로 설정값을 가져오는 등 **'이름표'**가 필요한 모든 곳에 해시맵이 사용됩니다. 이는 시스템의 탐색 효율성을 비약적으로 향상시킵니다.
🛠️ gogs 저장소 v5.7 설계: 시스템 권한 레지스트리
클로드코드(학생)에게 시스템 사용자별 권한 등급을 해시맵으로 관리하는 코드를 기록하게 하세요.
// [제4장: 데이터 구조화 v5.7]

use std::collections::HashMap; // 해시맵은 표준 라이브러리에서 가져와야 함

fn main() {
    println!("--- v5.7 Data Architecture: HashMap Storage ---");

    // 1. [설계] 시스템 권한 저장소 생성 (Key: 사용자명, Value: 권한등급)
    let mut permissions = HashMap::new();

    // 2. [삽입] 데이터 등록
    permissions.insert(String::from("Admin"), 10);
    permissions.insert(String::from("Operator"), 5);
    permissions.insert(String::from("Guest"), 1);

    // 3. [조회] 키를 이용한 안전한 접근 (Option 반환)
    let target_user = "Operator";
    match permissions.get(target_user) {
        Some(level) => println!("  [ACCESS] {}님의 권한 레벨: {}", target_user, level),
        None => println!("  [DENIED] 등록되지 않은 사용자입니다."),
    }

    // 4. [수정] 기존 키의 값 갱신
    permissions.entry(String::from("Admin")).and_modify(|lvl| *lvl += 1);

    // 5. [순회] 모든 권한 목록 출력
    println!("전체 권한 리포트:");
    for (name, level) in &permissions {
        println!("  - {}: Level {}", name, level);
    }

    println!("\n기록: 해시맵을 통해 키 기반의 고속 데이터 검색 체계를 구축함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 가이드: "v5.7 설계자의 검색 전략"
해시맵을 설계할 때 반드시 고려해야 할 '보안과 성능'입니다.
 * 소유권의 흐름: insert 시 키와 값의 소유권이 해시맵으로 이동합니다. 나중에 다시 쓰려면 참조를 사용하거나 복사해야 함을 잊지 마십시오.
 * entry API의 활용: "값이 없으면 넣고, 있으면 수정하라"는 복잡한 로직을 러스트의 entry 메서드는 단 한 줄로 안전하게 처리합니다.
 * 해시 공격 방어: 러스트의 기본 해시 알고리즘은 'SipHash'로, 서비스 거부(DoS) 공격에 강하도록 설계되어 보안성이 높습니다.
> 저장 필수, 너는 기록이 증명이다 gogs.
> 이제 당신의 엔진은 수많은 데이터 속에서도 단 하나의 단서를 가지고 원하는 정보를 즉시 낚아채는 **'베테랑 수사관'**의 눈을 가졌습니다.
