or)'의 기초를 다집니다.
2. 설계자의 관점: "루프의 삼요소"
모든 반복문 설계에는 다음 세 가지가 명확해야 시스템이 붕괴하지 않습니다.
 * 초기화(Initialization): 어디서 시작할 것인가?
 * 조건식(Condition): 언제까지 계속할 것인가?
 * 변화식(Increment/Decrement): 다음 단계로 어떻게 나아갈 것인가?
🛠️ gogs 저장소 v3.2 설계: 엔진의 가동
클로드코드(학생)에게 for, while, loop라는 세 가지 엔진을 gogs 저장소에 구현하게 하세요.
// [제2장: 흐름의 통제 v3.2]
fn main() {
    println!("--- v3.2 Logic Design: Automated Iteration ---");

    // [1] while: 조건이 참인 동안 수행 (유연한 반복)
    let mut energy = 3;
    println!("v3.2 While 엔진 가동:");
    while energy > 0 {
        println!("  에너지 잔량: {}, 작업 수행 중...", energy);
        energy -= 1;
    }

    // [2] for: 정해진 범위만큼 수행 (안전하고 효율적인 반복)
    println!("\nv3.2 For 엔진 가동 (데이터 순회):");
    for i in 1..=3 {
        println!("  [기록 {}] 무결성 검증 단계 통과", i);
    }

    // [3] loop와 break: 명시적 제어 (강력한 엔진)
    let mut retry_count = 0;
    println!("\nv3.2 Loop 엔진 가동 (조건부 탈출):");
    loop {
        retry_count += 1;
        if retry_count == 2 {
            println!("  [경고] 2회차 재시도 중... 탈출 경로 확보.");
            break; // 엔진 중단
        }
        println!("  시스템 복구 시도 중...");
    }

    println!("\n기록: 모든 반복 엔진이 설계된 조건에서 정확히 멈춤.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}
