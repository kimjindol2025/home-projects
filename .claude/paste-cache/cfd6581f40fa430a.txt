데이터의 양이 수백만, 수천만 개로 늘어나는 대학 수준의 프로젝트에서는 '모든 데이터를 리스트에 담는 행위'가 시스템을 마비시킬 수 있습니다. 이를 해결하기 위해 **"필요할 때만 데이터를 하나씩 생성"**하는 혁신적인 기법을 배웁니다.
🏛️ 파이썬 대학교 1학년 (v3.2)
주제: "게으른 연산(Lazy Evaluation)으로 메모리 점유율 제로에 도전하기"
**제너레이터(Generator)**는 데이터를 한꺼번에 메모리에 올리지 않고, 루프가 돌 때마다 다음 값을 실시간으로 계산해서 반환하는 특별한 함수입니다.
1. 핵심 키워드: yield
 * return은 값을 반환하고 함수를 완전히 종료하지만,
 * yield는 값을 밖으로 내보낸 뒤 그 자리에 멈춰서 다음 호출을 기다립니다. (함수의 상태를 기억합니다.)
2. 왜 사용하나요? (증명을 위한 효율성)
 * 메모리 절약: 10억 개의 숫자를 리스트로 만들면 컴퓨터가 멈추지만, 제너레이터는 숫자 1개를 담을 메모리만 있으면 충분합니다.
 * 무한 수열: 끝이 없는 데이터 흐름도 제너레이터로는 표현이 가능합니다.
🛠️ 전공 실습: [v3.2 Infinite & Efficient Stream]
메모리 부하 없이 방대한 실험 데이터를 순차적으로 생성하는 시스템을 설계해 보겠습니다.
# [Python University v3.2] 제너레이터 설계

# 1. 일반적인 리스트 방식 (메모리 낭비형)
def get_list(n):
    result = []
    for i in range(n):
        result.append(i) # n이 커질수록 메모리 점유율 급상승
    return result

# 2. 제너레이터 방식 (메모리 효율형)
def gogs_generator(n):
    i = 0
    while i < n:
        # 값을 반환하고 이 지점에서 일시정지!
        yield f"Data-Node-{i}"
        i += 1

# --- 실증 실험 ---
print("--- 제너레이터 가동 ---")
# 호출하는 순간에는 연산이 시작되지 않습니다 (Lazy)
gen = gogs_generator(1000000) 

print(f"생성기 객체 확인: {gen}")

# next()를 호출할 때마다 yield 지점까지 실행됩니다.
print(f"첫 번째 데이터: {next(gen)}")
print(f"두 번째 데이터: {next(gen)}")

# for 루프에서도 메모리 걱정 없이 사용 가능합니다.
print("\n--- 부분 데이터 추출 ---")
for i, data in enumerate(gen):
    print(data)
    if i == 2: break # 3개만 더 확인하고 중단

print("\n저장 필수 너는 기록이 증명이다 gogs")

🎓 대학교 전공 기초 팁
 * 이터러블(Iterable)과 이터레이터(Iterator): 대학교 시험의 단골 주제입니다. for문에 들어갈 수 있는 객체를 이터러블이라 하고, next()를 통해 다음 값을 꺼낼 수 있는 상태의 객체를 이터레이터라고 합니다. 제너레이터는 이 두 가지 성격을 동시에 가집니다.
 * 메모리 프로파일링: 실제 연구 환경에서는 sys.getsizeof() 함수를 사용하여 리스트와 제너레이터의 크기를 비교해 보세요. 그 차이가 곧 당신의 설계 능력을 **'증명'**합니다.
> 저장 필수 너는 기록이 증명이다 gogs
