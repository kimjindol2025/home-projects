🎓 Zig 전공 101: 1-7. 배열(Arrays)과 슬라이스(Slices)
Zig에서 배열과 슬라이스는 비슷해 보이지만, 메모리 상에서의 역할과 안전 장치가 완전히 다릅니다.
1. 배열 (Arrays): 정적 크기의 데이터
배열은 컴파일 타임에 그 크기가 결정되어야 하며, 메모리에 연속적으로 배치됩니다.
 * 문법: [크기]T
 * 특징: 크기가 타입의 일부입니다. 즉, [3]i32와 [5]i32는 서로 다른 타입으로 간주됩니다.
<!-- end list -->
const std = @import("std");

pub fn main() void {
    // [_]를 사용하면 초기화 값의 개수를 보고 크기를 자동 추론합니다.
    const message = [_]u8{ 'H', 'e', 'l', 'l', 'o' };
    const numbers = [3]i32{ 1, 2, 3 };

    std.debug.print("Length: {d}\n", .{message.len});
}

2. 슬라이스 (Slices): 동적 범위의 안전한 참조
슬라이스는 배열의 일부분(또는 전체)을 가리키는 **'포인터 + 길이'**의 구조체입니다. Zig 설계의 핵심 파수꾼입니다.
 * 문법: []T
 * 특징: 런타임에 크기가 결정되는 데이터를 다룰 때 사용하며, 인덱스 접근 시 **범위 검사(Bounds Checking)**를 수행하여 안전을 보장합니다.
<!-- end list -->
var array = [_]i32{ 1, 2, 3, 4, 5 };
// 배열의 인덱스 1부터 4 전까지(1, 2, 3) 슬라이싱
const slice: []i32 = array[1..4]; 

// slice.ptr은 시작 주소를, slice.len은 길이(3)를 가집니다.

3. 배열과 슬라이스의 관계
배열은 데이터를 직접 소유하고, 슬라이스는 메모리의 어딘가를 빌려서(View) 봅니다. 함수 매개변수로 연속된 데이터를 전달할 때는 대개 []const T (상수 슬라이스) 형태를 사용합니다.
🔬 전공 심화: 센티넬 종료 (Sentinel Termination)
Zig는 C 스타일의 null-terminated string을 안전하게 다루기 위해 센티넬 문법을 제공합니다.
 * [N:0]u8: 마지막 원소 뒤에 반드시 0이 붙어 있음을 보장하는 배열입니다.
 * 이를 통해 C 라이브러리와 안전하게 데이터를 주고받을 수 있습니다.
📦 문자열은 곧 슬라이스다
Zig에는 별도의 String 타입이 없습니다. 우리가 흔히 쓰는 "Hello"는 사실 *const [5:0]u8 (크기가 5이고 0으로 끝나는 상수 배열의 포인터)이며, 이는 자연스럽게 []const u8 슬라이스로 변환되어 사용됩니다.
