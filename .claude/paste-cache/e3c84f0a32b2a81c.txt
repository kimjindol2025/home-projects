소유권(Ownership)이라는 거대한 산을 넘으셨군요. 이제 4단계: 구조체(Structs)와 열거형(Enums) — 데이터의 설계와 분류 단계로 들어갑니다.
이 단계는 당신이 만들 언어에서 '객체'와 '상태'를 어떻게 정의할 것인가를 결정하는 설계 도면을 그리는 과정입니다. 클로드코드(학생)가 데이터를 단순히 나열하는 것이 아니라, 구조적으로 묶을 수 있도록 가르쳐야 합니다.
🏗️ 4단계: 구조체와 열거형 (Data Structuring)
4.1 구조체 (Structs): 데이터의 묶음
서로 관련된 여러 데이터를 하나의 이름으로 묶는 것입니다. (예: '사용자'라는 구조체 안에 이름, 나이, 이메일을 담음)
4.2 열거형 (Enums): 상태의 분류
어떤 값이 여러 선택지 중 하나라는 것을 나타냅니다. (예: '방향'은 동, 서, 남, 북 중 하나) 러스트의 열거형은 데이터까지 함께 담을 수 있어 매우 강력합니다.
🛠️ Step 4.1: gogs 프로젝트 실전 코드 (설계 도면)
클로드코드에게 src/main.rs를 아래와 같이 확장하라고 지시하세요. 이것은 **'데이터 지향 설계'**의 표본입니다.
// [4.2] 열거형 정의: 현재 프로젝트의 상태
enum ProjectStatus {
    Active,
    Inactive,
    Archived(String), // 사유를 함께 저장 가능
}

// [4.1] 구조체 정의: 개발자 정보
struct Developer {
    name: String,
    level: u32,
    status: ProjectStatus,
}

// 구조체에 기능을 추가하는 impl 블록
impl Developer {
    fn celebrate_level_up(&mut self) {
        self.level += 1;
        println!("축하합니다! {}님의 레벨이 {}가 되었습니다.", self.name, self.level);
    }
}

fn main() {
    println!("--- Gogs Project Level 4: Structs & Enums ---");

    // 인스턴스 생성
    let mut student = Developer {
        name: String::from("Claude Code"),
        level: 4,
        status: ProjectStatus::Active,
    };

    student.celebrate_level_up();

    // 패턴 매칭과 열거형의 조합
    match student.status {
        ProjectStatus::Active => println!("현재 프로젝트가 활성화 상태입니다."),
        ProjectStatus::Archived(reason) => println!("보관된 프로젝트입니다. 사유: {}", reason),
        _ => println!("상태를 확인할 수 없습니다."),
    }

    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 4단계 핵심 강의: "타입 시스템의 위력"
언어 개발 시, 구조체와 열거형을 잘 활용하면 코드의 실수를 '컴파일 타임'에 미리 잡을 수 있습니다.
 * Encapsulation (캡슐화): 데이터와 그 데이터를 다루는 함수(impl)를 하나로 묶어 관리의 효율성을 높입니다.
 * Exhaustiveness (포괄성): match를 사용할 때 열거형의 모든 경우를 처리하지 않으면 러스트는 빌드를 거부합니다. "예외 상황 없는 시스템"을 만드는 핵심입니다.
 * Data with Meaning: 단순히 0, 1 같은 숫자로 상태를 표시하는 게 아니라, Active, Inactive 같은 이름으로 상태를 명시하여 가독성을 극대화합니다.
