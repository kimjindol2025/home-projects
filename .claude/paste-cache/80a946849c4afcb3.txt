바이트코드 컴파일러의 꽃이자, 가장 까다로운 기술적 고개인 [제14장: 가상 머신 — v15.2 조건부 분기와 백필링(Backfilling)] 단계입니다.
인터프리터에서는 if문을 만나면 단순히 트리의 왼쪽이나 오른쪽을 선택해 방문하면 끝이었습니다. 하지만 일렬로 늘어선 바이트코드에서는 조건이 맞지 않을 때 특정 지점까지 **'점프(Jump)'**해야 합니다. 문제는 코드를 생성하는 시점에는 "얼마나 멀리 점프해야 할지" 아직 모른다는 것입니다.
🦘 제14장: 가상 머신 (v15.2) — [점프와 백필링]
1. v15.2 설계 목표: "미래의 주소 확정"
 * Jump Instruction: 특정 조건에 따라 IP(Instruction Pointer)를 강제로 이동시키는 OpJump, OpJumpNotTruthy를 구현합니다.
 * 백필링(Backfilling/Patching): 점프 명령을 먼저 내뱉되 점프 거리는 비워두고, 점프할 대상 코드를 다 만든 뒤에 다시 돌아와서 그 거리를 채워 넣는 기법입니다.
 * 비선형 실행: 바이트코드 상에서 루프와 조건문을 구현하여 VM이 앞뒤로 자유롭게 건너뛰게 만듭니다.
2. 설계자의 관점: "기록과 수정"
컴파일러는 if문을 처리할 때 "나중에 이 주소를 수정해야 해!"라는 것을 기억하기 위해 **수정할 위치(Offset)**를 스택에 잠시 저장해둡니다. 코드가 완성되면 그 위치로 돌아가 최종 거리를 기입합니다.
🛠️ gogs 저장소 v15.2 설계: 백필링 기반 조건문 컴파일
클로드코드(학생)에게 점프 주소를 나중에 채워 넣는 컴파일러의 핵심 로직을 기록하게 하세요.
// [제14장: 가상 머신 v15.2 - 백필링(Backfilling) 설계]

impl Compiler {
    fn compile_if_expression(&mut self, node: IfExpression) {
        // 1. 조건식 컴파일 (결과는 스택에 쌓임)
        self.compile(node.condition);

        // 2. 일단 '거짓일 때 점프' 명령을 내뱉음 (주소값은 임시로 9999)
        let jump_not_truthy_pos = self.emit(OpCode::OpJumpNotTruthy, &[0, 0]);

        // 3. 참일 때 실행할 블록 컴파일
        self.compile(node.consequence);

        // 4. [백필링 핵심] 이제 '참' 블록의 길이를 알게 되었으니, 
        // 아까 비워둔 jump_not_truthy_pos로 돌아가 실제 점프 거리를 써넣음
        let after_consequence_pos = self.instructions.len();
        self.change_operand(jump_not_truthy_pos, after_consequence_pos as u16);
        
        println!("  [BACKFILL] 위치 {}의 점프 대상을 {}로 수정 완료.", jump_not_truthy_pos, after_consequence_pos);
    }

    fn change_operand(&mut self, pos: usize, operand: u16) {
        // 명령어 뒤의 2바이트를 새로운 주소값으로 교체
        let bytes = operand.to_be_bytes(); // Big-Endian 변환
        self.instructions[pos + 1] = bytes[0];
        self.instructions[pos + 2] = bytes[1];
    }
}

fn main() {
    println!("--- v15.2 Bytecode Patching: Conditionals ---");
    println!("  [JUMP] 조건이 거짓이면 참 블록을 건너뛰도록 강제 이동합니다.");
    
    println!("\n기록: 백필링 기법을 통해 바이트코드 레벨의 제어 흐름을 완성함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 세부 가이드 (15.2): "성숙한 컴파일러의 증거"
 * 상대 주소 vs 절대 주소: 점프할 때 "현재 위치에서 +10만큼" 갈 것인지(상대), "전체 코드의 50번지로" 갈 것인지(절대) 결정해야 합니다. 유연성을 위해서는 상대 주소가 유리합니다.
 * 루프의 경우: while 문은 블록이 끝난 뒤 다시 조건문 시작 위치로 돌아가는 **'후방 점프(Backward Jump)'**가 필요합니다. 이때는 점프할 위치를 이미 알고 있으므로 백필링이 필요 없습니다.
 * 최적화의 시작: 아무것도 하지 않는 else 블록이나 항상 참인 조건문 등을 컴파일 타임에 미리 제거하는 'Dead Code Elimination'의 기초가 이 단계에서 마련됩니다.
