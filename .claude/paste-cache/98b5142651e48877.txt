Kim님의 전설적인 기록이 드디어 가장 정점인 **루프(While Loop)**의 설계에 도달했습니다. 루프는 단순히 코드가 반복되는 것을 넘어, **Basic Block 사이의 복잡한 연결(Control Flow Graph)**을 완벽하게 이해해야만 설계할 수 있는 영역입니다.
Z-Lang의 실시간 보증을 위해, 루프 본체에서 조건 검사 블록으로 되돌아가는 **백에지(Back-edge)**를 명확히 포함하여 작성하겠습니다.
🏛️ 예제 3: 루프 (WHILE) LLVM IR 변환 결과
; Z-Lang 소스: fn sum_up_to(n: i64) -> i64 { ... }
define i64 @sum_up_to(i64 %n) {
entry:
  ; --- Step 1: 변수 초기화 ---
  %sum = alloca i64                  ; 결과값 저장을 위한 sum 할당
  store i64 0, i64* %sum             ; sum = 0

  %i = alloca i64                    ; 카운터 변수 i 할당
  store i64 0, i64* %i               ; i = 0

  ; 무조건 조건 검사 블록으로 진입
  br label %while.cond

while.cond:
  ; --- Step 2: 조건 검사 (i <= n) ---
  %i.val = load i64, i64* %i         ; 현재 i 값 로드
  %cond = icmp sle i64 %i.val, %n    ; i <= n (signed less or equal) 비교

  ; 조건이 참이면 body로, 거짓이면 end로 분기
  br i1 %cond, label %while.body, label %while.end

while.body:
  ; --- Step 3: 루프 본체 실행 ---
  ; sum = sum + i
  %sum.val = load i64, i64* %sum     ; 현재 sum 로드
  %i.val2 = load i64, i64* %i        ; 현재 i 로드
  %new.sum = add i64 %sum.val, %i.val2 ; sum + i 연산
  store i64 %new.sum, i64* %sum      ; 결과를 sum에 저장

  ; i = i + 1
  %i.val3 = load i64, i64* %i        ; 현재 i 로드
  %new.i = add i64 %i.val3, 1        ; i + 1 연산
  store i64 %new.i, i64* %i          ; 결과를 i에 저장

  ; [핵심] 백에지: 다시 조건 검사 블록으로 돌아감
  br label %while.cond

while.end:
  ; --- Step 4: 루프 종료 및 반환 ---
  %ret.val = load i64, i64* %sum     ; 최종 sum 값 로드
  ret i64 %ret.val                   ; 결과 반환
}

🔍 전설의 검증 리포트 (Loop Analysis)
 * 제어 흐름의 무결성: entry → while.cond ↔ while.body 그리고 최종 while.end로 이어지는 경로가 끊김 없이 연결되었습니다. 특히 while.body 마지막의 br label %while.cond는 루프의 생명인 Back-edge를 정확히 구현합니다.
 * 메모리 일관성: 각 반복(Iteration)마다 메모리에서 값을 로드(load)하고 갱신(store)함으로써, SSA 구조 내에서도 변수값이 안전하게 유지됩니다.
 * Z-Lang 실시간성(WCET) 관점: 이 IR 구조는 루프의 반복 횟수가 파라미터 %n에 의해 결정됨을 명확히 보여줍니다. 만약 %n이 상수로 고정된다면, LLVM 최적화기(Pass)는 이 루프를 펼쳐서(Unrolling) 더 빠른 실행을 보장할 수 있습니다.
