🟢 Phase 3: Mark-Sweep + RC Hybrid GC — 구체적 구현 지시서

1️⃣ GC 구조

HeapManager 확장

Ptr → Value 매핑 유지

refCount + marked 플래그 추가

메서드:

allocate(obj: Value): Ptr

get(ptr: Ptr): Value

free(ptr: Ptr)

increment(ptr: Ptr)

decrement(ptr: Ptr)

mark(ptr: Ptr)

sweep()

markAndSweep()



Root 참조

Stack: 현재 연산 중인 Value

Globals: 모든 전역 변수

Frames: 현재 함수 호출 프레임의 지역 변수



---

2️⃣ Mark 단계

1. 모든 root 참조를 탐색


2. 방문한 객체 marked = true


3. 내부 참조도 재귀 탐색


4. 배열/객체의 모든 요소 재귀 mark



주의: 이미 marked이면 재귀 종료


---

3️⃣ Sweep 단계

1. HeapManager 전체 Ptr 순회


2. marked = false 객체는 free()


3. marked = true 객체는 marked = false 초기화 (다음 GC 대비)




---

4️⃣ RC + Hybrid 정책

객체 참조 감소 시 refCount--

refCount == 0 → 즉시 free

순환 참조 가능 시 Mark-Sweep 실행


if (obj.refCount === 0) {
    if (!detectCycle(obj)) heapManager.free(ptr);
    else markAndSweep(); // 순환 구조 처리
}

모든 Heap 객체는 RC 추적 + 필요시 Mark-Sweep 실행



---

5️⃣ 테스트 케이스

Suite 1 — 일반 RC

1. 단일 객체 생성/삭제 → refCount 0 시 free


2. 배열 참조 추가/삭제 → refCount 증가/감소


3. 객체 속성 참조 증가/감소



Suite 2 — 순환 구조

4. 2개 객체 순환 참조 → 제거 후 Mark-Sweep 확인


5. 중첩 배열 + 객체 순환 → GC 후 메모리 회수 확인



Suite 3 — 통합 검증

6. Phase 2 Step 1-5 연산 + GC 실행 → 모든 Value 정상 유지


7. Heap 메모리 잔여 확인 → 누수 없음


8. 성능 오버헤드 <5% 확인



총 8개 테스트 + 기존 62개 테스트 통합 → 최종 70개 테스트
