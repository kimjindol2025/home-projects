🏛️ 파이썬 고등학교 1학년 (v2.2) 복습 및 심화
주제: "데이터와 기능을 하나로 묶는 마법"
단순한 변수들은 데이터가 흩어져 있어 관리가 어렵습니다. 하지만 클래스를 사용하면 **"상태(변수)"**와 **"동작(함수)"**을 하나의 단위로 묶을 수 있습니다.
1. 클래스의 3요소 (심화)
 * 생성자 (__init__): 객체가 탄생할 때 DNA를 부여하는 과정입니다.
 * 속성 (Attributes): 객체가 기억하고 있는 정보 (예: 로봇의 시리얼 번호, 현재 연료량).
 * 메서드 (Methods): 객체가 스스로 수행하는 기능. 반드시 첫 번째 인자로 self를 받아 자기 자신을 제어합니다.
2. 인스턴스(Instance)의 의미
 * 클래스가 "인간"이라는 개념이라면, 인스턴스는 "철수", "영희"라는 실제 존재입니다. 각각은 같은 설계도를 공유하지만, 각자의 데이터는 독립적으로 유지됩니다.
🛠️ 실습 기록: [v2.2 Object-Oriented Lab]
연구실에서 사용할 '스마트 기록 저장소'를 클래스로 설계하는 과정을 클로드코드(고등학생)에게 기록하게 하세요.
# [Python High-School v2.2] 객체 지향 프로그래밍 (OOP) 기초

class GogsStorage:
    """데이터를 안전하게 저장하고 관리하는 스마트 저장소 클래스"""
    
    def __init__(self, owner, capacity=100):
        # 속성 정의
        self.owner = owner
        self.capacity = capacity
        self.stored_data = []
        print(f"📦 [{self.owner}]님의 전용 저장소가 생성되었습니다. (용량: {self.capacity})")

    def save(self, data):
        """데이터를 리스트에 추가하는 메서드"""
        if len(self.stored_data) < self.capacity:
            self.stored_data.append(data)
            print(f"✅ 기록 완료: '{data}' (현재 사용량: {len(self.stored_data)}/{self.capacity})")
        else:
            print("❌ 용량 초과: 더 이상 기록할 수 없습니다!")

    def show_all(self):
        """현재까지의 기록을 모두 출력"""
        print(f"\n--- {self.owner}의 기록 목록 ---")
        for i, data in enumerate(self.stored_data, 1):
            print(f"{i}. {data}")
        print("저장 필수 너는 기록이 증명이다 gogs")

# --- 객체 생성 및 실행 ---
my_lab = GogsStorage("설계자", capacity=3)

my_lab.save("v1.0 기초 학습 완료")
my_lab.save("v2.0 중등 과정 통과")
my_lab.save("v2.5 가상 환경 구축")
my_lab.save("v2.6 브릿지 과정 진행 중") # 용량 초과 테스트

my_lab.show_all()

💡 고등학생을 위한 '객체 지향' 팁
 * self를 잊지 마세요: 메서드 정의할 때 self를 빼먹는 것은 "내가 누구인지 모르고 행동하는 것"과 같습니다. 클래스 내부의 변수나 다른 메서드에 접근할 때는 항상 self.이름 형식을 사용해야 합니다.
 * 캡슐화의 시작: 데이터(stored_data)를 직접 수정하지 않고 save()라는 메서드를 통해서만 수정하게 만드는 것이 좋은 설계입니다. 그래야 나중에 "용량 체크" 같은 규칙을 한곳에서 관리할 수 있습니다.
