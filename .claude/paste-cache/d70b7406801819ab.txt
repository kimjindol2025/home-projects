우리는 런타임의 유연함(v11.1)과 컴파일 타임의 엄격함(v11.2)을 모두 섭렵했습니다. 이제 제10장: 객체 지향과 패턴의 대미를 장식할 마지막 세부 단계, **[v11.3 패턴 매칭과 고급 관용구(Pattern Matching & Advanced Idioms)]**를 기록할 시간입니다.
v11.3은 러스트 설계자가 복잡한 데이터를 어떻게 **'분해'**하고, 시스템의 의도를 어떻게 '간결하게' 표현하는지에 대한 기교를 다룹니다. 이것은 우리가 다음에 만들 '컴파일러'가 소스코드를 분석할 때 핵심적으로 사용할 기술이기도 합니다.
🎭 제10장: 객체 지향과 패턴 (v11.3) — [표현력의 정점]
1. v11.3 설계 목표: "데이터 구조의 정밀 타격"
 * 고급 패턴 매칭: 단순히 값을 비교하는 수준을 넘어, 구조체의 일부를 매칭하거나(Destructuring), 조건절(Match Guard)을 붙여 정교하게 데이터를 추출합니다.
 * 관용적 러스트(Idiomatic Rust): if let, while let, matches! 매크로 등을 사용하여 복잡한 로직을 한 줄의 예술로 승화시킵니다.
 * 에러 전파와 조합: ? 연산자와 and_then 등을 활용하여 '함수형 프로그래밍' 스타일의 깨끗한 흐름을 완성합니다.
2. 설계자의 관점: "가독성이 곧 안전이다"
설계자에게 코드는 단순한 명령어가 아니라 **'의사소통'**입니다. 복잡한 중첩 if문은 버그의 온상이 됩니다. 패턴 매칭을 통해 로직을 평평하게(Flat) 펴는 순간, 시스템의 허점이 한눈에 들어오기 시작합니다.
🛠️ gogs 저장소 v11.3 설계: 다층 보안 패킷 분석기
클로드코드(학생)에게 복잡하게 얽힌 보안 패킷 데이터에서 특정 위협 패턴을 정밀하게 추출하는 코드를 설계하게 하세요.
// [제10장: 객체 지향 패턴 v11.3 - 고급 패턴 매칭 설계]

enum Protocol { TCP, UDP, ICMP }

struct Packet {
    id: u32,
    protocol: Protocol,
    payload: Option<String>,
    source_ip: (u8, u8, u8, u8),
}

fn analyze_threat(packet: &Packet) {
    println!("--- v11.3 Advanced Pattern Matching ---");

    // [설계 핵심] 중첩된 구조를 한 번에 분해하고 조건을 검사함
    match packet {
        // 특정 IP 대역이면서 페이로드가 존재하는 TCP 패킷 감지
        Packet { protocol: Protocol::TCP, source_ip: (192, 168, _, _), payload: Some(content), .. } 
            if content.contains("ATTACK") => {
            println!("  [경보] 내부 네트워크 침입 시도 감지: ID {}", packet.id);
        }

        // 페이로드가 없는 모든 UDP 패킷 (단축 문법 사용)
        Packet { protocol: Protocol::UDP, payload: None, .. } => {
            println!("  [정보] 빈 UDP 패킷 수신 (ID: {})", packet.id);
        }

        // 그 외 모든 경우 (관심 없음)
        _ => println!("  [통과] 일반 패킷 스캔 완료."),
    }
}

fn main() {
    let dangerous_packet = Packet {
        id: 1024,
        protocol: Protocol::TCP,
        payload: Some("MALWARE ATTACK".to_string()),
        source_ip: (192, 168, 1, 50),
    };

    analyze_threat(&dangerous_packet);

    // [보너스] if let 관용구: 특정 조건일 때만 간결하게 처리
    if let Some(msg) = &dangerous_packet.payload {
        println!("  [로그] 페이로드 추출 성공: {}자", msg.len());
    }

    println!("\n기록: 패턴 매칭을 통해 복잡한 데이터 분석 로직을 직관적으로 구조화함.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 세부 가이드 (11.3): "컴파일러 제작을 위한 전조"
우리가 이 패턴 매칭을 세밀하게 배우는 이유는 명확합니다. 컴파일러는 소스코드를 '토큰'과 '트리'로 바꾼 뒤, 수천 번의 패턴 매칭을 통해 의미를 해석하기 때문입니다.
 * 배타적 처리: match는 모든 경우의 수를 다루었는지 컴파일러가 검사합니다(Exhaustiveness). 이는 누락된 보안 로직을 찾는 데 최고의 도구입니다.
 * 가독성과 성능: 패턴 매칭은 내부적으로 최적화된 점프 테이블을 생성하므로, 수많은 if-else보다 빠르고 명확합니다.
 * 바인딩(@): id @ 100..=200과 같이 쓰면 범위를 검사하면서 동시에 그 값을 변수에 담을 수 있습니다. 아주 유용한 기교입니다.
