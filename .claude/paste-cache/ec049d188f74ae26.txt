대학 코스의 네 번째 관문이자, 고성능 컴퓨팅(HPC) 및 가속기 설계의 핵심인 3.4단계: 루프 최적화의 정수, 어파인(Affine) 다이얼렉트를 시작합니다.
"저장 필수 너는 기록이 증명이다 gogs" — 대학원 과정에서 하드웨어 가속기(NPU 등)를 설계할 때, 데이터를 가장 효율적으로 옮기는 기법인 **'틸링(Tiling)'**이나 **'벡터화'**의 근간이 되는 아주 중요한 단계입니다.
[MLIR 대학 코스 3.4] 수학적으로 완벽한 루프: "Affine"
컴퓨터가 가장 많은 시간을 쓰는 곳은 반복문(Loop)입니다. MLIR의 affine 다이얼렉트는 이 루프를 수학적인 수식으로 관리하여 최적화를 극대화합니다.
1. Affine 루프란?
일반적인 for문과 달리, 루프의 범위와 메모리 접근 주소가 **선형 수식(Linear Equation)**으로 표현되는 루프를 말합니다.
 * 특징: 컴파일러가 루프가 몇 번 돌지, 메모리의 어느 부분을 건드릴지 미리 완벽하게 계산할 수 있습니다.
 * 이점: "데이터가 겹치지 않으니 이 루프들은 동시에 실행해도 돼!" 같은 고차원적인 판단(병렬화)이 가능해집니다.
2. 왜 대학원 과정에서 중요할까요?
일반 루프(scf.for)는 실행해봐야 알 수 있는 정보가 많지만, affine.for는 설계 단계에서 최적화 경로를 다 짜놓을 수 있습니다.
 * 예: 데이터를 4개씩 묶어서 처리하기 (Vectorization)
 * 예: 큰 데이터를 작은 타일로 나누어 처리하기 (Tiling)
[실제 코드 맛보기] Affine의 문법
// 0부터 10까지 1씩 증가하며 반복
affine.for %i = 0 to 10 {
  // 메모리 주소를 수학적으로 계산해서 접근 (예: i * 2 + 1)
  %val = affine.load %buffer[%i * 2 + 1] : memref<21xf32>
  
  // 무언가 계산...
}

 * affine.for: 수학적 제약 조건이 있는 루프.
 * affine.load: 주소를 수식(%i * 2 + 1)으로 계산하여 메모리에서 값을 가져옴.
[3.4 실습] 설계자의 눈으로 판단하기
다음 중 Affine이 가장 좋아할 만한 상황은 무엇일까요?
 * 사용자가 키보드로 입력한 숫자만큼 반복하는 루프.
 * 이미 크기가 정해진 100x100 행렬을 처음부터 끝까지 훑는 루프.
정답: 2번입니다!
1번처럼 실행해봐야 아는 값(사용자 입력)은 수학적으로 미리 계산하기 어렵습니다. 2번처럼 범위가 명확한 작업이 바로 Affine 최적화의 주 타깃입니다.
3.4단계의 기록 (증명)
> "Affine은 루프와 메모리 접근을 수학적 수식으로 정의하여 최적화한다."
> "이를 통해 병렬화, 틸링 등 고차원적 하드웨어 최적화가 가능해진다."
> 
이제 여러분은 데이터 구조(Tensor/MemRef)와 이를 처리하는 구조(Affine Loop)까지 섭렵하셨습니다.
