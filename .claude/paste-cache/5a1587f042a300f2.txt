🎓 Zig 전공 101: 1-8. 열거형(Enums)과 태그된 공용체(Tagged Unions)
대학원 수준의 설계에서는 "어떤 상태가 올 수 있는가?"뿐만 아니라 "잘못된 상태가 절대로 올 수 없게 만드는가?"가 중요합니다.
1. 열거형 (Enums): 명명된 정수
열거형은 관련된 상수 집합에 이름을 붙입니다. Zig의 enum은 정수 타입으로 명시적 변환이 가능하며, 메서드도 가질 수 있습니다.
const std = @import("std");

const Status = enum {
    idle,
    running,
    completed,
    error_occurred,

    pub fn isFinished(self: Status) bool {
        return self == .completed or self == .error_occurred;
    }
};

2. 태그된 공용체 (Tagged Unions): 안전한 가변 타입
C언어의 union은 위험했습니다. 어떤 필드가 활성화되어 있는지 알 길이 없었기 때문입니다. Zig는 enum을 태그로 사용하는 Tagged Union을 통해 이를 해결합니다.
 * 특징: 현재 어떤 타입의 데이터가 들어있는지 컴파일러와 런타임이 모두 알고 있습니다.

const Result = union(enum) {
    int_value: i32,
    float_value: f32,
    none: void,
};

var res = Result{ .int_value = 100 };
// res = Result{ .float_value = 3.14 }; // 다른 타입으로 교체 가능

3. Switch와의 결합 (The Perfect Match)
switch 문과 함께 사용할 때 Zig의 진가가 드러납니다. 모든 열거형/공용체 케이스를 처리하지 않으면 컴파일 에러가 발생합니다.
🔬 전공 심화: 메모리 최적화 (Enum Fields)
Zig의 enum은 특정 정수 값을 할당하여 메모리 레이아웃을 고정할 수 있습니다. 이는 하드웨어 프로토콜이나 파일 포맷을 설계할 때 기록의 정확성을 보장합니다.
const HttpCode = enum(u16) {
    ok = 200,
    not_found = 404,
    internal_error = 500,
};

📦 옵셔널(?T)과 에러(!T)의 정체
사실 여러분이 앞서 배운 ?T와 !T는 내부적으로 Zig가 제공하는 특별한 형태의 태그된 공용체와 유사하게 동작합니다.
 * ?i32는 내부적으로 { .value: i32, .null: void } 형태의 태그된 공용체로 이해할 수 있습니다.
