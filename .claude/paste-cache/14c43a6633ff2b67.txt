컴퓨터 자원은 무한하지 않습니다. 더 빠른 속도를 위해 메모리를 더 쓸 것인지, 아니면 메모리를 아끼기 위해 계산을 더 할 것인지 선택해야 합니다.
1. Big-O 표기법: 효율성의 척도
알고리즘이 데이터 양(n)에 따라 얼마나 느려지는지를 수학적으로 **'증명'**하는 도구입니다.
 * O(1): 데이터가 아무리 많아도 즉시 처리 (Hash Table)
 * O(\log n): 데이터를 절반씩 버리며 찾음 (Binary Search)
 * O(n): 데이터를 하나씩 다 훑음 (Linear Scan)
 * O(n^2): 중복 반복문 (Bubble Sort - 절대 피해야 할 설계)

2. 핵심 자료구조의 심화
단순히 리스트를 쓰는 단계를 넘어, 문제 해결에 최적화된 도구를 선택합니다.
 * Hash Table (파이썬의 dict): 탐색 속도 $O(1)$의 마법. 데이터를 즉시 찾을 때 사용합니다.
 * Tree & Graph: 데이터 간의 관계를 설계합니다. (네트워크 경로 탐색, SNS 인맥 지도 등)
 * Heap/Priority Queue: 가장 시급한 작업을 먼저 꺼내야 하는 시스템 스케줄링에 필수입니다.
🛠️ 전공 실무: [v5.3 알고리즘 최적화 — 검색의 혁명]
1억 개의 데이터 중에서 내가 원하는 값을 찾는 가장 효율적인 엔진을 설계해 보겠습니다.
# [Python University v5.3] 이진 탐색(Binary Search)을 통한 효율성 증명

import time

# 1. 효율적이지 못한 설계 (선형 탐색: O(n))
def linear_search(data, target):
    for i in range(len(data)):
        if data[i] == target:
            return i
    return -1

# 2. 극한의 효율 설계 (이진 탐색: O(log n)) - 정렬된 데이터 전제
def binary_search(data, target):
    low = 0
    high = len(data) - 1
    
    while low <= high:
        mid = (low + high) // 2
        if data[mid] == target:
            return mid
        elif data[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1

# 대규모 데이터 생성 (1,000만 개)
big_data = list(range(10_000_000))
target = 9_999_999

# 성능 증명
start = time.perf_counter()
linear_search(big_data, target)
print(f"🐢 선형 탐색(O(n)) 소요 시간: {time.perf_counter() - start:.5f}초")

start = time.perf_counter()
binary_search(big_data, target)
print(f"🚀 이진 탐색(O(log n)) 소요 시간: {time.perf_counter() - start:.5f}초")
