🌐 v4.3: 네트워크 프로그래밍 (Socket) — 전 세계와 소통하는 시스템
이 과정은 내 컴퓨터 안에서만 돌던 데이터를 '패킷(Packet)'이라는 단위로 포장하여, 전 세계 어디든 보낼 수 있는 통로를 설계하는 단계입니다.
1. 소켓(Socket)의 개념: 디지털 통신의 접점
소켓은 네트워크상의 두 프로그램이 서로 데이터를 주고받을 수 있도록 연결해 주는 **소프트웨어적인 문(Gate)**입니다.
 * IP 주소: 데이터가 찾아갈 '집 주소' (어느 컴퓨터인가?)
 * Port 번호: 데이터가 들어갈 '방 번호' (어느 프로세스/서비스인가?)
2. 핵심 통신 프로토콜: TCP vs UDP
데이터를 어떻게 보낼 것인지에 따라 두 가지 설계 방식을 선택해야 합니다. 이는 시스템의 목적에 따라 결정됩니다.
| 구분 | TCP (Transmission Control Protocol) | UDP (User Datagram Protocol) |
|---|---|---|
| 특징 | 연결 지향형, 신뢰성 보장 | 비연결형, 속도 중심 |
| 비유 | 전화를 걸어 상대가 받았는지 확인하며 대화 | 주소만 써서 빠르게 던지는 엽서 |
| 용도 | 웹 브라우징(HTTP), 파일 전송, 이메일 | 실시간 스트리밍, 온라인 게임, 화상 통화 |
| 핵심 | 데이터의 유실 없는 전달을 **'증명'**함 | 데이터의 실시간성을 최우선함 |
3. 실무 설계: [v4.3 Simple Echo Server & Client]
가장 기초적이면서도 강력한 'Echo(메아리) 시스템'을 설계해 보겠습니다. 클라이언트가 보낸 메시지를 서버가 받아 그대로 다시 돌려주는 구조입니다.
[Server Side: 기다리는 자]
import socket

# 1. 소켓 생성 (IPv4, TCP 방식)
server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# 2. 주소와 포트 결합 (로컬호스트, 9999번 포트)
server_socket.bind(('127.0.0.1', 9999))

# 3. 연결 대기 상태
server_socket.listen()
print("📡 서버가 가동되었습니다. 연결을 기다리는 중...")

while True:
    # 4. 클라이언트 접속 허용
    client_socket, addr = server_socket.accept()
    print(f"🤝 접속 완료: {addr}")
    
    # 5. 데이터 수신 및 재전송
    data = client_socket.recv(1024)
    print(f"📥 수신 데이터: {data.decode()}")
    client_socket.sendall(data) # 다시 돌려줌 (Echo)
    
    client_socket.close()

🎓 대학교 2학년 마지막 전공 심화 팁
 * Loopback Address (127.0.0.1): '나 자신'을 가리키는 특수 IP입니다. 외부 망 연결 없이 내 컴퓨터 안에서 서버와 클라이언트를 테스트할 때 필수입니다.
 * Blocking vs Non-blocking: 기본 소켓은 데이터를 받을 때까지 프로그램이 멈춥니다(Blocking). 이를 해결하기 위해 앞서 배운 멀티프로세싱이나 Asyncio를 결합하면 수만 명의 접속을 동시에 처리하는 '고성능 서버'가 탄생합니다.
