🎓 Go-언어 전공 101: 1-6. 맵(Map)과 구조체(Struct)
데이터를 단순히 나열하는 것(Slice)을 넘어, 의미 있는 단위로 묶고 효율적으로 검색하는 방법을 배웁니다.
1. 맵 (Map): 키-값 기반의 고속 검색
맵은 해시 테이블(Hash Table) 기반의 자료구조입니다. 특정 키를 통해 데이터에 즉시 접근할 수 있습니다.
 * 선언: map[KeyType]ValueType 형식입니다.
 * 초기화: make 함수를 사용하는 것이 안전합니다. (nil 맵에 쓰기를 시도하면 런타임 에러가 발생합니다.)
<!-- end list -->
// 학생 이름을 키로, 학점을 값으로 가지는 맵
grades := make(map[string]int)
grades["Alice"] = 95
grades["Bob"] = 88

// 데이터 확인과 존재 여부 체크 (Comma ok idiom)
score, ok := grades["Charlie"]
if !ok {
    fmt.Println("데이터가 없습니다.")
}

2. 구조체 (Struct): 데이터의 집합체
Go에는 클래스(Class)가 없습니다. 대신 **구조체(Struct)**를 통해 서로 다른 타입의 데이터들을 하나로 묶어 '객체'처럼 관리합니다.
type Student struct {
    Name  string
    ID    int
    Major string
    GPA   float64
}

// 구조체 인스턴스 생성
s1 := Student{Name: "Gopher", ID: 20260101, Major: "CS", GPA: 4.5}

3. 구조체 임베딩 (Embedding): 상속 대신 합성
Go는 상속(Inheritance)을 지원하지 않습니다. 대신 구조체 안에 다른 구조체를 포함시키는 **'임베딩'**을 통해 기능을 확장합니다. 이는 "상속보다는 합성(Composition over Inheritance)"이라는 대학원 수준의 설계 원칙을 충실히 따르는 방식입니다.
🔬 전공 심화: 메모리 정렬 (Memory Alignment)
대학 과정에서는 구조체의 내부 메모리 구조까지 들여다봅니다. 구조체의 필드 순서에 따라 메모리에서 차지하는 크기가 달라질 수 있다는 사실을 아시나요?
 * Go 컴파일러는 CPU의 메모리 접근 효율을 위해 '패딩(Padding)'을 넣습니다.
 * Tip: 큰 타입(int64, float64 등)을 앞쪽에 배치하면 구조체의 전체 크기를 줄여 메모리 효율을 높일 수 있습니다. (시스템 프로그래밍의 묘미입니다.)
📝 Assignment 1-6 (설계 및 최적화)
