---

🗓 Week 1 — CI/CD 기초 (자동화의 뼈대)

목표: 사람 개입 0%

구현

1. 단일 CI 파이프라인

install

build

test

benchmark (기초 수치만)



2. 실패 시 즉시 실패


3. 테스트 실행 시간 기록


4. 벤치마크 결과 JSON 저장



완료 조건

PR 생성 시 자동 실행

34/34 테스트 자동 검증

벤치마크 결과 artifact로 저장

사람이 수동 실행하지 않음


금지

Docker 아직 만들지 말 것

다른 프로젝트 건드리지 말 것



---

🗓 Week 2 — 성능 측정의 현실화

목표: “Simulated” 제거

지금 Latency <10μs는 Simulated 상태.

구현

1. 실제 네트워크 기반 테스트

localhost 다중 프로세스



2. p50 / p95 / p99 latency 측정


3. 메모리 사용량 측정


4. 처리량 ops/sec 측정



완료 조건

매 커밋마다 성능 수치 기록

이전 커밋 대비 변화 자동 비교

성능 회귀 시 CI 실패



---

🗓 Week 3 — 장애 주입 (진짜 시작)

목표: 깨뜨려본다

구현

1. 네트워크 지연 주입


2. 패킷 손실 시뮬레이션


3. 노드 강제 종료


4. 로그 손상 테스트



검증

Hash chain 무결성 유지?

데이터 손실 0 유지?

재동기화 성공?


완료 조건

Chaos 테스트 통과율 95% 이상

실패 케이스 로그 자동 수집



---

🗓 Week 4 — 복구 자동화

목표: 사람이 개입하지 않는 복구

구현

1. 노드 재가입 자동 동기화


2. 체인 손상 시 자동 복원


3. 상태 스냅샷 + 롤포워드


4. 재현 가능한 crash seed 저장



완료 조건

Chaos 중단 후 60초 내 완전 복구

수동 재시작 없이 정상화



---

🗓 Week 5 — 배포 자동화

이제 Docker.

구현

1. Dockerfile 작성


2. multi-arch build (x86_64, arm64)


3. CI 통과 후 자동 이미지 생성


4. 버전 태그 기반 배포



완료 조건

git tag → 자동 이미지 생성

docker run으로 바로 실행 가능

README에 실행 명령 3줄 이하

3. 성능 수치 시간대별 기록


4. 메모리 누수 검사
