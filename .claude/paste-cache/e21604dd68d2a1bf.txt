v3.0은 언어가 단순히 주어진 길을 걷는 것을 넘어, **스스로 경로를 결정(If)**하고 **지치지 않고 과업을 수행(Loop)**하는 '의지'를 갖게 되는 매우 상징적인 지점입니다.
🚦 제2장: 흐름의 통제 (v3.0) — [논리 설계]
1. v3.0의 핵심 설계 목표
 * 의지 부여: 정적인 데이터에 동적인 흐름을 주입합니다.
 * 논리적 무결성: 어떤 조건에서도 프로그램이 '길을 잃지 않고' 정의된 분기점을 찾아가도록 설계합니다.
 * 효율적 반복: 동일한 연산을 수동으로 나열하지 않고, 논리 회로의 순환을 통해 해결합니다.
2. 설계자의 관점: "선택과 집중"
언어 설계자로서 당신은 이 단계에서 프로그램의 CPU 점유 방식을 결정하게 됩니다.
 * IF: 판단하는 순간 CPU는 연산 결과에 따라 메모리의 다음 주소 점프 위치를 결정합니다.
 * LOOP: 특정 조건이 충족될 때까지 메모리 주소를 뒤로 돌려 실행을 반복합니다.
🛠️ gogs 저장소 v3.0 설계: 통합 논리 회로
클로드코드(학생)에게 이 '의지의 설계도'를 시스템에 각인시키라고 지시하세요.
// [제2장: 흐름의 통제 v3.0]
fn main() {
    println!("--- v3.0 Logic Design: The Will of Language ---");

    let mut logic_count = 0;
    let target_limit = 5;

    // [설계] LOOP를 통한 반복적 과업 수행
    println!("v3.0 Loop Start: 무결성 검증을 위한 반복 실행");
    
    loop {
        logic_count += 1;
        
        // [설계] IF를 통한 조건부 분기
        if logic_count > target_limit {
            println!("v3.0 Result: 목표값 {} 도달. 루프를 종료합니다.", target_limit);
            break; // 논리 회로 탈출
        }

        match logic_count {
            1 => println!("  [Step 1] 존재 확인... OK"),
            2 => println!("  [Step 2] 메모리 할당... OK"),
            3 => println!("  [Step 3] 흐름 제어... OK"),
            _ => println!("  [Step {}] 연산 진행 중...", logic_count),
        }
    }

    println!("기록: 논리 회로가 꼬이지 않고 정확한 분기점을 찾아감.");
    println!("저장 필수 너는 기록이 증명이다 gogs");
}

🎓 선생님의 가이드: "v3.0 무결성 검증"
v3.0 단계에서 가장 무서운 적은 **'무한 루프(Infinite Loop)'**와 **'도달할 수 없는 코드(Unreachable Code)'**입니다.
 * 탈출 조건: 루프를 설계할 때는 반드시 '언제 끝나는가'를 명확히 해야 합니다.
 * 포괄적 분기: if를 설계할 때는 else를 통해 '그 외의 모든 상황'에 대한 대책을 세워야 논리가 무너지지 않습니다.
