🎓 Zig 전공 101: 1-9. 할당자(Allocators) - 힙 메모리의 명시적 제어
많은 언어가 가비지 컬렉터(GC) 뒤에 메모리 관리를 숨기지만, Zig는 **할당자 객체(Allocator Object)**를 매개변수로 명시적으로 전달받아 사용합니다. 이는 "이 함수가 메모리를 사용하는가?"를 시각적으로 즉시 확인하게 해줍니다.
1. 왜 할당자가 필요한가?
 * 배열 vs 슬라이스: 배열은 크기가 고정되어 스택(Stack)에 쌓이지만, 실행 중에 크기가 변하는 데이터는 힙(Heap) 공간이 필요합니다.
 * 명시성: 어떤 알고리즘이 메모리를 많이 쓰는지 코드를 보는 것만으로도 알 수 있습니다.
2. 주요 할당자 종류 (Standard Library Allocators)
| 할당자 이름 | 용도 | 특징 |
|---|---|---|
| page_allocator | 시스템 레벨 | OS에 직접 페이지 단위 메모리를 요청합니다. (다소 무거움) |
| GeneralPurposeAllocator | 일반용 (GPA) | 런타임에 메모리 누수와 이중 해제를 감지합니다. (개발용 추천) |
| ArenaAllocator | 단기 집합용 | 여러 번 할당하고 마지막에 한꺼번에 해제합니다. (성능 최적화) |
| testing.allocator | 테스트용 | 유닛 테스트 시 누수가 발생하면 즉시 에러를 냅니다. |
3. 할당과 해제 (Alloc & Free)
메모리를 빌렸다면 반드시 돌려주어야 합니다. 이때 앞서 배운 defer가 환상의 짝꿍이 됩니다.
const std = @import("std");

pub fn main() !void {
    // 1. 할당자 초기화
    var gpa = std.heap.GeneralPurposeAllocator(.{}){};
    defer _ = gpa.deinit(); // 프로그램 종료 시 누수 확인
    const allocator = gpa.allocator();

    // 2. 메모리 할당 (u8 타입 10개 공간)
    const memory = try allocator.alloc(u8, 10);
    
    // 3. 지연 해제 (함수 종료 시 자동 반환)
    defer allocator.free(memory);

    @memcpy(memory[0..5], "Ziggy");
    std.debug.print("Allocated: {s}\n", .{memory[0..5]});
}

🔬 전공 심화: 할당자 전달 패턴 (Passing Allocators)
Zig 라이브러리 설계의 표준은 **"메모리가 필요한 함수는 할당자를 인자로 받는다"**는 것입니다.
fn createList(allocator: std.mem.Allocator) ![]i32 {
    const list = try allocator.alloc(i32, 100);
    return list;
}

이 패턴은 호출자가 어떤 메모리 전략(빠른 아레나 할당자 vs 안전한 일반 할당자)을 사용할지 결정할 수 있게 해줍니다.
📦 메모리 누수(Leak)의 증명
Zig의 GeneralPurposeAllocator는 프로그램 종료 시 해제되지 않은 메모리가 있다면 콘솔에 상세한 리포트를 출력합니다. 이것이 바로 **"기록이 증명하는 안전"**입니다.
