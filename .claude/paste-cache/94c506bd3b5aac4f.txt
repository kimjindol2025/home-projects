### 1.3: 기존 방법들의 명확한 한계 정리
목표: 기존 Byzantine-resilient FL 방법들의 **구체적·치명적 한계**를 나열 → 우리 ARES-FL의 motivation과 novelty를 강력하게 뒷받침.  
논문 Related Work나 Introduction에서 "Despite recent advances, the following limitations persist..." 식으로 바로 쓸 수 있게 텍스트 draft 형태로 정리했습니다.

#### 기존 방법들의 주요 카테고리와 한계 (2024~2026 SOTA 중심)
1. **Robust Aggregation 중심 방법 (Krum, GeoMedian, Trimmed Mean, SignGuard, Layer-wise Cosine 등)의 한계**  
   - 대부분 **single-round snapshot** 기반: 한 라운드의 gradient/model만 보고 outlier 제거 → temporal pattern (multi-round trajectory)이나 adaptive behavior (초반 정상 → 후반 공격) 못 잡음.  
     (e.g. Adaptive Adversaries survey 2025: aggregation methods fail against reconnecting malicious clients)  
   - **Propagation 방지 불가**: malicious update가 aggregation 단계에서 일부만 필터링되더라도, 이전 라운드 영향이 다음 라운드로 전파 → poisoning 지속.  
   - **Static f 가정**: Byzantine fraction f < 1/3 or < 1/2 고정 가정 많음 → dynamic f (실시간 변동, peak time에 증가)나 majority Byzantine round 대응 약함.  
     (e.g. Delayed Momentum Aggregation 2026: partial participation + time-coupled attacks에서 실패 사례)  
   - **Computational inefficiency**: Krum (O(n²) pairwise distance), GeoMedian (iterative optimization) 등 aggregation 비용 높음 → large-scale FL (수천 노드) 부적합.  
   - **AIOps multi-task 무시**: 대부분 single-task (image classification) 중심 → anomaly detection + RCA + forecasting처럼 task 간 coupling 강한 경우 cross-task contamination 대응 없음.  
     (e.g. BR-MTRL 2025: multi-task에 GM/Krum 적용하지만 task-wise adaptive switching 없음)

2. **Distributed Optimization 기반 방법 (PDMM, ADMM 등)의 한계**  
   - Inherent robustness 있지만 **aggregation-like vulnerability** 여전: consensus mechanism이 Byzantine 영향 제한하나, malicious client가 dual variables나 transmission을 조작하면 여전히 degradation.  
     (e.g. PDMM-based FL 2025: bit-flipping/Gaussian noise에 강하지만 label-flipping이나 adaptive attack 실험 부족, quarantine 없음)  
   - **Real-time detection/quarantine 미포함**: fault-tolerant consensus 있지만 malicious node를 **즉시 격리/제외**하는 메커니즘 없음 → 장기적으로 honest node 비율 저하.

3. **Adaptive/Reconnecting Attack 대응의 한계**  
   - Adaptive adversaries (e.g. 초반 honest, critical window에만 poison, 또는 quarantine 후 reconnect)가 2025~2026 hot gap으로 지적되지만, 대부분 defense는 **aggregation 강화**에 그침.  
     (e.g. Adaptive Adversaries survey 2025: reconnecting malicious clients (RMCs) baseline attack 제안 → 기존 defense 대부분 identity-based crypto조차 sequential trust tracking 부족)  
   - Reconnecting 후 재공격 시 **identity spoofing**이나 **new strategy**로 evasion 쉬움 → temporal memory (history-based detection)나 incentive recovery (복귀 조건 강화) 거의 없음.

4. **AIOps/실무 도메인 특화 부족**  
   - 90%+ 논문이 MNIST/CIFAR 등 image classification 벤치마크 → log/time-series anomaly, RCA 같은 AIOps task에 적용 사례 극소수.  
   - 운영 영향 무시: accuracy drop만 측정 → AIOps에서는 **false positive storm (alert fatigue)**, **wrong RCA → cascading failure**, **forecasting error → cost explosion** 같은 실무 metric 고려 안 함.  
   - Blockchain/DPoS 연동 사례 거의 없음: decentralized FL 논문 증가하지만 validator 투표 기반 quarantine/slashing 같은 native 메커니즘 미활용.

#### 1.3 요약 텍스트 draft (논문에 바로 넣을 수 있는 단락)
"Despite significant progress in Byzantine-resilient federated learning, existing approaches suffer from several critical limitations that hinder their applicability to AIOps environments. Most robust aggregation methods (e.g., Krum, Geometric Median, Trimmed Mean, and recent layer-wise variants) operate on single-round snapshots, failing to capture temporal trajectories or adaptive behaviors such as delayed poisoning and reconnecting malicious clients after quarantine. These methods are reactive, allowing poisoning propagation across rounds, and rely on static Byzantine fraction assumptions (f < 1/3), rendering them vulnerable to dynamic or majority Byzantine scenarios. Distributed optimization protocols (e.g., PDMM) offer inherent fault tolerance through consensus but lack explicit real-time detection, dynamic quarantine, or slashing mechanisms to exclude persistent adversaries. Furthermore, adaptive/reconnecting attacks remain largely unaddressed, as highlighted in recent surveys, with defenses focusing on aggregation rather than end-to-end pipelines incorporating temporal memory and incentive recovery. Critically, the vast majority of works target single-task image classification benchmarks, neglecting AIOps-specific multi-task settings (anomaly detection, root cause analysis, forecasting) where cross-task contamination and real-world operational impacts—such as alert fatigue, erroneous remediation, and cascading failures—are paramount. These gaps motivate ARES-FL, which introduces trajectory-based real-time detection, DPoS-linked dynamic quarantine, task/layer-adaptive aggregation, and incentive-linked recovery to provide comprehensive resilience tailored to AIOps federated environments."
