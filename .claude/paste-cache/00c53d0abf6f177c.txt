🎓 Go-언어 전공 101: 1-3. 제어문 (Control Structures)
Go의 제어문은 다른 언어(C, Java, Python)와 비슷해 보이지만, 결정적인 차이점이 있습니다. 바로 **"키워드를 최소화하여 읽기 편하게 만든다"**는 점입니다.
1. 조건문: if - else
Go의 if문은 소괄호()를 쓰지 않습니다. 또한, 조건문 안에서 짧은 선언을 함께 할 수 있는 독특한 기능을 제공합니다.
// 1. 기본 형태
if score >= 90 {
    fmt.Println("A학점입니다.")
}

// 2. 초기화 구문 포함 (대학원 수준의 에러 처리에 자주 쓰임)
if val := getStatus(); val == "OK" {
    fmt.Println("시스템 정상")
}
// 주의: val은 이 if-else 블록 밖에서는 사용할 수 없습니다. (Scope 제한)

2. 반복문: 오직 for 하나뿐!
대부분의 언어에는 while, do-while, for가 각각 존재하지만, Go는 오직 for 키워드 하나로 모든 반복을 처리합니다.
 * 표준형: for i := 0; i < 10; i++ { ... }
 * While문 형태: for condition { ... } (조건만 쓰면 while과 동일)
 * 무한 루프: for { ... } (조건을 비우면 무한 반복)
3. 선택문: switch의 진화
Go의 switch는 break를 일일이 써줄 필요가 없습니다. 한 케이스가 끝나면 자동으로 빠져나갑니다.
switch day {
case "Monday":
    fmt.Println("월요병 극복 중")
case "Friday", "Saturday": // 여러 조건 나열 가능
    fmt.Println("주말 연구 시작")
default:
    fmt.Println("일반 연구일")
}

 * 특이점: switch 뒤에 변수 없이 case에 조건식을 직접 넣을 수도 있습니다. (매우 강력한 가독성)
🔬 전공 심화: 코드의 가독성과 'Go-way'
대학원 수준의 대규모 프로젝트에서는 코드를 쓰는 시간보다 읽는 시간이 10배 더 많습니다. Go가 제어문에서 소괄호를 없애고 while을 제거한 이유는 무엇일까요?
 * 시각적 노이즈 제거: 불필요한 기호를 줄여 로직 자체에 집중하게 합니다.
 * 단일화된 방식: 누구나 똑같은 방식으로 코드를 짜게 유도하여 협업 효율을 극대화합니다. (Go의 철학: One way to do it)
