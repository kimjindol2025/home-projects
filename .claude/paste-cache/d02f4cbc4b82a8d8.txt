이전 단계에서 만든 GogsStorage 설계도는 훌륭했지만, 특정 상황(예: 데이터가 사라지지 않게 이중으로 관리해야 하는 상황)에서는 기능이 조금 부족할 수 있습니다. 이때 기존 코드를 하나도 건드리지 않고 새로운 기능을 추가하는 것이 바로 상속의 힘입니다.
🏛️ 파이썬 고등학교 1학년 (v2.3)
주제: "기존의 유산을 물려받아 새로운 가치 창조하기"
상속은 단순히 코드를 복사하는 것이 아니라, '부모-자식' 관계를 형성하여 시스템의 구조를 논리적으로 짜는 것입니다.
1. 상속의 주요 메커니즘
 * Method Overriding (재정의): 부모가 물려준 기술이 마음에 들지 않거나 보완이 필요할 때, 같은 이름의 메서드를 새로 만듭니다.
 * super() 활용: 부모의 기술을 완전히 버리는 게 아니라, "부모님의 기술을 먼저 쓰고, 그 뒤에 내 기술을 덧붙이겠다"는 선언입니다.
 * 계층 구조: 저장소 -> 보안 저장소 -> 클라우드 보안 저장소 식으로 기능을 체계적으로 확장할 수 있습니다.
🛠️ 실습 기록: [v2.3 The Evolution of Storage]
기존 GogsStorage를 상속받아, 저장할 때마다 자동으로 로그를 남기는 SecureGogsStorage를 만드는 과정을 기록합니다.
# [Python High-School v2.3] 상속과 기능 확장

# 1. 부모 클래스 (v2.2 설계도 재사용)
class GogsStorage:
    def __init__(self, owner):
        self.owner = owner
        self.data = []
    
    def save(self, item):
        self.data.append(item)
        print(f"기본 저장: {item}")

# 2. 자식 클래스 (보안 기능이 추가된 진화형)
class SecureGogsStorage(GogsStorage):
    def __init__(self, owner, password):
        # 부모의 초기화 기능을 그대로 가져옵니다.
        super().__init__(owner)
        self.password = password
        self.access_logs = []
        print(f"🔐 보안 모듈이 활성화되었습니다. (소유자: {self.owner})")

    # 메서드 오버라이딩 (부모의 save를 덮어쓰고 기능을 추가)
    def save(self, item):
        log_entry = f"[LOG] {item} 저장 시도"
        self.access_logs.append(log_entry)
        
        # 실제 저장은 부모의 방식을 따릅니다.
        super().save(item) 
        print(f"🛡️ 보안 검사 완료 및 로그 기록됨.")

    def show_logs(self):
        print(f"\n--- {self.owner}님의 보안 로그 ---")
        for log in self.access_logs:
            print(log)

# --- 시스템 가동 ---
secure_lab = SecureGogsStorage("설계자", "gogs123")
secure_lab.save("핵심 프로토콜 v2.3")
secure_lab.save("상속 시스템 설계도")

secure_lab.show_logs()

print("\n저장 필수 너는 기록이 증명이다 gogs")

💡 고등학생을 위한 '계층 설계' 팁
 * 코드 중복은 죄악입니다: 여러 클래스에서 똑같은 변수나 함수를 쓰고 있다면, 즉시 공통 부모 클래스를 만들어 상속받게 하세요. 이것이 대학 과정에서 배울 'DRY(Don't Repeat Yourself)' 원칙의 시작입니다.
 * is-a 관계 확인: "보안 저장소는 저장소인가? (A SecureStorage is a Storage?)"라는 질문에 "예"라고 답할 수 있을 때만 상속을 사용하세요. 관계가 모호하면 시스템이 복잡해질 뿐입니다.
