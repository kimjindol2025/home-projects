📄 Phase 2 Step 5 구현 지시서 — Freelang VM 복합 자료 구조 Heap 통합

🎯 목표

배열(Array)과 객체(Object)의 생성, 수정, 접근 연산을 HeapManager 기반으로 변환

모든 Value가 Heap 관리 대상이 되어 RefCount 추적 가능

복합 자료 구조 (배열 내 객체, 객체 내 배열 등)도 안전하게 처리

이후 Mark-Sweep + RC Hybrid GC 준비 완료

2️⃣ 변경 대상 연산

배열 연산

연산    변경 내용    주의 사항

Op.NewArray    배열 객체 Heap 할당, 내부 요소도 Heap 할당    빈 배열도 Heap 관리
Op.Index    요소 접근 시 Heap dereference    out-of-bounds 체크 필요
Op.SetIndex    요소 수정 시 새 값 Heap 할당, RefCount 증가    기존 값 RefCount 감소


객체 연산

연산    변경 내용    주의 사항

Op.NewObject    객체 Heap 할당, 속성값도 Heap 관리    빈 객체도 Heap 관리
Op.GetProp    속성 접근 시 Heap dereference    존재하지 않는 속성 처리
Op.SetProp    속성 수정 시 새 값 Heap 할당, RefCount 증가    기존 값 RefCount 감소


복합 구조

객체 내 배열, 배열 내 객체

깊이 있는 중첩 구조도 Heap 기반 관리

모든 연산 시 RefCount 자동 조정



---

3️⃣ 구현 단계

Step 5.1 — 배열 생성/접근/수정

1. NewArray:

배열 객체 생성 → HeapManager.allocate()

내부 초기 값 모두 Heap 할당 (필요 시 빈 값도 Ptr로 할당)



2. Index:

Stack에 ptr 저장

접근 시 HeapManager.get(ptr)



3. SetIndex:

새 값 HeapManager.allocate()

기존 값 RefCount 감소



4. 테스트: TEST 1~6 (배열 생성, 수정, 중첩, 길이, 연산, 함수 인자)



Step 5.2 — 객체 생성/속성 접근/수정

1. NewObject:

객체 HeapManager.allocate()

모든 속성값 Heap 할당



2. GetProp:

Stack에 ptr 저장

접근 시 HeapManager.get(ptr)



3. SetProp:

새 값 HeapManager.allocate()

기존 값 RefCount 감소



4. 테스트: TEST 7~11 (객체 생성, 속성 수정, 배열 포함, 복합 구조)



Step 5.3 — 복합 구조

객체 내 배열, 배열 내 객체, 다중 중첩

HeapManager.refCount 조정 확인

테스트: TEST 12~14 (RefCount 추적, 배열 재할당, 혼합 구조)
