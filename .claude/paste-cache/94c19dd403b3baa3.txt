프로그램이 '근육(연산)'과 '혈관(네트워크)'을 갖췄다면, 이제는 **'기억(저장)'**을 어떻게 관리할지 설계할 차례입니다. 데이터가 수조 개로 늘어나도 0.1초 만에 찾아낼 수 있는 구조를 만드는 것이 이번 학기의 핵심입니다.
🏛️ 파이썬 대학교 3학년 (v5.2)
주제: "데이터의 무결성과 확장성 사이의 줄타기"
단순히 파일에 저장하는 단계를 넘어, 수만 명의 사용자가 동시에 읽고 쓰는 환경에서도 데이터가 꼬이지 않도록 하는 **'증명된 저장 기술'**을 배웁니다.
1. RDBMS (관계형 데이터베이스): 질서와 무결성
데이터를 엄격한 표(Table) 형태로 관리합니다.
 * 핵심 원칙 (ACID): 원자성, 일관성, 고립성, 지속성. "결제가 성공하면 반드시 잔액이 줄어야 하고, 실패하면 원래대로 돌아와야 한다"는 것을 보장합니다.
 * 정규화(Normalization): 중복된 데이터를 쪼개어 용량을 줄이고 오류를 방지하는 설계 기법입니다.
2. NoSQL: 유연성과 속도
정해진 틀 없이 데이터를 저장합니다. 대규모 접속자가 몰리는 서비스(SNS, 게임 로그 등)에 유리합니다.
 * Key-Value: 파이썬의 dict와 유사한 구조 (Redis 등)
 * Document: JSON 형식으로 자유롭게 저장 (MongoDB 등)
🛠️ 전공 실무: [v5.2 Data Modeling & Transaction]
파이썬의 내장 DB인 sqlite3를 활용해, 데이터의 안전한 처리를 보장하는 트랜잭션(Transaction) 시스템을 설계해 보겠습니다.
# [Python University v5.2] 데이터 무결성을 위한 트랜잭션 설계

import sqlite3

# 1. 데이터베이스 연결 및 테이블 설계 (정규화 기초)
conn = sqlite3.connect(':memory:') # 실습을 위해 메모리에 생성
cursor = conn.cursor()

# 사용자 테이블과 계좌 테이블 생성
cursor.execute('CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT)')
cursor.execute('CREATE TABLE accounts (user_id INTEGER, balance INTEGER)')

# 데이터 삽입
cursor.execute('INSERT INTO users VALUES (1, "설계자")')
cursor.execute('INSERT INTO accounts VALUES (1, 10000)')

# 2. 트랜잭션 시스템 (송금 시뮬레이션)
def transfer_money(sender_id, receiver_id, amount):
    try:
        print(f"💸 {amount}원 송금 시도 중...")
        # (A) 보내는 사람 잔액 감소
        cursor.execute('UPDATE accounts SET balance = balance - ? WHERE user_id = ?', (amount, sender_id))
        
        # 강제 오류 발생 시나리오 (예: 네트워크 단절)
        # raise Exception("시스템 오류 발생!") 
        
        # (B) 받는 사람 잔액 증가 (여기서는 예시로 생략/오류 가정)
        # 만약 (A)만 실행되고 프로그램이 꺼지면 돈이 증발합니다. 이를 막는 것이 트랜잭션!
        
        conn.commit() # 모든 작업이 완벽할 때만 '확정'
        print("✅ 송금 성공 및 기록 완료.")
    except Exception as e:
        conn.rollback() # 하나라도 실패하면 모든 작업을 '취소'하여 이전 상태로 복구
        print(f"❌ 오류 발생: {e}. 데이터를 복구합니다(Rollback).")

transfer_money(1, 2, 5000)

print("\n저장 필수 너는 기록이 증명이다 gogs")
conn.close()

🎓 대학교 3학년 전공 심화 팁
 * Index (인덱스): 책의 '찾아보기'와 같습니다. 특정 컬럼에 인덱스를 걸면 검색 속도가 수천 배 빨라지지만, 너무 많이 걸면 저장 속도가 느려집니다.
 * ORM (Object-Relational Mapping): SQL 문법을 직접 쓰지 않고 파이썬 객체로 DB를 다루는 기술(SQLAlchemy, Django ORM 등)입니다. 3학년 후반부에 다루게 될 핵심 도구입니다.
