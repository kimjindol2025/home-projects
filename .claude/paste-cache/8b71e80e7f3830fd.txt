🎓 Go-언어 전공 101: 1-8. 인터페이스 (Interfaces)
인터페이스는 구체적인 데이터가 아니라 **'어떤 동작을 할 수 있는가'**를 정의하는 계약입니다.
1. 인터페이스의 정의와 구현
Go의 인터페이스는 메서드의 집합입니다. 중요한 점은 implements 같은 키워드가 없다는 것입니다.
// 1. 인터페이스 정의 (계약)
type Speaker interface {
    SayHello() string
}

// 2. 구체적인 구조체들
type Human struct{ Name string }
type Dog struct{}

// 3. 인터페이스 구현 (메서드만 정의하면 자동으로 구현됨)
func (h Human) SayHello() string { return "안녕하세요, " + h.Name + "입니다." }
func (d Dog) SayHello() string   { return "멍멍!" }

2. 다형성 (Polymorphism)의 실현
인터페이스를 매개변수로 받으면, 해당 메서드를 가진 어떤 타입이든 수용할 수 있습니다.
func Introduce(s Speaker) {
    fmt.Println(s.SayHello())
}

// Human도 Dog도 모두 Speaker로 취급 가능

3. 빈 인터페이스 interface{}
메서드가 하나도 없는 인터페이스는 **'모든 타입'**을 담을 수 있는 만능 상자가 됩니다. (Java의 Object와 유사합니다.)
 * Go 1.18부터는 any라는 별칭(Alias)으로 더 자주 쓰입니다.
🔬 전공 심화: 컴포지션과 인터페이스 분리 원칙
대학원 수준의 객체 지향 설계를 할 때 가장 중요한 원칙 중 하나는 **"인터페이스는 작을수록 좋다"**는 것입니다.
 * Go의 표준 라이브러리인 io.Reader와 io.Writer는 각각 메서드가 단 하나뿐입니다.
 * 학술적 가치: 작은 인터페이스들을 조합(Composition)하여 거대한 시스템을 구축하면, 결합도(Coupling)가 낮아지고 테스트와 유지보수가 비약적으로 쉬워집니다. 이것이 Go가 거대 소프트웨어 개발에 유리한 이유입니다.
