🏛️ LLVM 전설: 2.4 코드 생성 - AST에서 기계의 언어로
이제 Kim님이 작성하신 src/codegen/CodeGenerator.cpp는 Annotated AST(타입 정보가 기록된 트리)를 훑으며 실제 LLVM IR 명령어를 쏟아내야 합니다. 1단계(1.1~1.5)에서 배운 기초 체력이 여기서 빛을 발합니다.
1. 전역 문맥(Global Context) 설정
컴파일러의 진입점에서 LLVM의 기본 객체들을 초기화합니다.
 * llvm::LLVMContext: 전역 상태 관리.
 * llvm::Module: Z-Lang 소스 파일 하나에 대응하는 컨테이너.
 * llvm::IRBuilder<>: 명령어를 생성하고 삽입하는 '커서'.
2. 함수 및 블록 생성 (Function & Block Gen)
Z-Lang의 fn 정의 노드를 만났을 때의 처리입니다.
 * 함수의 이름을 바탕으로 llvm::Function을 생성합니다.
 * 함수의 시작 지점에 entry 베이직 블록을 만들고 빌더를 배치합니다.
 * 전설의 한 끗: 실시간 시스템을 위해 함수의 스택 크기나 정렬(Alignment) 정보를 하드웨어 타겟에 맞춰 명시적으로 설정합니다.
3. 변수 할당과 스택 관리
let x = 10;과 같은 선언을 처리할 때, LLVM의 alloca 명령어를 사용합니다.
 * Memory Management: Z-Lang은 소유권(Ownership) 기반이므로, 변수가 스코프를 벗어날 때(popScope) 해당 메모리가 안전하게 처리되도록 drop 로직이나 store 명령어를 정교하게 배치해야 합니다.
🔬 전공 심화: Z-Lang 특화 최적화 (Real-Time Optimization)
일반적인 컴파일러는 '평균 속도'를 높이려 하지만, Z-Lang은 **'예측 가능성(Predictability)'**을 높여야 합니다.
 * Loop Unrolling: 루프 횟수가 고정된 경우, LLVM Pass를 통해 루프를 펼쳐서 분기 예측 실패(Branch Misprediction)를 원천 차단합니다.
 * Inline Assembly: Z-Lang 코드 내에서 직접 하드웨어 레지스터를 건드릴 수 있도록 LLVM의 인라인 어셈블리 인터페이스를 연결합니다.
📦 기록의 가치: zlang --emit-llvm
이제 Kim님의 컴파일러는 사용자가 작성한 .z 파일을 읽어 .ll (LLVM IR 텍스트) 파일을 뱉어내야 합니다. 이 .ll 파일이 바로 Kim님이 창조한 언어가 논리적으로 완벽함을 증명하는 **'기록의 정수'**입니다.
