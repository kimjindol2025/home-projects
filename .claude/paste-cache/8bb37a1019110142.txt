🎓 Zig 전공 201: 2-1. 고성능 네트워크 프로그래밍 (TCP/UDP)
대규모 시스템 아키텍처에서 가장 중요한 것은 "데이터가 어떻게 이동하는가"입니다. Zig는 운영체제의 소켓(Socket) API를 직접 다루면서도, 현대적인 안전성을 결합하여 초고성능 서버를 설계할 수 있게 해줍니다.
1. 소켓(Socket) 아키텍처의 이해
Zig의 std.net 패키지는 저수준 시스템 호출(Syscall)을 추상화하여 제공합니다.
 * StreamServer: TCP 연결을 수락(Accept)하고 관리하는 서버 객체입니다.
 * Address: IP 주소와 포트를 결합하여 네트워크 상의 위치를 특정합니다.
2. 에코 서버(Echo Server) 설계 실습
가장 기본이 되는 서버 구조를 통해 데이터의 수신과 송신을 제어해 봅시다.
const std = @import("std");
const net = std.net;

pub fn main() !void {
    const address = try net.Address.parseIp("127.0.0.1", 8080);
    var server = net.StreamServer.init(.{ .reuse_address = true });
    defer server.deinit();

    try server.listen(address);
    std.debug.print("서버가 {any}에서 대기 중입니다...\n", .{address});

    while (true) {
        // 클라이언트 접속 대기
        const connection = try server.accept();
        defer connection.stream.close();

        std.debug.print("클라이언트 접속: {any}\n", .{connection.address});
        
        // 데이터 읽기/쓰기
        var buffer: [1024]u8 = undefined;
        const bytes_read = try connection.stream.read(&buffer);
        try connection.stream.writeAll(buffer[0..bytes_read]);
    }
}

🔬 전공 심화: 논블로킹(Non-blocking) I/O와 이벤트 루프
수만 명의 사용자가 동시에 접속하는 시스템에서는 스레드 하나당 연결 하나를 할당하는 방식은 한계가 있습니다.
 * 문제: 스레드가 너무 많아지면 컨텍스트 스위칭 비용이 커져 시스템이 느려집니다.
 * 해결: epoll (Linux) 이나 kqueue (BSD/macOS) 같은 OS의 이벤트 알림 시스템을 사용하여, 데이터가 도착했을 때만 CPU가 일하도록 설계합니다.
📦 데이터 직렬화 (Serialization)
네트워크를 통해 구조체를 보낼 때는 데이터를 바이트 스트림으로 변환해야 합니다. Zig의 std.json이나 comptime을 활용한 커스텀 바이너리 포맷 설계를 통해 성능 손실 없는 통신을 기록하겠습니다.
