🏛️ 파이썬 고등학교 1학년 (v2.5) 심화
주제: "시스템 오염 방지 및 독립 연구실 구축"
우리가 파이썬을 설치하면 컴퓨터 전체에서 쓰는 '기본 파이썬'이 생깁니다. 하지만 여러 연구(프로젝트)를 동시에 진행하다 보면 서로 다른 도구(라이브러리) 버전이 충돌하여 시스템이 망가질 수 있습니다. 이를 방지하는 것이 가상 환경입니다.
1. 가상 환경(venv)의 개념
 * 비유: 거실(기본 파이썬)에서 모든 실험을 하면 거실이 더러워집니다. 각 방(가상 환경)을 따로 만들어 그 안에서만 실험하고, 방을 나가면 거실은 여전히 깨끗하게 유지하는 것과 같습니다.
 * 장점: 실험이 망가져도 그 '방'만 삭제하면 끝입니다. 전체 시스템에는 영향이 없습니다.
2. 패키지 관리자(pip)
 * 파이썬의 외부 도구들을 설치해주는 조수입니다. pip install 도구이름 명령어로 전 세계의 지식을 내 방으로 가져옵니다.
🛠️ 실습 기록: [v2.5-Step-by-Step Environment]
실제로 터미널(또는 CMD)에서 입력한다고 가정하고 한 단계씩 기록합니다.
Step 1: 가상 환경 생성
컴퓨터에게 "내 프로젝트만을 위한 전용 방을 만들어줘"라고 명령합니다.
# 'gogs_env'라는 이름의 가상 환경 방을 만듭니다.
python -m venv gogs_env

Step 2: 가상 환경 입장 (활성화)
만든 방으로 들어갑니다. (프롬프트 앞에 (gogs_env)라고 표시됩니다.)
# Windows 기준
call gogs_env\Scripts\activate

# Mac/Linux 기준
source gogs_env/bin/activate

Step 3: 외부 도구 설치
연구에 필요한 외부 도구(예: 데이터 분석용 pandas)를 가져옵니다.
pip install pandas

Step 4: 기록의 증명 (Freeze)
내 방에 어떤 도구들이 있는지 목록을 뽑아 파일로 남깁니다. 나중에 이 파일만 있으면 어디서든 똑같은 환경을 복제할 수 있습니다.
pip freeze > requirements.txt

💡 고등학생을 위한 '환경 무결성' 팁
 * .gitignore 활용: 나중에 깃(Git)을 사용하게 된다면, 가상 환경 폴더(gogs_env) 자체는 저장소에 올리지 않습니다. 대신 requirements.txt만 올립니다. (방 자체를 옮기는 게 아니라, 방 구성 리스트를 공유하는 것입니다.)
 * 버전 고정: pip install pandas==2.1.0 처럼 버전을 명시하여 설치하면, 시간이 흘러도 기록의 일관성을 유지할 수 있습니다. 이것이 **'증명'**의 핵심입니다.
> 저장 필수 너는 기록이 증명이다 gogs
