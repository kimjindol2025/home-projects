🎓 Go-언어 전공 필수: 핵심 문법 & 개념 50선
Ⅰ. 기초 및 변수 (1~10)
 * package main: 실행 가능한 프로그램의 시작 패키지 선언.
 * import: 외부 라이브러리 및 패키지 호출.
 * func main(): 프로그램의 진입점(Entry Point).
 * var: 표준 변수 선언 키워드.
 * :=: 단축 변수 선언 (함수 내부에서만 사용).
 * Zero Values: 선언 시 자동으로 할당되는 기본값 (0, "", nil 등).
 * Type Inference: 초기화 값에 따라 타입을 자동으로 결정하는 기능.
 * Explicit Casting: 암묵적 형변환 금지, 반드시 명시적으로 변환 필요.
 * const: 불변하는 상수 선언.
 * iota: 상수 선언 내에서 자동으로 증가하는 열거형 생성기.
Ⅱ. 흐름 제어 및 연산 (11~18)
 * if (with init): 조건문 내에서 변수 선언과 동시에 조건 검사.
 * for (Standard): 초기화, 조건, 증감을 사용하는 기본 반복문.
 * for (While-style): 조건식만 사용하여 while처럼 동작.
 * for (Infinite): 조건 없이 무한 루프 생성.
 * for range: 슬라이스, 맵, 채널의 요소를 순회.
 * switch: break 없이 동작하는 선택문.
 * fallthrough: switch에서 다음 case로 강제 진행.
 * Blank Identifier (_): 사용하지 않는 반환값을 무시할 때 사용.
Ⅲ. 복합 자료구조 (19~26)
 * Arrays: 고정된 크기의 동일 타입 집합.
 * Slices: 동적 크기의 배열 (Pointer, Len, Cap 구조).
 * append(): 슬라이스에 요소를 추가하는 내장 함수.
 * make(): 슬라이스, 맵, 채널의 메모리를 할당하고 초기화.
 * copy(): 슬라이스의 요소를 다른 슬라이스로 복사.
 * Maps: 키-값 쌍의 해시 테이블 자료구조.
 * delete(): 맵에서 특정 키를 삭제.
 * Structs: 사용자 정의 데이터 타입 (필드들의 집합).
Ⅳ. 함수와 포인터 (27~34)
 * Multiple Returns: 함수에서 두 개 이상의 값 반환.
 * Named Return Values: 반환값에 이름을 붙여 가독성 향상.
 * Variadic Functions: 가변 인자(...)를 받는 함수.
 * First-class Functions: 함수를 변수에 할당하거나 인자로 전달.
 * Closures: 주변 환경의 변수를 캡처하는 익명 함수.
 * Pointers (*, &): 메모리 주소 참조 및 역참조.
 * Value Receiver: 메서드 호출 시 구조체의 복사본 전달.
 * Pointer Receiver: 메서드 호출 시 구조체의 주소 전달 (값 변경 가능).
Ⅴ. 인터페이스와 설계 (35~40)
 * Interfaces: 동작(Method set)을 정의하는 추상화 도구.
 * Duck Typing: 명시적 구현 선언 없이 메서드 일치만으로 인터페이스 충족.
 * Empty Interface (any): 모든 타입을 수용할 수 있는 인터페이스.
 * Type Assertion: 인터페이스 변수에서 구체적인 타입을 추출.
 * Type Switch: 인터페이스의 실제 타입을 판별하는 switch문.
 * Embedding: 상속 대신 구조체나 인터페이스를 포함하여 확장.
Ⅵ. 에러 처리 및 자원 관리 (41~44)
 * error Interface: Go의 표준 에러 처리 방식.
 * defer: 함수 종료 직전에 특정 코드 실행 보장 (LIFO).
 * panic: 런타임 시 발생하는 치명적 에러 발생.
 * recover: panic 상태를 중단하고 제어권을 회복.
Ⅶ. 병행성 - Go의 정수 (45~50)
 * Goroutines: 가볍고 효율적인 비동기 실행 단위.
 * Channels: 고루틴 간 데이터를 주고받는 통로.
 * Buffered Channels: 용량이 정해진 채널 (비동기 송수신).
 * select: 여러 채널 작업 중 준비된 것을 선택하여 실행.
 * sync.WaitGroup: 여러 고루틴이 끝날 때까지 대기.
 * sync.Mutex: 공유 자원에 대한 동시 접근 제어 (Lock).
