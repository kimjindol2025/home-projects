### 1.2: 2024~2026 최신 Byzantine-resilient FL 논문 리뷰 & Gap 정리
목표: **기존 SOTA**를 체계적으로 훑어서 우리 ARES-FL의 독자적 기여가 어디에 위치하는지 명확히 보여줌.  
특히 AIOps 특화 (multi-task, time-series/log anomaly + RCA + forecasting), adaptive/reconnecting attack, trajectory-based detection, dynamic quarantine, task/layer adaptive aggregation에 초점.

#### 리뷰 방법
- 2024~2026 주요 키워드: Byzantine-resilient FL, adaptive adversaries, multi-task, trajectory detection, dynamic quarantine, reconnecting attack, layer-wise aggregation 등.
- 주요 출처: arXiv 최신 논문 (2025~2026 제출/게재), IEEE/NeurIPS/ICLR 등.
- 총 30+편 중 핵심 10~12편 추출 → 테이블로 Gap 분석.

#### Gap 분석 요약 (우리 ARES-FL의 Novelty 포인트 명확화)
1. **실시간 Trajectory-based Detection**: 대부분 single-round (cosine sim, magnitude). Gradient history/temporal pattern 활용 논문 적음 (Sony memorization 정도). LSTM-AE나 Temporal Isolation Forest 같은 multi-round trajectory anomaly detection은 아직 SOTA gap.
2. **DPoS-Linked Dynamic Quarantine & Slashing**: 거의 없음. Aggregation 시 outlier 제거(Krum, GeoMedian, layer-wise)가主流. Malicious node를 **실시간 격리/제외 + stake penalty**하는 blockchain-native 메커니즘은 신규 (특히 reconnecting attack 대응).
3. **Task-wise / Layer-wise Adaptive Aggregation**: Layer-wise는 나오고 있음 (2026 cosine). 하지만 **AIOps multi-task** (anomaly det + RCA + forecasting)에서 task별 f 추정 → switching은 전무. Cross-task propagation 무시.
4. **Adaptive/Reconnecting Attack 대응**: Survey(2025)에서 hot gap으로 지적되지만, 실질적 defense (quarantine + recovery boost)는 부족. 우리 incentive-linked recovery (정상 복귀 증명 필요)가 차별화.
5. **AIOps 도메인 특화**: Byzantine FL 논문 90%+가 image classification (CIFAR/MNIST) 중심. Log/time-series anomaly, RCA 같은 AIOps 적용 사례 매우 적음 → 운영 영향 (alert fatigue, downtime, cost explosion) 고려한 motivation 자체가 novelty.

**결과**:  
- 기존: Aggregation 중심, static f, single-task, single-round snapshot.  
- 우리 gap 채우기: End-to-end pipeline (trajectory detect → DPoS quarantine → task-adaptive agg → incentive recovery) + AIOps multi-task/time-series 특화 + dynamic f + reconnecting 대응.
