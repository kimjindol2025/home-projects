🎓 Go-언어 전공 101: 입문 및 철학
Go(Golang)는 구글에서 개발한 프로그래밍 언어로, **'복잡성과의 전쟁'**에서 승리하기 위해 탄생했습니다. 대학 과정의 첫 번째 수업으로 Go의 핵심 철학과 환경 설정을 다뤄보겠습니다.
1. Go의 탄생 배경과 핵심 철학
Go는 C++의 복잡함과 Java의 무거움, Python의 느린 속도를 해결하기 위해 만들어졌습니다.
 * Simplicity (단순성): 키워드가 25개 내외로 매우 적습니다. 코드를 읽기 쉽게 만드는 것이 최우선입니다.
 * Concurrency (병행성): '고루틴(Goroutine)'이라는 혁신적인 메커니즘을 통해 수만 개의 작업을 동시에 처리할 수 있습니다.
 * Efficiency (효율성): 컴파일 언어로서 실행 속도가 매우 빠르며, 정적 타입을 지원하여 안정성이 높습니다.
2. 환경 구축 (Lab Setting)
실제 연구를 시작하기 전, 실험실(개발 환경)을 세팅해야 합니다.
 * 설치: golang.org에서 OS에 맞는 버전을 설치합니다.
 * 버전 확인: 터미널에서 go version을 입력하여 설치를 확인합니다.
 * 작업 공간: Go 1.11 이후로는 Go Modules를 사용하므로, 원하는 폴더에서 시작하면 됩니다.
3. 첫 번째 실습: Hello, Gopher!
가장 기본적인 구조를 분석해 보겠습니다.
package main // 1. 패키지 선언

import "fmt" // 2. 라이브러리 임포트

// 3. 메인 함수 (프로그램의 시작점)
func main() {
    fmt.Println("대학 과정 Go 학습을 시작합니다.")
}

[코드 분석]
 * package main: 이 파일이 실행 가능한 프로그램의 시작점임을 알립니다.
 * import "fmt": 표준 입출력을 담당하는 'format' 패키지를 가져옵니다.
 * func main(): 프로그램 실행 시 가장 먼저 호출되는 함수입니다.
