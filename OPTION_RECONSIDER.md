# 🎯 옵션 A 재검토: 완전 자체호스팅의 현실

## 현황 분석

### 필요한 변환
```
TypeScript → FreeLang:
- index.ts (100줄) → interpreter.fl
- evaluator.ts (362줄) → interpreter.fl
- runtime.ts (?) → runtime.fl
- file.ts (493줄) → 파일 I/O 함수
- hashmap.ts (329줄) → 자료구조
- ... 기타 (5,631줄)
```

### 핵심 문제점
```
1. evaluator.ts는 복잡한 AST 재귀 순회 (Type annotations 많음)
2. Environment (Map 기반) 관리 필요
3. 함수 호출, 스코프 관리 등 고급 기능

2. FreeLang이 이런 기능을 지원하는가?
   - 재귀? ✅ 가능
   - Map/Dict? ⚠️ 제한적
   - 클래스/구조체? ⚠️ 확인 필요
   - Exception? ⚠️ 불명확
```

## 3가지 현실적 선택

### A1: 완전 포팅 (100%, 전문성 요구)
```
시간: 5-7일 (밤새우기 포함)
위험: 높음 (FreeLang 한계 도달 가능)
결과: 완전 FreeLang 자체호스팅

필요:
- evaluator.ts 논리를 FL로 완전히 재작성
- 모든 복잡한 부분 직접 구현
- 테스트 모두 재작성
```

### A2: 하이브리드 (90%, 현실적)
```
시간: 2-3일
위험: 낮음
결과: 거의 자체호스팅 (10% TS는 FFI)

방법:
- evaluator 핵심만 FL로 포팅
- 성능 임계값(JIT)은 TS 유지
- CLI 입출력은 FL로 새작성
```

### B: 현실 타협 (80%, 제시된 목표)
```
시간: 1-2일 ✅
위험: 거의 없음 ✅
결과: 목표 달성 ✅

방법:
- 불필요한 TS 파일만 제거
- 필수 기능만 최적화
- 기존 테스트 유지
```
